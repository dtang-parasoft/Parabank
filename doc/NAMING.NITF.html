Use a naming convention for interfaces [NAMING.NITF]



DESCRIPTION 


This rule identifies interfaces that do not follow the user-defined
naming convention (as specified in a regular expression). By default, the
rule requires that each interface name starts with a capital letter. An error
is reported for each occurrence.
SCOPE LEVEL 
CLASS
SINCE 

v4.5
PARAMETERS 

-Top-level interface
    -regular expression of top-level interface
     (Interface names should match the listed regular expression.
      Default is not set.)
    -check public top-level interface. (default is true)
    -check package-private top-level interface. (default is true)

-Member interface
    -regular expression of member interface
     (Interface names should match the listed regular expression.
      Default is not set.)
    -check public member interface. (default is true)
    -check protected member interface. (default is true)
    -check package-private member interface. (default is true)
    -check private member interface. (default is true)

-Negate regular expression (Option is available on both tabs.)
    -If this parameter is enabled, a violation will be flagged if the name of an
     interface matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for interface names which do not match the regular expression
     specified.
-Ignore case (Option is available on both tabs.)
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of interfaces 
     must match the regular expression specified, including in case.
BENEFITS 

This helps enforce a naming convention.
EXAMPLE 

package examples.rules.naming;

public interface NITF {  
 
}

interface nitf2 extends NITF {  // VIOLATION

}
REPAIR 

Change the identifier to follow the convention.

package examples.rules.naming;

public interface NITFFixed {  

}

interface NITF2Fixed extends NITFFixed {

}
