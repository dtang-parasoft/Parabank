Enforce number of space character(s) after the opening parenthesis "(" of a conditional statement [APSC_DV.003215.SAP]



DESCRIPTION 


This rule identifies opening parentheses in conditional statements that are not
followed by the required number of space characters. With the default parameters,
an error is reported if any parenthesis that appears in the header of a 'for'
loop, an 'if' statement, an 'else' statement, a 'while' loop, a 'switch' statement,
or a 'catch' statement is not followed by a single ' ' space character.

There is also a parameter where the user can specify whether or not to make an
exception for the common idiom 'for (;;)'.
SCOPE LEVEL 
LINE
SINCE 

v7.5
PARAMETERS 

-Number of spaces after the opening parenthesis in a conditional statement
 (default value is 1)
-Ignore rule for the special case "for(;;)" (default is true)
   -When this parameter is true, a "for" statement of the form "for(;;)..."
    (with no spaces around the semicolons) will be ignored regardless of the
    setting of the other parameter.
BENEFITS 

If all team members follow the same formatting style, it will be easier for each
team member to read and understand the code.
EXAMPLE 

package examples.rules.format;

public class SAP {
    public void method () {
        for (  int i = 0;i < 10;i++) {  // VIOLATION
            System.out.println ("i: " + i);
        }
    }
}
REPAIR 

Place a single space after the opening parenthesis of each conditional
statement.

package examples.rules.format;

public class SAPFixed {
    public void method () {
        for ( int i = 0;i < 10;i++) {  // FIXED
            System.out.println ("i: " + i);
        }
    }
}
