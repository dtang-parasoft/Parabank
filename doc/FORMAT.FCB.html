Enforce the position of '{' brace [FORMAT.FCB]



DESCRIPTION 


This rule is a parameterized rule that identifies code that lacks:
1) '{' braces at the end of the line or
2) '{' braces on their own line or
3) '{' braces on their own line with an indentation or
4) '{' braces at the end of the line when no parentheses are present, or parentheses 
         only span one line
   '{' braces on their own line with an indentation when parentheses span multiple
         lines
The requirement enforced depends on the rule parameters. An error is
reported for each occurrence.  A user can also choose to selectively ignore constructs.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-Braces preceded by comments are not considered a violation (default is true)
-Open and close braces on the same line are not considered a violation (default is true)
-Class declaration (default is 'same line')
-Anonymous class declaration (default is 'same line')
-Constructor declaration (default is 'same line')
-Enum declaration (default is 'same line')
-Enum constant body (default is 'same line')
-Annotation type declaration (default is 'same line') 
-Method declaration (default is 'same line')
-Static initializer (default is 'same line')
-Array initializer (default is 'same line')
-Block (default is 'same line')
-"if" statement (default is 'same line')
-"else" statement (default is 'same line')
-"for" statement (default is 'same line')
-"while" statement (default is 'same line')
-"do-while" statement (default is 'same line')
-"try" statement (default is 'same line')
-"synchronized" statement (default is 'same line')
-"catch" clause (default is 'same line')
-"finally" block (default is 'same line')
-"switch" statement (default is 'same line')
-Block in "switch-case" (default is 'same line')
BENEFITS 

If all team members follow the same formatting style, it will be easier for each
team member to read and understand the code.
EXAMPLE 

package examples.rules.format;

public class FCB 
{  // VIOLATION
}
REPAIR 

Move the '{' opening brace to the previous line.

package examples.rules.format;

public class FCBFixed {  // FIXED
}
