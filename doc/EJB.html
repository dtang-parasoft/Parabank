Enterprise JavaBeans [EJB] 

Rules related to Enterprise JavaBeans (EJB).

EJB classes must meet certain requirements, see:

Sun's "Enterprise JavaBeans Developer's Guide".
file:///C|/j2sdkee/doc/guides/ejb/html/TOC.html

The rules in this category enforce requirements
from that document.
RULES

Enterprise JavaBeans 3 [EJB.EJB3]
Do not use Servlet code in EJB classes [EJB.ABCS]
Do not use Entity Beans as fine-grained objects [EJB.ABFG]
Do not access, use, or create a class loader within a bean class [EJB.ACL]
Do not use EJB code in java swing or servlet classes [EJB.ADCB]
Avoid excessive inter-entity bean communication [EJB.AIEBC]
Do not use JDBC code inside of EJB classes [EJB.AJDBC]
Do not access or modify security configuration objects [EJB.AMSC]
Avoid one-to-one mapping between session beans and entity beans [EJB.AOTO]
Do not use sockets in EJBs [EJB.AUS]
Declare bean classes "public" [EJB.CDP]
Do not declare bean classes as "abstract" [EJB.CNDA]
Do not declare bean classes as "final" [EJB.CNDF]
Declare 'ejbCreate()' methods "public", but neither "static" nor "final" [EJB.CRTE]
Avoid granting access permission for EJB methods to the 'ANYONE' role [EJB.DPANY]
Do not call finder methods in the 'ejbLoad()' [EJB.EJBLOAD]
Declare finder methods "public" and neither "final" nor "static" [EJB.FNDM]
Implement one or more 'ejbCreate()' methods in bean classes [EJB.IECM]
Implement one or more 'ejbPostCreate()' methods in EntityBean classes [EJB.IEPM]
Do not use types from the "java.io" package within bean classes [EJB.JIO]
Avoid loading native libraries in a Bean class [EJB.LNL]
Implement a no-argument 'ejbCreate()' method for all Message-driven bean classes [EJB.MDBC]
Define a matching 'ejbPostCreate()' method for each 'ejbCreate' method in entity bean classes [EJB.MEC]
Do not define 'finalize()' method in bean classes [EJB.MNDF]
Throw 'java.rmi.RemoteException' in the methods of remote interface and remote home interface [EJB.MRE]
Declare a "public" constructor that takes no parameters in bean classes [EJB.NFDC]
Declare all "static" fields in EJB bean classes "final" [EJB.NFS]
Declare 'ejbPostCreate()' "public" and neither "static" nor "final" [EJB.PCRTE]
Do not throw 'java.rmi.RemoteException' in a bean's local interface and local home interface [EJB.RILH]
Do not declare entity beans as remote [EJB.RR]
The return type of EJB finder methods should be the primary key or a collection of primary keys [EJB.RT]
Make the return type "void" for SessionBeans or MessageDrivenBeans' 'ejbCreate()' methods [EJB.RTC]
Make the return type "void" for the 'ejbPostCreate()' method [EJB.RTP]
Reuse EJB homes [EJB.RUH]
Do not set or create a new SecurityManager in EJBs [EJB.SMSN]
Ensure EJB beans include necessary methods and EJB bean and interface classes follow the name format [EJB.STD]
Throw 'javax.ejb.CreateException' in create methods of remote home or local home interfaces [EJB.TCE]
Throw 'javax.ejb.FinderException' in finder methods of remote home or local home interfaces [EJB.TFE]
Avoid passing the "this" reference as an argument [EJB.THISARG]
Do not return "this" [EJB.THISRET]
Avoid starting, stopping, or managing threads in any way in bean classes [EJB.THREAD]
Cache reusable JNDI resources to minimize the use of expensive operations [EJB.UCIC]
Use a Session Facade to manage access to entity beans [EJB.USF]
Use Value Objects to reduce the granularity of calls to the server [EJB.UVO]
