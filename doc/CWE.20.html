CWE-20: Improper Input Validation [CWE.20]

RULES

Do not extend from the Struts classes 'ActionForm' and 'DynaActionForm' [CWE.20.AEAF]
Inspect usage of standard API calls that bypass security [CWE.20.APIBS]
Avoid accessing arrays out of bounds [CWE.20.ARRAY]
Avoid unvalidated input in array indexes [CWE.20.ARRAYSEC]
Do not use an integer outside the range of [0, 31] as the amount of a shift [CWE.20.BSA]
Prevent external processes from blocking on output or error streams [CWE.20.BUSSB]
Avoid using compound assignment operators in cases which may cause overflow [CWE.20.CACO]
Always check parameters before use in array access [CWE.20.CAI]
Do not cast primitive data types to lower precision [CWE.20.CLP]
Always call 'super.validate()' from validation methods in 'ActionForm' classes [CWE.20.CSVFV]
Avoid duplicated forms in the 'validation.xml' [CWE.20.DFV]
Ensure validators are enabled in the 'struts-config.xml' [CWE.20.EV]
Do not use resources that have been freed [CWE.20.FREE]
Avoid calculations which result in overflow or NaN [CWE.20.ICO]
Avoid integer overflows [CWE.20.INTOVERF]
Use unsigned right shift instead of division when overflow is possible [CWE.20.IOF]
Use wrapper methods to secure native methods [CWE.20.NATIW]
Do not use user-defined "native" methods [CWE.20.NATV]
Ensure Plugins are added in the 'struts-config.xml' [CWE.20.PLUGIN]
Do not access or set System properties [CWE.20.SYSP]
Validate potentially tainted data before it is used to determine the size of memory allocation [CWE.20.TDALLOC]
Exclude unsanitized user input from format strings [CWE.20.TDINPUT]
Protect against Library injection [CWE.20.TDLIB]
Protect against log forging [CWE.20.TDLOG]
Protect against HTTP response splitting [CWE.20.TDRESP]
Protect against Reflection injection [CWE.20.TDRFL]
Use a Context Object to manage HTTP request parameters [CWE.20.UCO]
