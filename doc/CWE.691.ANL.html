Avoid using negative logic in if-else statement [CWE.691.ANL]



DESCRIPTION 


This rule identifies negative logic in 'if-else' control flow.
'if-else' control flow includes the patterns below:

-if (cond)
    expr
 else 
    expr
  
-cond ? expr1 : expr2 

An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v6.0
SECURITY RELEVANCE 

"The code does not sufficiently manage its control flow during execution,
creating conditions in which the control flow can be modified in unexpected
ways." [CWE-691]
PARAMETERS 

-Ignore special cases (default is true)
    There are some commonly used negative logics that should not
    be reported as errors; these include:
    -if (var!= null) 
    -if (var!= 0)
    -if (var!= -1)
    -if (!methodInvocation) where methodInvocation is not "equals" method
    
    If 'Ignore special cases' is set to true, all the above cases would
    be ignored.
BENEFITS 

Using negative logic could sometimes lead to complicated control flow.
The logic could easily be modified to avoid negation and provide
a more intuitive flow.
EXAMPLE 

package examples.rules.codsta.read;

public class ANL {
    public int specialAdd(int i, int j) {
        if (i!= j) //VIOLATION
            return i+ j;
        else
            return i;
    }
}
REPAIR 

package examples.rules.codsta.read;

public class ANLFixed {
    public int specialAdd(int i, int j) {
        if (i== j) //FIXED
            return i;
        else 
            return i+ j;
    }
}
