Do not use "break" and/or "continue" statements [CODSTA.BP.CONTINUE]



DESCRIPTION 


This rule identifies code that uses "break" and/or "continue" 
patterns. An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v6.0
PARAMETERS 

-Flag "break" statements. (default is false)
-Flag "continue" statements. (default is true)
BENEFITS 

Using "continue" and "break" makes it difficult to later break the
construct into smaller constructs or methods. It also forces the
developer to consider more than one end point for a construct.
By following this rule, you make the code more readable and modular.
EXAMPLE 

package examples.rules.codsta.bp;

public class CONTINUE {
    public int getNum (int max){
        int i= 0;
        while (true) {
            i+= max;
            max--;
            if (max== 0) {
                break; //VIOLATION
            } 			
        }
        return i;
    }
}
REPAIR 

Reorganize the logic of the code segment in order to 
avoid those patterns.

package examples.rules.codsta.bp;

public class CONTINUEFixed {
    public int getNum (int max){
        int i= 0;
        while (max> 0) {
            i+= max;
            max--;   //FIXED
        }
        return i;
    }
}
