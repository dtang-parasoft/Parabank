Enforce returning a defensive copy in 'clone()' methods [SECURITY.EAB.CPCL]



DESCRIPTION 


This rule identifies classes that implement 'java.lang.Cloneable', but do not
initialize the mutable fields of a clone object to defensive copies in the
'clone()' method. An error is reported for each occurrence.
SINCE 

v8.4
SECURITY RELEVANCE 

category: Erratic Application Behavior

If a mutable field in a Cloneable object is not initialized by a defensive copy,
when users change the state of that mutable field on the Cloneable object, the 
field in the original object will be changed as well.
EXAMPLE 

package examples.rules.security.eab;

import java.util.Date;

public class CPCL implements Cloneable {
    
    private Date date;
    private int[] numbers;
    private int index;
    
    public void setIndex(int idex) {
        index = idex;
    }
    
    public int getIndex() {
        return index;
    }
    
    public void setNumbers(int[] num) {
        numbers = num;
    }
    
    public int[] getNumbers() {
        return numbers;
    }
    
    public void setDate(Date d) {
        date = d;
    }
    
    public Date getDate() {
        return date;
    }
    
    public Object clone() throws CloneNotSupportedException {
        CPCL clone = (CPCL)super.clone();  // VIOLATION : 'numbers' is not a defensive copy
        theClone.date = this.date;  // VIOLATION : 'date' is not a defensive copy
        theClone.index = index;
        
        return clone;
    }
    
    public static void main(String[] args) throws CloneNotSupportedException {
        CPCL cpcl = new CPCL();
        cpcl.setDate(new Date(System.currentTimeMillis());
        CPCL cloneCPCL = (CPCL)cpcl.clone();
        cloneCPCL.data.setTime(0); // Changing 'date' of the Cloneable object 'cloneCPCL' 
                                   // will also change the 'date' of the original object 
                                   // 'cpcl'
    }
}
REPAIR 

Initialize the fields 'date' and 'numbers' with defensive copies.

package examples.rules.security.eab;

import java.util.Date;

public class CPCLFixed implements Cloneable {
    
    private Date date;
    private int[] numbers;
    private int index;
    
    public void setIndex(int idex) {
        index = idex;
    }
    
    public int getIndex() {
        return index;
    }
    
    public void setNumbers(int[] num) {
        numbers = num;
    }
    
    public int[] getNumbers() {
        return numbers;
    }
    
    public void setDate(Date d) {
        date = d;
    }
    
    public Date getDate() {
        return date;
    }
    
    public Object clone() throws CloneNotSupportedException {
        CPCLFixed clone = (CPCLFixed)super.clone();
        theClone.date = new Date(date.getTime());  // FIXED
        theClone.index = index;
        System.arraycopy(numbers, 0, theClone.numbers, 0, numbers.length); // FIXED        
        return clone;
    }
    
    public static void main(String[] args) throws CloneNotSupportedException {
        CPCLFixed cpcl = new CPCLFixed();
        cpcl.setDate(new Date(System.currentTimeMillis());
        CPCLFixed cloneCPCL = (CPCLFixed)cpcl.clone();
        cloneCPCL.data.setTime(0);
    }
}
