Use "File.pathSeparator" or "File.pathSeparatorChar" instead of the corresponding literals [PORT.PSC]



DESCRIPTION 


Instead of using the String literals ";" or ":" or the character literals
';' or ':', "File.pathSeparator" (for Strings) or "File.pathSeparatorChar" (for 
characters) should be used if the literal is intended as a path-separator.  This 
rule will flag as a violation any use of the literals ":",";",':', and ';' where 
these literals could be replaced with either "File.pathSeparator" or 
"File.pathSeparatorChar".
SCOPE LEVEL 
LINE
SINCE 

v8.0
NOTES 

This rule may flag some false positives because the rule cannot tell where a
character or String literal is intended as a path-separator and where a literal
is being used for a different purpose.
BENEFITS 

Using the constants "File.pathSeparator" (for Strings) or 
"File.pathSeparatorChar" instead of the equivalent literals helps increase
the portability of the code.  This is because the path-separator character is 
system-dependent.  On Windows systems, it is ";".  On UNIX systems, it is ":".
Using "File.pathSeparator" and "File.pathSeparatorChar" ensures that your code
will work on any platform.
EXAMPLE 

This example assumes that the rule is being run on a Windows system.  

package examples.rules.port; 

class PSC{
    private static final String ex= "This"+";"+   //VIOLATION
                                           ';';   //VIOLATION
}
REPAIR 

package examples.rules.port; 

class PSCFixed{
    private static final String ex= "This"+File.pathSeparator+       //FIXED
                                           File.pathSeparatorChar;   //FIXED
}
