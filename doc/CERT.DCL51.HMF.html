Do not give method local variables and parameters the same name as class fields [CERT.DCL51.HMF]



DESCRIPTION 


This rule identifies method local variables and parameters 
with the same name as class fields. An error is reported
for each occurrence.  By default, both local variables and
parameters are checked.  The rule can be parameterized to 
only check one or the other.
SCOPE LEVEL 
LINE
SINCE 

v4.5
BENEFITS 

When class fields are hidden with method local variables and 
parameters of the same name, your code may be confusing and
difficult to read.  It is good practice to use unique names
for variables.

PARAMETER

-Check parameters (default is true)
-Check local variables (default is true)
EXAMPLE 

package examples.rules.oop;

public class HMF {
    public void method (int j) {  // VIOLATION
        final int i = 5;  // VIOLATION
        System.out.println (i + j);
    }

    private int i = 0;
    private int j = 2;
}
REPAIR 

Give the local variable and parameter a unique name.

package examples.rules.oop;

public class HMFFixed {
    public void method (int pj) {
        final int x = 5;  // FIXED
        System.out.println (x + pj);
    }

    private int i = 0;
    private int j = 2;
}
