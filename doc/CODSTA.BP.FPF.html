Declare all formal parameters as "final" [CODSTA.BP.FPF]



DESCRIPTION 


This rule identifies formal parameters that are not declared
as "final". With the default setting of the parameters,  
parameters of "catch" blocks are checked in addition to method 
parameters.  An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v6.0
NOTES 

This coding style is a matter of personal preference. Ignoring
this rule will not lead to software errors.

This rule may report false positives.
SECURITY RELEVANCE 

category: Code Quality
PARAMETERS 

-Ignore parameters which are declared in a "catch" block. 
 (Default value is "false".)
BENEFITS 

Trying to modify parameters could be very confusing. 
Declaring all parameters as final could prevent this 
confusion.
EXAMPLE 

package examples.rules.codsta.bp;

class FPF {
    public int sum(int a, int b) { //VIOLATION
        return (a + b); 
    }
}
REPAIR 

package examples.rules.codsta.bp;

class FPFFixed {
    public int sum(final int a, final int b) { //FIXED
        return (a + b); 
    }
}
