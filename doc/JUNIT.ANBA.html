Avoid negating boolean parameters to assertTrue() or assertFalse() [JUNIT.ANBA]



DESCRIPTION 


This rule detects when the 'assertTrue()' or 'assertFalse()' method is called
with a negated boolean as an argument.

'assertTrue()' called with a negated boolean can be replaced by 'assertFalse()'.
'assertFalse()' called with a negated boolean can be replaced by 'assertTrue()'.
SINCE 

v9.0
NOTES 

This rule does not trigger if method parameters contain comparison
to "null". See the JUNIT.ANAT rule.
BENEFITS 

This rule helps you improve test code readability.
EXAMPLE 

package examples.rules.junit;

import junit.framework.*;

public class ANBA extends TestCase {  
    public void testMethod() {
        boolean b = someMethod();
        assertTrue(!b); //VIOLATION
    }

    private boolean someMethod() {
       ...
    }
}
REPAIR 

package examples.rules.junit;

import junit.framework.*;

public class ANBAFixed extends TestCase {
    public void testMethod() {
        boolean b = someMethod();
        assertFalse(b); //FIXED
    }

    private boolean someMethod() {
       ...
    }
}
