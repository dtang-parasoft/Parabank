6.2.4.1 Injection attacks [PCIDSS40.6241]

RULES

Do not extend from the Struts classes 'ActionForm' and 'DynaActionForm' [PCIDSS40.6241.AEAF]
Avoid unvalidated input in array indexes [PCIDSS40.6241.ARRAY]
Avoid parsing untrusted data with XMLDecoder [PCIDSS40.6241.AUXD]
Avoid untrusted input when logging messages with Seam Logging API [PCIDSS40.6241.DCEMSL]
Validate potentially tainted data before it is used to determine the size of memory allocation [PCIDSS40.6241.TDALLOC]
Protect against Command injection [PCIDSS40.6241.TDCMD]
Protect against Jakarta Digester injection [PCIDSS40.6241.TDDIG]
Protect against Environment injection [PCIDSS40.6241.TDENV]
Protect against File names injection [PCIDSS40.6241.TDFNAMES]
Exclude unsanitized user input from format strings [PCIDSS40.6241.TDINPUT]
Protect against JXPath injection [PCIDSS40.6241.TDJXPATH]
Protect against LDAP injection [PCIDSS40.6241.TDLDAP]
Protect against log forging [PCIDSS40.6241.TDLOG]
Protect against network resource injection [PCIDSS40.6241.TDNET]
Protect against Reflection injection [PCIDSS40.6241.TDRFL]
Protect against SQL injection [PCIDSS40.6241.TDSQL]
Protect against XML data injection [PCIDSS40.6241.TDXML]
Protect against XPath injection [PCIDSS40.6241.TDXPATH]
Use 'prepareCall' or 'prepareStatement' instead of 'createStatement' [PCIDSS40.6241.UPS]
Avoid XPath injection when evaluating XPath queries [PCIDSS40.6241.XPIJ]
