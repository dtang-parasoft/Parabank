6.2.4.2 Attacks on data and data structures [PCIDSS40.6242]

RULES

Avoid accessing arrays out of bounds [PCIDSS40.6242.ARRAY]
Do not expose data wrapped by a buffer to untrusted code [PCIDSS40.6242.BUFEXP]
Do not use an instance lock to protect shared static data [PCIDSS40.6242.INSTLOCK]
Do not create multiple buffered wrappers on a single byte or character stream [PCIDSS40.6242.MULBUF]
Do not change the input streams of 'java.lang.System' in a web component [PCIDSS40.6242.SIS]
Validate potentially tainted data before it is used to determine the size of memory allocation [PCIDSS40.6242.TDALLOC]
Protect against File names injection [PCIDSS40.6242.TDFNAMES]
Exclude unsanitized user input from format strings [PCIDSS40.6242.TDINPUT]
Protect against network resource injection [PCIDSS40.6242.TDNET]
Avoid misconfiguring timestamps in WebSphere 'ibm-webservicesclient-ext.xmi' files [PCIDSS40.6242.WCMT]
Avoid unsigned timestamps in WebSphere 'ibm-webservicesclient-ext.xmi' files [PCIDSS40.6242.WCUTS]
