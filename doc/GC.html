Garbage Collection [GC] 

Rules related to Garbage Collection.
RULES

Do not use unnecessary temporaries when converting primitive types to String [GC.AUTP]
Avoid using 'Date[]', use 'long[]' instead [GC.DUD]
Call 'super.finalize()' from 'finalize()' [GC.FCF]
Do not use 'finalize()' methods to unregister listeners [GC.FM]
Do not explicitly call 'System.gc()' or 'Runtime.gc()' [GC.GC]
Reuse calls to 'getClipBounds()' [GC.GCB]
Call 'super.finalize()' in the "finally" block of 'finalize()' methods [GC.IFF]
Be cautious when calling methods which may cause memory leaks [GC.MML]
Do not call 'finalize()' explicitly [GC.NCF]
Prevent potential memory leaks in ObjectOutputStreams by calling 'reset()' [GC.OSTM]
Reuse immutable constant objects to conserve memory [GC.RCO]
Avoid "static" collections or maps; they can grow without bounds [GC.STV]
