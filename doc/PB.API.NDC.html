Do not define direct or indirect subclasses of 'Error' and 'Throwable' [PB.API.NDC]



DESCRIPTION 


This rule identifies direct and indirect subclassing of 'Error' and 'Throwable'.
An error is reported for each occurrence.
SCOPE LEVEL 

CLASS
SINCE 

v4.5
SECURITY RELEVANCE 

category: Erratic Application Behavior
category: Error Handling

Potential erroneous exception handling. See BENEFITS for more information.
BENEFITS 

The class 'java.lang.Error' is meant only for covering abnormal Java Virtual Machine
conditions.  If you define a direct or indirect subclass of class
'Error', it is implied that the error is also an abnormal Java Virtual
Machine condition, which is not the case.  Exception handling of 'Error' is not
checked by the Java compiler, so erroneous exception handling may not be
noticed.
        
The class 'java.lang.Throwable' is the super class of 'java.lang.Exception' and
'java.lang.Error'. User-defined exceptions should always be defined as
subclasses of 'java.lang.Exception'.
EXAMPLE 
        
package examples.rules.pb.api;
        
public class NDC extends Error {  // VIOLATION
    public NDC (String s) {
        super(s);
    }
}
REPAIR 

Change the class to extend Exception.
        
package examples.rules.pb.api;
        
public class NDCFixed extends Exception {  // FIXED
    public NDCFixed (String s) {
        super(s);
    }
}
