Declare "public/protected" fields as inaccessible as possible [GLOBAL.DPPF]



DESCRIPTION 


This rule identifies "public" and "protected" fields that are not as
inaccessible as possible. An error is reported for each occurrence.

Global analysis is required in this rule.
SCOPE LEVEL 
LINE
SINCE 

v4.5
SECURITY RELEVANCE 

category: Weak Security Controls
category: Code Quality

Avoid granting more access than is required.
PARAMETERS 

-Ignore "final" fields
    -Default is "true"
    -"final" modifier could imply the declared field is a constant for later
     use. Such a constant should be ignored.
-Ignore fields declared in "abstract" classes
    -Default is "false"
-Ignore "protected" fields which can be made package-private
    -Default is "false"
    -If this parameter is enabled, the rule will only flag a violation on
     "protected" fields if they can be made "private".
    -Default is "true"
-Ignore the following classes: (specify with Glob expressions)
    -Default "org.eclipse.osgi.util.NLS" "*"
    superclass name should be fully qualified name, and class name should not be
BENEFITS 

Following this rule makes the code more object-oriented and makes its
dependencies easier to understand.
REPAIR 

Change the field's accessibility or document the reason for the
excessive accessibility.
