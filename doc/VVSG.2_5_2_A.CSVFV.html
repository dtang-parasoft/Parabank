Always call 'super.validate()' from validation methods in 'ActionForm' classes [VVSG.2_5_2_A.CSVFV]



DESCRIPTION 



This rule checks to make sure that overridden validation methods in classes extending
from "org.apache.struts.action.ActionForm" call "super.validate()".  This enforces usage 
of the validation mechanism.  A violation will be flagged for each "validate" method in a 
class extending from "ActionForm" which does not call "super.validate()".
SINCE 

v8.0
SECURITY RELEVANCE 

category: Input-Based Attacks

See BENEFITS section.
BENEFITS 

Enforcing this rule makes sure that validation is performed within the validation methods.
EXAMPLE 

package examples.rules.security.iba;

import javax.servlet.http.HttpServletRequest;
import org.apache.struts.action.ActionErrors;
import org.apache.struts.action.ActionMapping;
import org.apache.struts.validator.ValidatorForm;

public class CSVFV extends ValidatorForm {
    public CSVFV() {
        super();
    }

    public ActionErrors validate(ActionMapping map, HttpServletRequest req) {
        return new ActionErrors();
    }
}
REPAIR 

Call "super.validate()" from the validate method.

package examples.rules.security.iba;

import javax.servlet.http.HttpServletRequest;
import org.apache.struts.action.ActionErrors;
import org.apache.struts.action.ActionMapping;
import org.apache.struts.validator.ValidatorForm;

public class CSVFVFixed extends ValidatorForm {
    public CSVFVFixed() {
        super();
    }

    public ActionErrors validate(ActionMapping map, HttpServletRequest req) {
        return super.validate(map, req);
    }
}
