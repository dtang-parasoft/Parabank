Do not make assignments to method parameters [CODSTA.EPC.AFP]



DESCRIPTION 


This rule identifies assignment to method parameters.
An error is reported for each occurrence.
SINCE 

v4.5
BENEFITS 

Assignment operations on method parameters can lead to mistakes 
when the value of the parameter is used more than once in a block.
It is also discouraged because changing the value of a method parameter
is a side effect that the method caller may not be aware of.
EXAMPLE 

This example illustrates a simple case where changing method parameters alters
the value of the parameter in the caller's scope.

package examples.rules.codsta.epc;

public class AFP {
    public void method (int[] a) {
        a[0] = 0;  // VIOLATION
    }
}
REPAIR 

Create a local copy of the variable.

package examples.rules.codsta.epc;

public class AFPFixed {
    public void method (int[] a) {
        int[] copy = new int [a.length];
        System.arraycopy (a, 0, copy, 0, a.length);
        copy[0] = 0;  // FIXED
    }
}
