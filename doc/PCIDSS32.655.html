6.5.5 Improper error handling [PCIDSS32.655]

RULES

Do not call the 'printStackTrace()' method of "Throwable" objects [PCIDSS32.655.ACPST]
Avoid "try", "catch" and "finally" blocks with empty bodies [PCIDSS32.655.AECB]
Avoid using 'return's inside 'finally blocks if thare are other 'return's inside the try-catch block [PCIDSS32.655.ARCF]
Catch all exceptions which may be thrown within Servlet methods [PCIDSS32.655.CETS]
Consistently check the returned value of non-void methods [PCIDSS32.655.CHECKRET]
Check the return value of methods which read or skip input [PCIDSS32.655.CRRV]
Ensure all exceptions are either logged with a standard logger or rethrown [PCIDSS32.655.LGE]
Do not pass exception messages into output in order to prevent the application from leaking sensitive information [PCIDSS32.655.PEO]
Prevent exposure of sensitive data [PCIDSS32.655.SENS]
Always specify error pages in web.xml [PCIDSS32.655.SEP]
Avoid calling print methods of 'System.err' or 'System.out' [PCIDSS32.655.SIO]
