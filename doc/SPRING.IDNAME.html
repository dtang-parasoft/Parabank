Avoid using ''name'' and ''id'' simultaneously [SPRING.IDNAME]



DESCRIPTION 


This rule identifies simultaneous usage of the 'id' and 'name' attributes 
in the <bean/> element within Spring xml configuration files.
SINCE 

v9.2
NOTES 
Inserting two attributes makes one of them behave like an alias.
This behavior could confuse other team members.
BENEFITS 

Following this rule makes code easier to understand and maintain.
EXAMPLE 

<bean id="listOrdersController1" name="listOrdersController2" class="org.springframework.samples.jpetstore.web.spring.ListOrdersController" />
REPAIR 

Remove name:

<bean id="listOrdersController" class="org.springframework.samples.jpetstore.web.spring.ListOrdersController" />
REFERENCES 
1. Spring Framework Documentation 
"3.3.1 Naming beans"
http://static.springsource.org/spring/docs/3.0.x/spring-framework-reference/html/beans.html#beans-beanname
