Use a naming convention for local variables [NAMING.NLV]



DESCRIPTION 


This rule identifies local variables that do not follow the user-defined
naming convention (as specified in a regular expression). By default, the rule 
enforces that the names of local variables begin with a lower case letter. An 
error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-regular expression
    -Local variable names should match this expression.
    -The default is "^[a-z]".  This enforces that the names of local variables
     begin with a lower case letter.
-Negate regular expression
    -If this parameter is enabled, a violation will be flagged if the name of a
     local variable matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for local variable names which do not match the regular expression
     specified.
-Ignore case
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of local 
     variables must match the regular expression specified, including in case. 
-Ignore "final" local variables
    -By default, this parameter is off.
    -If this parameter is enabled, violations will not be flagged for "final"
     local variables.  You may have a different naming convention for "final" 
     local variables.  If this is the case, you should enable this parameter and
     use the rule NAMING.NFL to enforce your convention for "final" local 
     variables.
