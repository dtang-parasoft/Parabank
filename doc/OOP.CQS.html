Ensure methods are either a command(change state) or a query(get state) [OOP.CQS]



DESCRIPTION 


This rule checks for Command Query Separation. A method can either change
the state of an object (command) or get the state of an object (query), but
not both.
SCOPE LEVEL 
METHOD
SINCE 

v8.3
PARAMETERS 

-Check the following accessibilities:
    -Check "public" methods. (default true)
    -Check "protected" methods. (default true)
    -Check "package-private" methods. (default true)
    -Check "private" methods. (default false)
-Ignore the following methods:
    -"nextToken" (default)
-Allow lazy initialization. (default true)
    -Allows basic cases of lazy initialization, where a field is explicitly
    checked for a null value and then initialized;
BENEFITS 

Use of the Command Query Separation may have a simplifying effect on the program
by making it easier to understand and allowing for clearer specifications.  It 
also ensures that queries do not change the program state.
EXAMPLE 

package examples.rules.oop;

public class CQS {
    int x;
    int squareInt() {
        x = x * x;
        return x;
    }
}
REPAIR 

package examples.rules.oop;

public class CQSFixed {
    int x;
    void squareInt() {
        x = x * x;
    }
    int getInt() {
        return x;
    }
}
