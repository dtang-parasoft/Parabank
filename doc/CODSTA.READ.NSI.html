Avoid non-static initializers [CODSTA.READ.NSI]



DESCRIPTION 


This rule identifies nonstatic initializers. An error is 
reported for each occurrence.
SINCE 

V6.0
BENEFITS 

A nonstatic initializer would be executed before every constructor
invocation. It makes more sense to integrate the code from any 
nonstatic initializer into a constructor, which is a more
organized and readable structure.
EXAMPLE 

package examples.rules.codsta.read;

public class NSI {
    { //VIOLATION
        System.out.println("Hi"); 
    }
    public NSI() {
        System.out.println("Executing constructor");
    }
}
REPAIR 

package examples.rules.codsta.read;

public class NSIFixed {
    //FIXED
    public NSI() {
        System.out.println("Hi");
        System.out.println("Executing constructor");
    }
}
