Avoid unnecessary "boolean" comparisons [UC.BCMP]



DESCRIPTION 


This rule identifies unnecessary boolean comparisons. An error is 
reported if the ==/!= operator is used in a comparison while at least
one side is a boolean literal.
SINCE 

v6.0
BENEFITS 

Comparing something with a boolean literal is essentially
redundant. Code is simplified when you avoid these unnecessary comparisons.
EXAMPLE 

package examples.rules.uc;

public class BCMP {
    public void myMethod(boolean b) {
        if (b== true)  //VIOLATION
            System.out.println("got a true");
        else
            System.out.println("got a false");
    }
}
REPAIR 

package examples.rules.uc;

public class BCMPFixed {
    public void myMethod(boolean b) {
        if (b) //FIXED
	    System.out.println("got a true");
	else
	    System.out.println("got a false");
    }
}
