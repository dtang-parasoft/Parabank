Design by Contract [DBC] 

Rules that are related to Design By Contract.

"Design by Contract" contracts are expressed with
Java code embedded in Javadoc comments in the source file.

The reserved Javadoc tags for DBC are:

  - @invariant: specifies class invariant condition.
  - @pre: specifies method pre-condition.
  - @post: specifies method post-condition.
  - @concurrency: specifies method concurrency.

Jtest also support the following tags:

  - @assert: to put assertions in the method bodies.
  - @throws/@exception: to document the exceptions thrown
    by a method.
RULES

Do not include a postcondition saying that "$result!=null" in methods which can return null [DBC.CPT]
Provide an '@invariant' contract for all getter methods [DBC.IGM]
Do not invoke a method on a reference that is not guaranteed to be non-null [DBC.IMNR]
Include a '@pre != null' tag for each parameter that is dereferenced before being checked for null [DBC.IPAN]
Provide an '@invariant' contract for all package-private classes [DBC.PKGC]
Provide a '@post' contract for all package-private methods [DBC.PKGMPOST]
Provide a '@pre' contract for all package-private methods [DBC.PKGMPRE]
Provide an '@invariant' contract for all "private" classes [DBC.PRIC]
Provide a '@post' contract for all "private" methods [DBC.PRIMPOST]
Provide a '@pre' contract for all "private" methods [DBC.PRIMPRE]
Provide an '@invariant' contract for all "protected" classes [DBC.PROC]
Provide a '@post' contract for all "protected" methods [DBC.PROMPOST]
Provide a '@pre' contract for all "protected" methods [DBC.PROMPRE]
Provide an '@invariant' contract for all "public" classes [DBC.PUBC]
Provide a '@post' contract for all "public" methods [DBC.PUBMPOST]
Provide a '@pre' contract for all "public" methods [DBC.PUBMPRE]
Avoid rechecking @pre and @post conditions [DBC.RCC]
Use correct syntax in the DbC contracts [DBC.SYNTAX]
