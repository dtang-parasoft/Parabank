Do not declare "public" or package-private constructors in "abstract" classes [OOP.NPAC]



DESCRIPTION 


This rule identifies "abstract" classes with "public" or package-private (the 
default accessibility) constructors.  By default, the rule will check only for
"public" constructors, but the values of the parameters can be changed to make 
it also check for package-private constructors.  An error is reported for each 
occurrence.
SCOPE LEVEL 
LINE
SINCE 

v7.5
NOTES 

Violations will only be flagged for inner classes if the inner class is static 
and has either "public" or "protected" access.
PARAMETERS 

-Check for "public" constructors
    -Default is "true".
-Check for package-private constructors
    -Default is "false".
    -Turn this parameter on if you want the rule to flag package-private 
     constructors (constructors with the default accessibility) in addition to 
     "public" constructors.
BENEFITS 

"abstract" classes cannot be instantiated. The constructors should only be 
called by subclasses, so it is not necessary to set the accessibility to 
"public" or package-private.
EXAMPLE 

package examples.rules.oop;

public abstract class NPAC {

    private int _id;

    public NPAC() {  // VIOLATION
        this(0);
    }

    protected NPAC(int id) {
        _id = id;
    }
}
REPAIR 

Change the "public" constructor to "protected".

package examples.rules.oop;

public abstract class NPACFixed {

    private int _id;

    protected NPACFixed() {  // FIXED
        this(0);
    }

    protected NPACFixed(int id) {
        _id = id;
    }
}
