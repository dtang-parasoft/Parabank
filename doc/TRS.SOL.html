Do not perform synchronization using the "synchronized" keyword on implementations of "java.util.concurrent.locks.Lock" [TRS.SOL]



DESCRIPTION 


This rule identifies code that performs synchronization on an implementation of 
"java.util.concurrent.locks.Lock".  Instead of synchronization, the programmer
should use the "lock()" and "unlock()" methods.
SCOPE LEVEL 
LINE
SINCE 

v8.0
EXAMPLE 

package examples.rules.trs;

import java.util.concurrent.locks.*;
import java.util.concurrent.*;

public class SOL{
    int i;
    SOL(){
        i= 0;
    }

    public void method(Lock l){
        synchronized(l){        //VIOLATION
            i++;
        }
    }
}
REPAIR 

Use the "lock()" and "unlock()" methods instead of synchronization.

package examples.rules.trs;

import java.util.concurrent.locks.*;
import java.util.concurrent.*;

public class SOLFixed{
    int i;
    SOLFixed(){
        i= 0;
    }

    public void method(Lock l){
        l.lock();                //FIXED
        i++;
        l.unlock();                //FIXED
    }
}
