Avoid inner classes in interfaces [OOP.ICIF]



DESCRIPTION 


This rule identifies inner classes within interfaces.  An error is 
reported for each occurrence.
SINCE 

v8.2
BENEFITS 

Declaring an inner class in an interface is confusing and unnecessary.  
Java bytecode has no concept of inner classes.  Inner classes in 
interfaces will be translated by the compiler into ordinary classes 
anyway, so there is no point in declaring them as inner classes.  These 
classes should be declared as top-level classes instead.
EXAMPLE 

package examples.rules.oop;

public interface ICIF {
    class InnerClass {  //VIOLATION
        //some code
    }
}
REPAIR 

Change the inner class to a top-level class.

package examples.rules.oop;

public interface ICIF_Fixed {
}

class InnerClass {  //FIXED
    //some code
}
