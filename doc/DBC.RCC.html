Avoid rechecking @pre and @post conditions [DBC.RCC]



DESCRIPTION 


This rule flags a violation when a parameter is checked for null and the method
has a @pre condition that states it will not be null. This rule also flags
a violation when the return value of an invoked method is checked for null and
the invoked method has a @post condition that states it will not be null. See
BENEFITS and DRAWBACKS for more information.
SINCE 

v8.4
NOTES 

Methods with no javadoc comments are ignored, as they are assumed to be not
designed according to the DBC methodology.
PARAMETERS 

-Check @pre conditions (default true)
-Check @post conditions (default true)
BENEFITS 

Removal of unnecessary null checks can simplify code and reduce complexity.
DRAWBACKS 

If the condition in the contract was erroneous, then removal of the null check
may cause a null pointer exception to be thrown.
EXAMPLE 

package examples.rules.dbc;

public class RCC {
    /**
     * @pre o != null
     **/
    void print(Object o) {
        if (o != null) { //VIOLATION
            System.out.println(o.toString());
        }
    }

    /**
     *
     */
    void printField() {
        String value = getValue();
        if (value != null) { //VIOLATION
            System.out.println(value);
        }
    }

    String _value = "Hello";
    /**
     * @post $result != null
     */
    String getValue() {
        return _value;
    }
}
REPAIR 

If the DBC conditions are correct, then remove the unnecessary code.

package examples.rules.dbc;

public class RCCFixed {
    /**
     * @pre o != null
     **/
    void print(Object o) {
        System.out.println(o.toString()); //FIXED
    }

    /**
     *
     */
    void printField() {
        String value = getValue();
        System.out.println(value); //FIXED
    }

    String _value = "Hello";
    /**
     * @post $result != null
     */
    String getValue() {
        return _value;
    }
}
