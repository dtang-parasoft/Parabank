Declare fields with uppercase character names as "final" [CODSTA.BP.CFNF]



DESCRIPTION 


This rule identifies any possible constant field that is not 
declared as "final". If a field's name does not contain any
lowercase characters, it is considered to be a constant. An error
is reported for each occurrence.
SINCE 

v6.0
BENEFITS 

If a field is meant to be a constant, it should be declared
as "final".
EXAMPLE 

package rules.examples.codsta.bp;

public class CFNF {
    public static int MAX= 100; //VIOLATION
    public static int MIN= 0; //VIOLATION
}
REPAIR 

package rules.examples.codsta.bp;

public class CFNFFixed {
    public static final int MAX= 100; //FIXED
    public static final int MIN= 0; //FIXED
}
