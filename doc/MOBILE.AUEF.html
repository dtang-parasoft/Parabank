Avoid using enhanced "for" loop to walk through a "java.lang.Iterable" object [MOBILE.AUEF]



DESCRIPTION 


This rule will flag a violation if using enhanced "for" loop to walk through "java.lang.Iterable" object.
SINCE 

v9.0
BENEFITS 

JDK 1.5 Compiler will create additional object if walk through a "java.lang.Iterable" 
object by enhanced "for" loop, which slower down the performance.
EXAMPLE 


package examples.rules.mobile;

import java.util.*;

public class AUEF {

    public int getTotal(List array) {
        int sum = 0;
        for (Integer i : array) {  // VIOLATION
            sum += i.intValue();
        }
        
        return sum;
    }
}
REPAIR 


package examples.rules.mobile;

import java.util.*;

public class AUEFFixed {

    public int getTotal(List array) {
        int sum = 0;
        Iterator iter = array.iterator();
        while (iter.hasNext() {  // FIXED
            sum += ((Integer)iter.next()).intValue();
        }
        
        return sum;
    }
}
