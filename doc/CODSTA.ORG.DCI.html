Define constants in an "interface" [CODSTA.ORG.DCI]



DESCRIPTION 


This rule identifies constants that are not defined in "interface"s. 
An error is reported when a non-private named constant is not defined in an
"interface".
SINCE 

v4.5
NOTES 

Jtest provides another rule, CODSTA.ISACF, which conflicts with this rule.  
Some experts recommend constants in an "interface", and others
recommend avoiding them. In response, Parasoft added rules that check both 
conventions. You should decide which convention to use, then enable the
appropriate rule.
EXAMPLE 

package examples.rules.codsta.org;

public class DCI {
    public int getMax () {
        return MAX;
    }
    static final int MAX = 1000;  // VIOLATION
}
REPAIR 

Declare the constant in an "interface":

package examples.rules.codsta.org;

public class DCIFixed {
    public int getMax () {
        return Constants.MAX;
    }
}

interface Constants {
    int MAX = 1000;
}
