Do not have more than one type in each file [CODSTA.ORG.AMOC]



DESCRIPTION 


This rule identifies any file that has more than one type(class/interface/enum/annotation).
An error is reported for each occurrence.
SCOPE LEVEL 

FILE
SINCE 

v6.0
BENEFITS 

Having only one type definition per file
can provide a clearer overview of classes in the file system.
EXAMPLE 

package examples.rules.codsta.org;

public class AMOC //VIOLATION: more than 1 class presented
{
    //...
}

class myClass  //VIOLATION: more than 1 class presented
{
   //...
}
REPAIR 

package examples.rules.codsta.org;

public class AMOCFixed
{
    //...
}
//FIXED
