Avoid catch clauses with the same content [EXCEPT.CDUPL]



DESCRIPTION 

This rule identifies try-catch blocks where two or more catch clauses have the same content. 
The rule ignores comments, spaces and tabs when comparing content.
SINCE 
v9.4
NOTES 
This rule has one quickfix - "Catch duplicated blocks in one catch block"
which removes duplicated blocks and adds exception to exceptions list in first parentheses.
BENEFITS 
In Java SE 7 and later, a single catch block can handle more than one type of exception. 
This rule limits duplication of code and reduces the likelihood that additional general 
exceptions will be caught.
EXAMPLE 

        try {
            if(a == 1) {
                throw new ValidationException("test");
            }
            throw new NullPointerException();
        } catch (NullPointerException e) { // VIOLATION
            e.printStackTrace();
        } catch (ValidationException e) { // VIOLATION 
            e.printStackTrace();
        }
REPAIR 

        try {
            if(a == 1) {
                throw new ValidationException("test");
            }
            throw new NullPointerException();
        } catch (NullPointerException|ValidationException e) { // FIXED
            e.printStackTrace();
        }
