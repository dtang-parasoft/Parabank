APSC_DV.002590 The application must not be vulnerable to overflow attacks. [APSC_DV.002590]

RULES

Do not cast primitive data types to lower precision [APSC_DV.002590.CLP]
Avoid calculations which result in overflow or NaN [APSC_DV.002590.ICO]
Avoid integer overflows [APSC_DV.002590.INTOVERF]
Use unsigned right shift instead of division when overflow is possible [APSC_DV.002590.IOF]
