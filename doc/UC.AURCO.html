Avoid collection objects that are never read [UC.AURCO]



DESCRIPTION 


This rule identifies unread collection objects in local variables and private 
fields. Collection objects which are modified via mutator methods or sorted (if 
applicable), but never read are flagged. The rule checks objects of type 
"java.util.List", "java.util.Map", "java.util.Set" and their respective
subclasses.
SCOPE LEVEL 
CLASS
SINCE 

v8.4
BENEFITS 

An unread collection object may indicate a logical error or code mistakingly
left in place. Removal of unnecessary code can improve performance and simplify
code.
EXAMPLE 

package examples.rules.uc;

public class AURCO {
    private Map _data = new HashMap();

    public void reset() {
        _data.clear();
    }
    public void addProperty(String name, String value) {
        _data.put(name, value);
    }
}
REPAIR 

Use the collection object or remove if unnecessary.

package examples.rules.uc;

public class AURCOFixed {
    private Map _data = new HashMap();

    public void reset() {
        _data.clear();
    }
    public void addProperty(String name, String value) {
        _data.put(name, value);
    }
    public Map getData() { //now possesses a way to read the data
        return _data;
    }
}
