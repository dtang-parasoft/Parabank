Enforce number of blank line(s) to separate "imports" from different packages [APSC_DV.003215.BLSIM]



DESCRIPTION 


This rule identifies code that does not have the specified number of blank line(s) to separate
"import" statements from different packages. An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-Number of blank lines (default value is 2)
-Ignore adjacent imports if they are from subpackages of the same package
    -Default is "false".
    -If this parameter is enabled, the rule will not flag a violation for 
     adjacent imports if they are in different subpackages of the same package 
     (or if one import is from the top-level package and the other is from a 
     subpackage of this package).  For instance, if there are adjacent imports 
     from the "java.util" and the "java.util.logging" package, the rule will not 
     flag a violation since "java.util.logging" is a subpackage of "java.util".  
     When this parameter is enabled, the rule will ignore adjacent imports when 
     the first two segments of their package names (i.e., everything up to the 
     second dot in the package name) are the same.
BENEFITS 

If all team members follow the same formatting style, it will be easier for each
team member to read and understand the code.
EXAMPLE 

package examples.rules.format;

import java.util.*;
import java.awt.Window;  //  VIOLATION 
import java.awt.Frame;

class BLSIM {
}
REPAIR 

Use a blank line to separate the "import" statements from different packages.

package examples.rules.format;

import java.util.*;

import java.awt.Window;  // FIXED
import java.awt.Frame;

class BLSIMFixed {
}
