Place a closing brace on its own line [APSC_DV.003215.CBRACE]



DESCRIPTION 


As a general rule, the closing brace should be set on a separate line.
This rule detects closing braces that are not on their own lines, with 
the following exceptions:
- closing braces in array initializers.
- closing braces that are on the same line as their corresponding opening braces.

By default, the rule ignores the following keywords and tokens that appear after
the closing brace on the same line if the closing brace the first non-whitespace
token of a line:
    - else
    - catch
    - finally
    - while (only if part of a do...while loop)
    - }
    - ;
    - /*
    - //
The rule can be parameterized to report the above cases (see PARAMETERS).
SCOPE LEVEL 
LINE
SINCE 

v4.5
NOTES 

The rule does not analyze lambdas and anonymous class declarations.
PARAMETERS 

-Ignore case "} else {" (enabled by default)
-Ignore case "} catch {" (enabled by default)
-Ignore case "} finally {" (enabled by default)
-Ignore case "} while();" (enabled by default)
-Ignore case "}}" (enabled by default) - applies only to double closing braces that are not followed by any characters 
-Ignore case "} ;" (enabled by default)
-Ignore case "} /* " (enabled by default)
-Ignore case "} // " (enabled by default)
BENEFITS 

This rule helps all team members follow the same formatting style to make it easier
to read and understand the code.
EXAMPLE 

package examples.rules.format;

public class CBRACE {
    private int index; }  // VIOLATION
REPAIR 

Put the closing "}" brace on its own line.

package examples.rules.format;

public class CBRACEFixed {
    private int index;
}  // FIXED
