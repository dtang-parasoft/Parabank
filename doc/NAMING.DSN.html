Avoid using dollar signs in names [NAMING.DSN]



DESCRIPTION 


This rule flags a violation for names containing '$'. The names which are
checked can be specified through the parameters.
SINCE 

v9.0
PARAMETERS 

-Check type declarations (default true)
    -Check annotation declarations (default true)
    -Check enum declarations (default true)
    -Check class declarations (default true)
    -Check interface declarations (default true)
-Check method declarations (default true)
-Check variable declarations (default true)
    -Check field declarations (default true)
    -Check local variable declarations (default true)
    -Check parameter declarations (default true)
BENEFITS 

Java may use '$' in auto-generated names. Standard practice is to avoid using
'$' in names in spite of it being a valid identifier.
EXAMPLE 

package examples.rules.naming;

public class DSN {
    class Inner$Class { //VIOLATION
    }
}
REPAIR 

package examples.rules.naming;

public class DSNFixed {
    class Inner$Class { //FIXED
    }
}
