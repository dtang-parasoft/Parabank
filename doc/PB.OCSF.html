Ensure that exactly one type has the same name as the file name [PB.OCSF]



DESCRIPTION 


This rule identifies any Java file that does not have one non-inner type(class/interface/enum/annotation)
with the same name as the file name. An error is reported for each occurrence.
SCOPE LEVEL 

FILE
SINCE 

v6.0
NOTES 

Inner classes are not counted.
BENEFITS 

Rule violations typically point to a typo in either the file
name or the type name.
EXAMPLE 

file name: OCSF.java

package examples.rules.pb;

class OCSF_ //VIOLATION
{
}
REPAIR 

file name: OCSF.java

package pb;

class OCSF    //FIXED
{
}
