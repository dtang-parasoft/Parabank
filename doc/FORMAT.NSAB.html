Do not place statements on the same line as the '{' opening brace [FORMAT.NSAB]



DESCRIPTION 


This rule identifies statements that follow '{' opening braces on the same line.
An error is reported for each occurrence. No error is reported if the matching
closing brace also appears on the same line.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

- Allow statements on the same line as the opening brace if the opening and closing 
  braces appear on the same line. (default is true)
  
  If enabled, this rule would not flag this special case:
  
  if (...) {  statement;  }
BENEFITS 

If all team members follow the same formatting style, it will be easier for each
team member to read and understand the code.
EXAMPLE 

package examples.rules.format;

public class NSAB {
    void foo () { int i = 0;  // VIOLATION
    }
}
REPAIR 

Place the statement on the next line.

package examples.rules.format;

public class NSABFixed {
    void foo () {   
    int i = 0;  // FIXED
    }
}
