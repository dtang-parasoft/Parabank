Do not declare any thrown exceptions in the 'main()' method [EXCEPT.MTE]



DESCRIPTION 


This rule will flag a violation for each 'main()' method which is declared
to throw any exceptions.  The rule will only check methods which have the 
signature 'static void main(String[] args) ...'.
SINCE 

v8.1
BENEFITS 

The user should be able to execute the 'main()' method without having to
worry about any exception handling.  Thus, the 'main()' method should not
declare any thrown exceptions.
EXAMPLE 

package examples.rules.except;

public class MTE {
    public static void main(String args[]) throws Exception {  //VIOLATION
        foo();
    }
    
    private static void foo() throws Exception {
        throw new Exception();
    }
}
REPAIR 

Instead of declaring the exception to be thrown, handle it appropriately
in the 'main()' method using a 'try' block.

package examples.rules.except;

public class MTEFixed {
    public static void main(String args[]) {  //FIXED
        try{
            foo();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
    
    private static void foo() throws Exception {
        throw new Exception();
    }
}
