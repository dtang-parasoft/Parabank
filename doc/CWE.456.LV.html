Initialize all local variables explicitly at the declaration statement [CWE.456.LV]



DESCRIPTION 



This rule identifies local variables that are not initialized in their 
declaration. An error is reported for each occurrence.
SINCE 

v4.5
NOTES 

This rule will not flag "final" local variables since the compiler enforces
that "final" local variables are explicitly initialized.
BENEFITS 

Uninitialized variables are a potential source of bugs.
EXAMPLE 

package examples.rules.init;

public class LV {
    public boolean method (int size) {
        int max;  // VIOLATION
        return size > 0;
    }
}
REPAIR 

Initialize all local variables in their declaration.

package examples.rules.init;

public class LVFixed {
    public boolean method (int size) {
        int max = 0;  // FIXED
        return size > 0;
    }
}
