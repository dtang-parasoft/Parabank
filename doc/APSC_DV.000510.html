APSC_DV.000510 The application must execute without excessive account permissions. [APSC_DV.000510]

RULES

Avoid using anonymous "privileged" classes when invoking "AccessController.doPrivileged()" [APSC_DV.000510.ACDP]
Declare subclasses of 'PrivilegedAction', 'PrivilegedExceptionAction', and 'PrivilegedActionException' "final" [APSC_DV.000510.PAF]
Wrap "privileged" method invocations in "final" methods [APSC_DV.000510.PCFM]
Limit the number of lines in "privileged" code blocks [APSC_DV.000510.PCL]
Wrap "privileged" method invocations in "private" methods [APSC_DV.000510.PCPM]
Avoid using dynamically loaded classes in "privileged" code blocks [APSC_DV.000510.PDLC]
Ensure that all Permissions, PrivilegedActions, and PrivilegedActionExceptions are declared in the same package [APSC_DV.000510.PPKG]
Avoid operating on tainted data in privileged blocks [APSC_DV.000510.PRIVIL]
Declare the 'run()' method of 'PrivilegedAction' and 'PrivilegedExceptionAction' implementations "final" [APSC_DV.000510.PRMF]
