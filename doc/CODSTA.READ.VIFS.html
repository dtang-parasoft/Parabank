Limit the number of initialization and update statements in "for" loops [CODSTA.READ.VIFS]



DESCRIPTION 


In the initialization and update parts of a "for" statement, there should be an
upper limit on the number of statements that are allowed.  This rule will flag
any case where the programmer puts more than the maximum allowed number of 
statements in the comma-separated list of statements for the initialization
or update.  By default, the rule checks to make sure there are no more than 3
statements in either the initialization or the update part of any "for" 
statement.
SCOPE LEVEL 
LINE
SINCE 

v8.0
PARAMETERS 

-Maximum number of statements to allow in the initialization and update:
(Default is 3.)
BENEFITS 

Having too many statements in the initialization or update part of a "for"
statement can make the code difficult to read.  If there is a violation of this
rule, some of the statements should be moved out of the initialization/update 
and put either before the loop or at the end of the loop body.
EXAMPLE 

package examples.rules.codsta;

public class VIFS {
    for(i=0, j=0, k=0, l=0; i<10; i++, j++, k++, l++){ //VIOLATION
        i++;
    }
}
REPAIR 

Move one of the initialization statements to immediately before the "for" loop
and one of the update statements to the end of the "for" loop body.
 
package examples.rules.codsta.read;

public class VIFSFixed {
    l= 0;
    for(i=0, j=0, k=0; i<10; i++, j++, k++){ //FIXED
        i++;
        l++;
    }
}
