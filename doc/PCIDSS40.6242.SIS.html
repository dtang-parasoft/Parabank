Do not change the input streams of 'java.lang.System' in a web component [PCIDSS40.6242.SIS]



DESCRIPTION 


Web components should not change the input streams of  'java.Lang.System'. This
rule will flag any call to the method 'System.setIn()' in a web component. A
class or interface is considered a web component if it extends or implements a
type from the "javax.servlet" package.
SINCE 

v7.5
SECURITY RELEVANCE 

category: Erratic Application Behavior

Changing the input streams of 'System' is a suspicious security pattern.
PARAMETERS 

-Maximum depth to look for "System.setIn()" calls (default "2")
    This parameter allows for indirect checking of "System.setIn()" calls.
    Specifying a value of "1" will only check for direct calls to
    "System.setIn()". A value of "2" will check if any of the methods called
    from the web component call "System.setIn()", and so on.
    Note that using a large value may significantly slow execution of the rule.
BENEFITS 

Changing the input streams of 'System' is something that is typically not 
necessary for a web application.  Doing so also indicates a suspicious security 
pattern.
EXAMPLE 

package examples.rules.security.eab;

import javax.servlet.Servlet;

public class SIS implements Servlet {
    public void testMethod() {
        System.setIn(null); //VIOLATION
    }
}
REPAIR 

Remove any calls to the 'setIn()' method.

package examples.rules.security.eab;

import javax.servlet.Servlet;

public class SISFixed implements Servlet {
    public void testMethod() {
        //FIXED
    }
    ...
}
