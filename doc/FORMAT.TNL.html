Make sure all files are terminated with a newline character [FORMAT.TNL]



DESCRIPTION 


This rule identifies any file that is not terminated by a newline character.
An error is reported for each occurrence.
SINCE 

v6.0
BENEFITS 

It is good practice to always terminate a file with a
newline character. Failure to follow this convention 
might cause errors during CVS checking.
REFERENCES 

DISA Application Security and Development Security Technical Implementation Guide (STIG)
APSC-DV-003215: The application development team must follow a set of coding standards
