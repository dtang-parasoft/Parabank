Ensure that a setup method is defined in the test class [JUNIT.OSUM]



DESCRIPTION 


This rule identifies JUnit test classes that fail to perform set up before
the test cases are executed.
The rule detects:
- In JUnit 3, classes that not override the 'setUp()' method.
- In JUnit 4, classes that do not contain any methods annotated with @Before.
- In JUnit 5, classes that do not contain any methods annotated with @BeforeEach.
SCOPE LEVEL 

CLASS
SINCE 

v4.5
NOTES 

This rule assumes that test classes are classes that meet the following
conditions:
- In JUnit 3, a test class is a class that extends the junit.framework.TestCase
  class.
- In JUnit 4, a test class is a class that contains a method marked with
  the @Test annotation.
- In JUnit 5, a test class is a class that contains a method marked with
  one of the following annotations: @Test, @TestFactory, @TestTemplate,
  @ParameterizedTest, @RepeatedTest
BENEFITS 

This rule helps you ensure that all variables, open database, and network
connections are initialized before the tests defined in the class are executed.
EXAMPLE 

package examples.rules.junit;

import org.junit.jupiter.api.*;

public class OSUM {  // VIOLATION: missing 'BeforeEach' method

    @Test
    public void testMethod() {
        //some test code
    }

    @AfterEach
    public void cleanUp () {
        // cleanup code
    }
}
REPAIR 

'@BeforeEach' method has been added.

package examples.rules.junit;

import org.junit.jupiter.api.*;

public class OSUMFixed {

    @BeforeEach
    public void setUp () {  // FIXED
        //setup code
    }

    @Test
    public void testMethod() {
        //some test code
    }

    @AfterEach
    public void cleanUp () {
        // cleanup code
    }
}
