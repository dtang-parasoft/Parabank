Do not override "abstract" methods of a parent class with "abstract" methods [UC.AMAMI]



DESCRIPTION 


This rule identifies code that overrides "abstract" methods 
of a parent class with "abstract" methods. An error is reported
for the "abstract" method declaration that overrides the "abstract"
method.
SCOPE LEVEL 

METHOD
SINCE 

v9.0
NOTES 

N/A
SECURITY RELEVANCE 

N/A
PARAMETERS 

N/A
BENEFITS 

If a parent class has an abstract method, there is no reason to 
override it with an abstract method.  The second declaration can
not change program execution, and is extra code.
DRAWBACKS 

N/A
EXAMPLE 

package examples.rules.uc;

interface SUPER {
    void m();
}

interface AMAMI extends SUPER{
    void m(); //VIOLATION
}
REPAIR 

Remove the override.

package examples.rules.uc;

interface SUPER {
    void m();
}

interface AMAMIFixed extends SUPER{
    //FIXED
}
