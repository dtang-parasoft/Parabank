CWE-287: Improper Authentication [CWE.287]

RULES

Avoid using cryptographic keys which are too short [CWE.287.CKTS]
Avoid DNS lookups for decision making [CWE.287.DNSL]
Avoid using hard-coded cryptographic keys [CWE.287.HCCK]
Avoid passing hardcoded usernames/passwords/URLs to database connection methods [CWE.287.HCCS]
Do not rely on IP addresses obtained from HTTP request headers for authentication [CWE.287.HTTPRHA]
Ensure the HostnameVerifier.verify() method validates the certificate [CWE.287.HV]
Use the 'minlength' validator for password fields in 'validation.xml' [CWE.287.MLVP]
Ensure sufficient protection against multiple failed authentication attempts [CWE.287.PBFA]
Avoid storing usernames and passwords in plain text in Castor 'jdo-conf.xml' files [CWE.287.PCCF]
Password information should not be included in properties file in plaintext [CWE.287.PLAIN]
Avoid using plain text passwords in Axis 'wsdd' files [CWE.287.PTPT]
Ensure that passwords are not stored as plaintext and are sufficiently long [CWE.287.PWDPROP]
Ensure that passwords are not stored as plaintext and are sufficiently long [CWE.287.PWDXML]
Ensure that an appropriate security manager is set [CWE.287.SSM]
Protect against using unprotected credentials [CWE.287.TDPASSWD]
Avoid storing unencrypted Hibernate usernames and passwords in 'web.xml' files [CWE.287.UPWD]
Use the SSL-enabled version of classes when possible [CWE.287.USC]
Avoid using plain text passwords in Axis2 configuration files [CWE.287.UTAX]
Properly validate server identity [CWE.287.VSI]
Avoid unencrypted passwords in WebSphere 'ibm-webservicesclient-ext.xmi' files [CWE.287.WCPWD]
Avoid unencrypted passwords in WebSphere 'ibm-webservices-ext.xmi' files [CWE.287.WPWD]
