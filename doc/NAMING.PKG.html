Use a naming convention for "package" names [NAMING.PKG]



DESCRIPTION 


This rule identifies "package" names that do not follow the
user-defined naming convention (as specified in a regular expression).
An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v6.0
PARAMETERS 

-regular expression
    -Package names should match this expression.
    -The default is "^[a-z]([a-z]|[0-9]|\.|_)+$".  This enforces that package
     names always begin with a lower case letter and contain only lower case
     letters, numbers, and the "." and "_" characters.
-Negate regular expression
    -If this parameter is enabled, a violation will be flagged if the name of a
     package matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be
     flagged for package names which do not match the regular expression
     specified.
-Ignore case
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that package names must
     match the regular expression specified, including in case.
BENEFITS 

This helps enforce a naming convention.
EXAMPLE 

package examples.rules.NAMING;  // VIOLATION

public class PKG {
}
REPAIR 

Change upper case letters to lower case letters.

package examples.rules.naming;  // FIXED

public class PKGFixed {
}
