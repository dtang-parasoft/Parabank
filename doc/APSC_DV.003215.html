APSC_DV.003215 The application development team must follow a set of coding standards. [APSC_DV.003215]

RULES

Use '()' to separate complex expressions [APSC_DV.003215.APAREN]
Ensure proper spacing in array references [APSC_DV.003215.ASPACE]
Do not place empty whitespace at the end of a line [APSC_DV.003215.ATS]
Put a blank line before each C-style comment [APSC_DV.003215.BLBC]
Enforce number of blank line(s) before type declarations [APSC_DV.003215.BLCD]
Enforce number of blank line(s) to separate "imports" from different packages [APSC_DV.003215.BLSIM]
Place a closing brace on its own line [APSC_DV.003215.CBRACE]
Place a single space character or no space character after type casting [APSC_DV.003215.CMS]
Do not leave spaces between qualified names and method invocations [APSC_DV.003215.DOT]
Use spaces instead of tabs (or tabs instead of spaces) [APSC_DV.003215.DUT]
Enforce the position of '{' brace [APSC_DV.003215.FCB]
Place the type that has the same name as the file as the first type [APSC_DV.003215.FCN]
Declare arrays with '[]' brackets after the array type and before the variable name(s) [APSC_DV.003215.IAD]
Enforce number of space(s) for indentation [APSC_DV.003215.IND]
Limit the maximum length of a line [APSC_DV.003215.LL]
Include a meaningful file header comment in every source file [APSC_DV.003215.MCH]
Enforce the order of annotations and modifiers [APSC_DV.003215.MO]
Place a single space character or no space character between a method name and the opening "(" parenthesis [APSC_DV.003215.MSP]
Do not place statements on the same line as the '{' opening brace [APSC_DV.003215.NSAB]
Write one statement per line [APSC_DV.003215.OSPL]
Enforce number of space character(s) after every comma [APSC_DV.003215.SAC]
Enforce number of space character(s) on each side of an assignment operator [APSC_DV.003215.SAOP]
Enforce number of space character(s) after the opening parenthesis "(" of a conditional statement [APSC_DV.003215.SAP]
Enforce number of space character(s) after every semicolon [APSC_DV.003215.SAS]
Enforce number of space character(s) between a prefixed unary operator and its operand [APSC_DV.003215.SAUOP]
Enforce number of space character(s) on each side of a bitwise operator [APSC_DV.003215.SBOP]
Enforce number of space character(s) between a postfixed unary operator and its operand [APSC_DV.003215.SBUOP]
Enforce number of space character(s) between a conditional statement and the opening "(" parenthesis [APSC_DV.003215.SC]
Enforce number of space character(s) before and after the "?" conditional operator [APSC_DV.003215.SCOP]
Enforce number of space character(s) on each side of a logical operator [APSC_DV.003215.SLOP]
Enforce number of space character(s) on each side of a relational operator [APSC_DV.003215.SROP]
Avoid using trailing comments [APSC_DV.003215.TC]
Avoid or enforce the use of trailing commas in array initializers [APSC_DV.003215.TCOMMA]
Use the preferred formatting for conditional expressions [APSC_DV.003215.TE]
Make sure all files are terminated with a newline character [APSC_DV.003215.TNL]
Enforce number of blank line(s) between major sections [APSC_DV.003215.U2BL]
Avoid unnecessary parentheses [APSC_DV.003215.UP]
