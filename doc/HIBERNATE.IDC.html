Declare a no-argument constructor for Hibernate persistent classes [HIBERNATE.IDC]



DESCRIPTION 


This rule looks for Hibernate persistent classes which do not define a 
constructor with no arguments (and do not have a default no-argument 
constructor).  It only checks classes which are mapped in a "Hibernate-mapping"
document.
SCOPE LEVEL 
LINE
SINCE 

v8.0
NOTES 

This rule only checks classes which are mapped in a "Hibernate-mapping"
document.  It looks for these mappings in all files in the tested project whose
names end in the ".hbm.xml" extension.

This rule was created for use with Hibernate 3.1.  It is not guaranteed to apply
to all versions of Hibernate.  You should not enable this rule if it does not
apply to the version of Hibernate you are using.
BENEFITS 

According to the Hibernate website, a no-argument constructor is required for
persistent classes since Hibernate has to create instances of these classes
through reflection.
EXAMPLE 

This example assumes that there is a mapping for the class "IDC" in a 
".hbm.xml" file in the tested project.

package examples.rules.hibernate;
         
public class IDC {  //VIOLATION- no no-argument constructor
    private Long id;

    public IDC(Long l) {
        id= l;
    }

    public Long getId() {
        return id;
    }

    public void setId(Long id) {
        this.id = id;
    }
}
REPAIR 

Add a no-argument constructor.
       
package examples.rules.hibernate;
         
public class IDCFixed {
    private Long id;

    public IDCFixed() {  //FIXED
    }
    
    public IDCFixed(Long l) {
        id= l;
    }

    public Long getId() {
        return id;
    }

    public void setId(Long id) {
        this.id = id;
    } 
}
