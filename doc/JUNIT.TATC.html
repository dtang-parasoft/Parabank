Use '@Test' annotation instead of extending 'TestCase' [JUNIT.TATC]



DESCRIPTION 


This rule detects classes that extend the junit.framework.TestCase class
to facilitate migration from JUnit 3 to a higher version of the framework.
See BENEFITS.
SINCE 

v9.0
NOTES 

See also the JUNIT.ETCTA rule.
PARAMETERS 

-Only check files which import from the org.junit package (default false)
    With this parameter enabled, the rule will only check files that include
	import of the org.junit package.
BENEFITS 

This rule helps you migrate from JUnit 3 to JUnit 4 or higher formatting,
which allows for greater flexibility in naming methods and may improve
readability.
In addition, it allows you to introduce a consistent format across all test
cases to improve clarity.
EXAMPLE 

package examples.rules.junit;

import junit.framework.TestCase;

public class TATC extends TestCase { //VIOLATION
    public void testMethod() {
        ...
    }
}
REPAIR 

package examples.rules.junit;

import org.junit.Test;

public class TATC_Fixed { //FIXED
    @Test
    public void method() {
        ...
    }
}
