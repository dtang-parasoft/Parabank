Naming Conventions [NAMING] 

Rules that enforce common naming conventions.
RULES

Follow class name conventions defined for annotations [NAMING.ANNS]
Use conventional variable names [NAMING.CVN]
Do not declare types with the same name as types in the Java platform [NAMING.DJLO]
Avoid using dollar signs in names [NAMING.DSN]
Ensure class names reflect classes which they extend [NAMING.ECN]
Follow class name conventions defined for base classes [NAMING.EXTENDS]
Follow a naming convention for getter methods [NAMING.GETA]
Follow a naming convention for "boolean" getter methods [NAMING.GETB]
Avoid using potential Java keywords as identifiers [NAMING.ID]
Use all uppercase letters for the names of fields in an "interface" [NAMING.IFV]
Follow class name conventions defined for implemented interfaces [NAMING.IMPLS]
Use 'is...' only for naming methods that return a "boolean" [NAMING.IRB]
Follow limits for the lengths of type, method, field, parameter, and variable names [NAMING.LLI]
Use a naming convention for annotations [NAMING.NA]
Use a naming convention for array and collection variables [NAMING.NAC]
Use a naming convention for "abstract" classes [NAMING.NACL]
Use a naming convention for classes [NAMING.NCL]
Use a naming convention for exceptions [NAMING.NE]
Use a naming convention for enum type declarations [NAMING.NENUM]
Use a naming convention for "final" local variables [NAMING.NFL]
Use a naming convention for non-"static" fields [NAMING.NIF]
Use a naming convention for interfaces [NAMING.NITF]
Use a naming convention for local variables [NAMING.NLV]
Use a naming convention for non-"static" methods [NAMING.NM]
Use a naming convention for method parameters [NAMING.NMP]
Use a naming convention for type parameters [NAMING.NPH]
Use a naming convention for non-"final" "static" fields [NAMING.NSF]
Use a naming convention for "static" methods [NAMING.NSM]
Use a naming convention for JUnit test classes [NAMING.NTEST]
Use a naming convention for "package" names [NAMING.PKG]
Do not use a package name that is reserved by Sun [NAMING.RPKG]
Follow a naming convention for setter methods [NAMING.SETA]
Use a naming convention for singleton classes [NAMING.SINGLETON]
Use a naming convention for tag handlers and associated classes [NAMING.THAC]
Use Hungarian notation for variables [NAMING.UHN]
Do not use lowercase letters in "final" "static" field names [NAMING.USF]
Use a naming convention for utility classes [NAMING.UTIL]
Do not use variables with identical names or names that differ only in case [NAMING.UUVN]
