6.2.4.3 Attacks on cryptography usage [PCIDSS40.6243]

RULES

Avoid using custom MessageDigest implementations [PCIDSS40.6243.ACMD]
Avoid using insecure cryptographic algorithms for data encryption with Spring [PCIDSS40.6243.AISSAJAVA]
Avoid using insecure cryptographic algorithms in Spring XML configurations [PCIDSS40.6243.AISSAXML]
Avoid using the javax.crypto.NullCipher class in non-test classes [PCIDSS40.6243.AUNC]
Avoid using cryptographic keys which are too short [PCIDSS40.6243.CKTS]
Avoid using hard-coded cryptographic keys [PCIDSS40.6243.HCCK]
Avoid using insecure algorithms for cryptography [PCIDSS40.6243.ICA]
Use hash functions with a salt [PCIDSS40.6243.MDSALT]
Avoid storing sensitive data in plaintext in a cookie [PCIDSS40.6243.PLC]
Ensure that passwords are not stored as plaintext and are sufficiently long [PCIDSS40.6243.PWDPROP]
Ensure that passwords are not stored as plaintext and are sufficiently long [PCIDSS40.6243.PWDXML]
Use 'java.security.SecureRandom' instead of 'java.util.Random' or 'Math.random()' [PCIDSS40.6243.SRD]
Ensure SOAP messages are encrypted in WebSphere 'ibm-webservicesclient-ext.xmi' files [PCIDSS40.6243.WCMC]
Ensure SOAP messages are digitally signed in WebSphere 'ibm-webservicesclient-ext.xmi' files [PCIDSS40.6243.WCMI]
Ensure WS-Security is enabled in WebSphere 'ibm-webservicesclient-ext.xmi' files [PCIDSS40.6243.WCMWS]
Avoid unencrypted passwords in WebSphere 'ibm-webservicesclient-ext.xmi' files [PCIDSS40.6243.WCPWD]
Avoid unencrypted passwords in WebSphere 'ibm-webservices-ext.xmi' files [PCIDSS40.6243.WPWD]
