Avoid misformatted Javadoc tags [JAVADOC.MISFORMAT]



DESCRIPTION 

This rule verifies that block tags are placed in a new line after the header. An error is reported
for each occurrence.
PARAMETERS 
-check "public" methods/annotation member types (default as true)
-check "protected" methods/annotation member types (default as true)
-check "package-private" methods/annotation member types (default as true)
-check "private" methods/annotation member types (default as false)
BENEFITS 
The correct use of block tags ensures that:
1) Java and other tools properly parses the tags and correctly displays them Eclipse.
2) Tools for generating documentation from javadoc generates the correct documentation.



Please see details:
http://www.oracle.com/technetwork/java/javase/documentation/index-137868.html#format
EXAMPLE 
/**Header @firstInncorrectBlocktag //VIOLATION, there is a "Header" string before "@"
*
* @blocktag @secondInncorrectBlocktag //VIOLATION, not in separate line
*/
REPAIR 
Block tags must be place at the beginning of a new javadoc line using the following syntax:
@blocktag. The first incorrect block tag is misplaced and treated as a part of a header. Second
incorrect block tag is treated as a part of another block tag because it does not start on a new
line.


/**Header
*
* @firstInncorrectBlocktag //FIXED
* @blocktag
* @secondInncorrectBlocktag //FIXED
*/
