Avoid @prefix, @suffix, and @attribute for unnamed '<action>' elements in 'struts-config.xml' [STRUTS.APSN]



DESCRIPTION 


This rule checks 'struts-config.xml' files. A violation is flagged for each
'<action>' element with no @name attribute and one of the following attributes:
@attribute, @prefix, @suffix.
SINCE 

v8.4
SECURITY RELEVANCE 

category: Unsafe Environment Configuration

The attributes @attribute, @prefix, and @suffix all have meaning in relation to
the @name attribute. If they are used and no @name is present it may be a sign
of an erroneous or inconsistent setup.
EXAMPLE 

The '<action>' element in the sample 'struts-config.xml' file has a @prefix
attribute but does not have a @name attribute.

struts-config.xml:

<?xml version="1.0" encoding="ISO-8859-1" ?>
<struts-config>
    <form-beans>
        <form-bean name="logonForm" type="app.LogonForm"/>
    </form-beans>
    <action-mappings>
        <!-- VIOLATION: No @name attribute  -->
        <action
            path="/LogonSubmit"
            type="app.LogonAction"
            scope="request"
            validate="true"
            attribute="actionAttribute"
        </action>
    </action-mappings>
</struts-config>
REPAIR 

Add a @name if it was mistakingly left out or remove the unnecessary @attribute.

struts-config.xml:
    
<?xml version="1.0" encoding="ISO-8859-1" ?>
<struts-config>
    <form-beans>
        <form-bean name="logonForm" type="app.LogonForm"/>
    </form-beans>
    <action-mappings>
        <!-- FIXED -->
        <action
            path="/LogonSubmit"
            type="app.LogonAction"
            name="logonForm"
            scope="request"
            validate="true"
            attribute="actionAttribute"
        </action>
    </action-mappings>
</struts-config>
