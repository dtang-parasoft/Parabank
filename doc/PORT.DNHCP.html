Do not hard code an absolute pathname when calling a constructor from the 'File' class [PORT.DNHCP]



DESCRIPTION 


This rule checks that the path that is passed to the constructor for 
'java.io.File' is not an absolute pathname.  A violation will be flagged for
each call to a 'File' constructor which takes in a path starting in either "/" 
or "\".
SCOPE LEVEL 
LINE
SINCE 

v8.0
NOTES 

This rule not only checks 'java.io.File', but also checks the following classes
   - 'java.io.FileInputStream'
   - 'java.io.FileOutputStream'
   - 'java.io.FileReader'
   - 'java.io.FileWriter'
BENEFITS 

Using hardcoded pathnames limits the portability of code because it requires
that the person using the code have the same directory structure as the
programmer.
EXAMPLE 

package examples.rules.port;

import java.io.*;

public class DNHCP {
    File f;
    
    DNHCP() {
        f= new File("/home/dannyc/workspace/j2ee/src/share/com/sun/enterprise/deployment"); //VIOLATION   
    }
}
REPAIR 

Do not hard code the path passed to "File"'s constructor.

package examples.rules.port;

public class DNHCPFixed {
    File f;
    
    DNHCPFixed(String path) {
        f= new File(path); //FIXED  
    }
}
