Use ids as bean identifiers [SPRING.USEID]



DESCRIPTION 


This rule identifies uses of the 'name' attribute of the <bean/> element in Spring xml configuration files. 
A violation is reported for each occurrence of this attribute where the id attribute is not specified.
SINCE 

v9.2
NOTES 

You can specify either an id or name as the bean identifier. Using id is preferred for the reason
below.
BENEFITS 

If you follow this rule, the XML parser can validate the bean references.
EXAMPLE 

<bean name="listOrdersController" class="org.springframework.samples.jpetstore.web.spring.ListOrdersController" />
REPAIR 

Change name to id:

<bean id="listOrdersController" class="org.springframework.samples.jpetstore.web.spring.ListOrdersController" />
REFERENCES 

1. Twelve Best Practices For Spring XML Configurations by Jason Zhicheng Li (O'Reilly ONJava.com)
