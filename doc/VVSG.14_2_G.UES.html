Avoid unnecessary "else" statements [VVSG.14_2_G.UES]



DESCRIPTION 


This rule checks that unnecessary "else" statements are not used.  An "else" 
statement is unnecessary if every path of execution through the body of the 
corresponding "if" block ends with either a "return" statement which causes a
return from the enclosing method or a "break" or "continue" statement which 
causes an exit from the enclosing loop.  In this case, the only way for the 
"else" block to be executed is if the corresponding "if" condition is false 
(even without the "else" statement).  Thus, it is not necessary to use an 
explicit "else" statement.
SCOPE LEVEL 
LINE
SINCE 

v8.0
PARAMETERS 
-Ignore "else if" statements (default on)
BENEFITS 

In the case described in the "DESCRIPTION" section, the "else" statement is 
unnecessary.  Leaving unnecessary code in a file will clutter up the file and is
potentially confusing.
EXAMPLE 

package examples.rules.uc;

public class UES {
    public boolean isLessThanZero(int i){
        if(i<0){
            return true;
        }
        else{  //VIOLATION- unnecessary "else" statement
            return false;
        }
    }
}
REPAIR 

package examples.rules.uc;

public class UESFixed {
    public boolean isLessThanZero(int i){
        if(i<0){
            return true;
        }
        return false;  //FIXED
    }
}
