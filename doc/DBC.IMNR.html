Do not invoke a method on a reference that is not guaranteed to be non-null [DBC.IMNR]



DESCRIPTION 


Statements of the form "methodA(...).methodB(...)" are common and are based on 
the assumption that "methodA(...)" returns a non-null result.  If "methodA()" is 
guaranteed to return a non-null result, it should have the DbC annotation "@post 
$result != null" to indicate this.  This rule will flag a violation for each
method invocation of the form "methodA(...).methodB(...)" where "methodA()" is
not guaranteed by a DbC annotation to return a non-null result.  A violation 
will be flagged for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v8.0
BENEFITS 

Enforcing this rule will prevent unexpected "NullPointerExceptions" from 
occurring at runtime.
EXAMPLE 

package examples.rules.dbc;

public class IMNR {

    public void example() {
        method1().method0(); // VIOLATION
    }
    
    public void method0() {
        return;
    }
    
    public IMNR method1() {
        return null;
    }
}
REPAIR 

Make sure the offending method cannot return "null", and add the DbC annotation
"@post $result != null" to the method. 

package examples.rules.dbc;

public class IMNRFixed {

    public void example() {
        method1().method0(); // FIXED
    }
    
    public void method0() {
        return;
    }
    
    /**
     * @post $result != null
     **/
    public IMNRFixed method1() {
        return new IMNRFixed();
    }
}
