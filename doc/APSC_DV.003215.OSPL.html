Write one statement per line [APSC_DV.003215.OSPL]



DESCRIPTION 


This rule identifies any line with more than a single statement. An error is
reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v4.5
NOTES 

Generally, this rule uses the same definition of the term 'statement' that a 
Java parser uses. However, there are certain exceptions:

- 'break' is not considered a statement; this allows for the common pattern
  of single-line cases in a switch statement (e.g., 'case 0: a = 0; break;')
- Field declarations, this(...)/super(...) expressions, and catch clauses are
  considered statements; these constructs are very similar to statements in
  their use and are therefore treated like statements
BENEFITS 

If all team members follow the same formatting style, it will be easier for each
team member to read and understand the code.
EXAMPLE 

package examples.rules.format;

public class OSPL {
    int method (int a, int b) {
        int i = a + b; return i;  // VIOLATION
    }
}
REPAIR 

package examples.rules.format;

public class OSPLFixed {
    int method (int a, int b) {
        int i = a + b;  // FIXED
        return i;  // FIXED
    }
}
