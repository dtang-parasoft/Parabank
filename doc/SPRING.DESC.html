Use ''description'' tag in configuration file headers [SPRING.DESC]



DESCRIPTION 


This rule identifies Spring xml configuration files that are missing the <description> tag.
SINCE 

v9.2
NOTES 

Use the Spring Framework configuration file header <description>...</description>
BENEFITS 

Following this rule makes it easy for externals tools (example Ant, Maven plugins) 
to pick up the description from the Spring Framework configuration.
EXAMPLE 

<?xml version="1.0" encoding="UTF-8"?>

<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd"

    <bean id="personSetterByName"
        class="org.springbyexample.springindepth.chapter03.autowire.PersonContainer" />
</beans>
REPAIR 

<?xml version="1.0" encoding="UTF-8"?>

<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd"
    
    <description> Some description...
    </description>
    
    <bean id="personSetterByName"
        class="org.springbyexample.springindepth.chapter03.autowire.PersonContainer" />

</beans>
