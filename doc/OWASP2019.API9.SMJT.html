Use the '@concurrency' Javadoc tag on "synchronized" methods and blocks [OWASP2019.API9.SMJT]



DESCRIPTION 


This rule identifies "synchronized" code (i.e., methods and "{ }" blocks)  
that lacks a Javadoc comment with '@concurrency' tag. 
An error is reported for each occurrence.
SCOPE LEVEL 
METHOD
SINCE 

v4.5
NOTES 

The '@concurrency' tag can be followed by one of four valid values:
    $none - Equivalent to having no tag.
    concurrent - No synchronization is necessary.
    guarded - Synchronization is necessary and is provided by the method.
    sequential - Synchronization is necessary and must be provided by client
        code using the method.
For more information see 'The Design by Contract Specification' in REFERENCES.
BENEFITS 

Ideally, your header will explain how the code maintains thread-safety and how
the users of the code should avoid threading problems.
EXAMPLE 

package examples.rules.javadoc;

public class SMJT {
    /**
     *  Thread safety is maintained because this method doesn't do anything
     */
    public synchronized void method () {  // VIOLATION
    }
}
REPAIR 

Provide the '@concurrency' tag

package examples.rules.javadoc;

public class SMJTFixed {
    /**
     *  Thread safety is maintained because this method doesn't do anything
     *  @concurrency guarded
     */
    public synchronized void method () {  // FIXED
    }
}
