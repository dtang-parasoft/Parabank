Enforce number of blank line(s) before type declarations [FORMAT.BLCD]



DESCRIPTION 


This rule identifies code that does not have the required number of blank lines before a 
"class" or "interface" or "enum" or "annotation" declaration. An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-Number of blank lines (default value is 2)
BENEFITS 

If all team members follow the same formatting style, it will be easier for each
team member to read and understand the code.
EXAMPLE 

package examples.rules.format;
public class BLCD {  // VIOLATION

}
REPAIR 

Place two blank lines before the "class" declaration.

package examples.rules.format;


public class BLCDFixed {  // FIXED

}
