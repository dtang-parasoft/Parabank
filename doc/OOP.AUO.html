Do not use an object to access "static" fields or methods [OOP.AUO]



DESCRIPTION 


Requires that you reference "static" members of a class through the class name.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-Allow use of an object to access "static" methods for chained method calls.
    -Default is false.  If this parameter is turned on, the rule will not flag
     a violation for chained method calls even if the methods are static 
     methods.
BENEFITS 

This improves the code's clarity.
EXAMPLE 

package examples.rules.oop;

public class AUO {
    public static void staticMethod () {}
    
    public void method () {}

    public static void main (String[] args) {
        AUO object = new AUO ();
        object.staticMethod ();  // VIOLATION
        object.method (); 
    }
}
REPAIR 

Access "static" methods and fields through the "class".

package examples.rules.oop;

public class AUOFixed {

    public static void staticMethod () {}
    
    public void method () {}

    public static void main (String[] args) {
        AUOFixed object = new AUOFixed ();
        AUOFixed.staticMethod ();  // FIXED
        object.method ();
    }
}
