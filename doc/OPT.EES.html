Use 'String.length() == 0' instead of 'String.equals("")' [OPT.EES]



DESCRIPTION 


This rule checks that the programmer uses 'String.length() == 0' instead of
'String.equals("")'.  Although the two expressions perform the same check,
'String.length() == 0' is preferred because it may be more efficient and removes 
a String constant from the class file.  A violation is flagged for each place
where the programmer uses 'String.equals("")'.
SCOPE LEVEL 
LINE
SINCE 

v8.0
BENEFITS 

Mobile platforms have reduced resources, optimizations are especially important.

Using 'String.length() == 0' instead of 'String.equals("")' may be more 
efficient and removes the String constant "" from the class file.
EXAMPLE 

package examples.rules.opt;

public class EES {
    void method(String s){
        System.err.println("is empty String: "+s.equals(""));
    }
}
REPAIR 

Use 'String.length() == 0' instead of 'String.equals("")'.

package examples.rules.opt;

public class EESFixed {
    void method(String s){
        System.err.println("is empty String: "+(s.length() == 0));
    }
}
