Ensure that "REST" is disabled in 'axis2.xml' configuration files [APSC_DV.000170.REST]



DESCRIPTION 


This rule checks that "REST" is disabled in Apache Axis 2 'axis2.xml' 
configuration files.  A violation will be flagged for each 'axis2.xml' file 
which sets the "disableREST" parameter to "false".
SINCE 

v8.4
SECURITY RELEVANCE 

category: Unsafe Environment Configuration

"REST" does not have any message security standards, so it is dangerous to use 
"REST".  Some people prefer to always have "REST" disabled.
EXAMPLE 

axis2.xml:

<axisconfig name="AxisJava2.0">
    ...
    <!-- Following parameter will completely disable REST handling in Axis2-->
    <parameter name="disableREST" locked="false">false</parameter>
    ...
</axisconfig>
REPAIR 

Disable "REST" in the 'axis2.xml' file.

axis2.xml:

<axisconfig name="AxisJava2.0">
    ...
    <!-- Following parameter will completely disable REST handling in Axis2-->
    <parameter name="disableREST" locked="false">true</parameter>  <!-- FIXED -->
    ...
</axisconfig>
