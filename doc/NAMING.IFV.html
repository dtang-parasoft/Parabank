Use all uppercase letters for the names of fields in an "interface" [NAMING.IFV]



DESCRIPTION 


This rule identifies "interface" fields whose names are not in all
uppercase letters. An error is reported for each occurrence.
SCOPE LEVEL 

CLASS
SINCE 

v4.5
BENEFITS 

Fields in an "interface" are always "static" and "final".  Therefore, they
should follow the same naming convention as named constants.
EXAMPLE 

package examples.rules.naming;

public interface IFV {
    int max = 1000;  // VIOLATION
}
REPAIR 

Make the field name all uppercase.

package examples.rules.naming;

public interface IFVFixed {
    int MAX = 1000;  // FIXED
}
