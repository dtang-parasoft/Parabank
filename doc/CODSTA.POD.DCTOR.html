Define a no argument constructor whenever possible [CODSTA.POD.DCTOR]



DESCRIPTION 


This rule identifies classes that do not define a no argument constructor
whenever possible. An error is reported if a class lacks a 
constructor that takes no arguments.
SCOPE LEVEL 

CLASS
SINCE 

v4.5
NOTES 

By default, Jtest will ignore utility classes and 
classes which are "abstract" or "static".
BENEFITS 

Constructor that takes no argument allows classes of unknown types to be dynamically 
loaded and instantiated at runtime (for example, when loading unknown 
Applets from html pages). In Java 1.1 and later, reflection somewhat 
alleviates the need for no-argument constructors, but many classes that 
dynamically instantiate other classes at runtime still depend on their 
presence.
EXAMPLE 

package examples.rules.codsta.pod;

public class DCTOR {  // VIOLATION: no default constructor defined
    public DCTOR (int size) {
        _size = size;
    }
    private int _size;
}
REPAIR 

Define a default constructor whenever a class might be dynamically loaded by
code that does not know the type of the class.

package examples.rules.codsta.pod;

public class DCTORFixed {  // FIXED
    public DCTORFixed () {
        _size = 0;
    }
    public DCTORFixed (int size) {
        _size = size;
    }
    private int _size;
}
