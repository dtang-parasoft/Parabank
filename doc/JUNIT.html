JUnit Test Case [JUNIT] 

Rules that enforce common coding standards for JUnit test classes.
RULES

Prefer assertEquals and assertNotEquals over assertTrue and assertFalse [JUNIT.AEAT]
Do not hard code the location to data used by a unit test [JUNIT.AHLOD]
Prefer assertNull and assertNotNull over assertTrue and assertFalse [JUNIT.ANAT]
Avoid negating boolean parameters to assertTrue() or assertFalse() [JUNIT.ANBA]
Prefer assertSame and assertNotSame over assertTrue and assertFalse [JUNIT.ASAT]
Include a message string in JUnit assertions [JUNIT.ASSERT]
Do not include assertion statements in threads other than the main thread [JUNIT.AST]
Avoid using loops in JUnit tests [JUNIT.AUL]
Avoid calling 'assert' methods in catch blocks [JUNIT.CBA]
Call the superclass' 'setUp' and 'tearDown' methods in the 'setUp' and 'tearDown' methods of JUnit test classes [JUNIT.CSUPER]
Ensure that 'setUp()' and 'tearDown()' methods are implemented correctly [JUNIT.CSUTD]
Place each test class in the same location as the class that it tests [JUNIT.DIR]
Avoid extending 'TestCase' in test classes which contain the '@Test' annotation [JUNIT.ETCTA]
Use the fail() method instead of forcing a failed condition using the assertTrue(false) or assertFalse(true) method [JUNIT.FAIL]
Do not invoke the 'fail()' method in a 'catch' block in a JUnit test method [JUNIT.FICB]
Ensure that JUnit 5 test classes that use @Ignore are annotated with @ExtendWith(IgnoreCondition.class) or @EnableJUnit4MigrationSupport [JUNIT.IGNORE]
Do not implement the 'suite()' method in JUnit test classes [JUNIT.ISMTC]
Ensure that JUnit classes have a main() allowing them to be executed in isolation [JUNIT.MAIN]
Do not use the constructor to set up test cases [JUNIT.OSIC]
Ensure that a setup method is defined in the test class [JUNIT.OSUM]
Ensure that a cleanup method is defined in the test class [JUNIT.OTDM]
Do not use JUnit 4 annotations when migrating tests to JUnit 5 [JUNIT.RPL5]
Ensure JUnit test cases include assertion methods [JUNIT.SIA]
Set all fields which are initialized in the 'setUp()' method to "null" in the 'tearDown()' method [JUNIT.SIFN]
Make 'suite()' methods "public" and "static" [JUNIT.SUITE]
Use '@Test' annotation instead of extending 'TestCase' [JUNIT.TATC]
Include at least one test method in each 'TestCase' class [JUNIT.TCWNT]
Do not use the TemporaryFolder Rule in JUnit 5 tests [JUNIT.TDIR]
Make sure all methods have at least one JUnit test method [JUNIT.TEST]
Include an appropriate Javadoc tag in the Javadoc for JUnit test methods [JUNIT.UPJT]
Use the correct signature for the 'suite()' method in JUnit test classes [JUNIT.UPSS]
