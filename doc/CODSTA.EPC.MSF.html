Do not use too many non-"final" "static" fields [CODSTA.EPC.MSF]



DESCRIPTION 


This rule identifies classes that use more than 2(by default) non-"final" "static" fields.
An error is reported for each occurrence.
SCOPE LEVEL 
CLASS
SINCE 

v4.5
PARAMETERS 

-Maximum non-"final" "static" fields (default value is 2)
-Ignore the following types and their subtypes: (Specify fully qualified names)
    "org.eclipse.osgi.util.NLS" (default)
BENEFITS 

Static fields are the equivalent of global variables in non-OO languages.  They
make methods more context-dependent, hide possible side-effects, sometimes
present synchronized access problems, and cause fragile,
non-extensible constructions.  Also, neither static variables nor methods can
be overridden in any useful sense in subclasses.
EXAMPLE 

package examples.rules.codsta.epc;

public class MSF {
    private static int s_size;    
    public static int s_count;
    static String s_title;  // VIOLATION

    static void setFields (int size, String title) {
        s_size = size;    
        s_title = title;
        s_count++;
    }     
}
REPAIR 

Try to minimize non-"final" "static" fields if possible.
