Use a naming convention for "abstract" classes [NAMING.NACL]



DESCRIPTION 


This rule identifies abstract classes that do not follow the user-defined
naming convention (as specified in a regular expression). By default, the
rule requires that the class name either starts with 'Abstract' or ends with
'Factory'. An error is reported for each occurrence.
SCOPE LEVEL 
CLASS
SINCE 

v6.0
PARAMETERS 

-regular expression
    -Abstract class names should match this expression.
    -The default is "^Abstract.*$|^.*Factory$".  This enforces that the names of
     "abstract" classes either begin with "Abstract" or end with "Factory".
-check public class. (default is true)
-check protected class. (default is true)
-check package-private class. (default is true)
-check private class. (default is true)
-Negate regular expression
    -If this parameter is enabled, a violation will be flagged if the name of an
     "abstract" class matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for "abstract" class names which do not match the regular 
     expression specified.
-Ignore case
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of "abstract"
     classes must match the regular expression specified, including in case.
BENEFITS 

This helps enforce a naming convention.
EXAMPLE 

package examples.rules.naming;

public abstract class NACL {  //VIOLATION
}
REPAIR 

package examples.rules.naming;

public abstract class AbstractNACLFixed {  //FIXED
}
