Declare "public/protected" methods as inaccessible as possible [VVSG.2_4_A.DPPM]



DESCRIPTION 


This rule identifies "public" and "protected" methods that are not as
inaccessible as possible. An error is reported for each occurrence.

Global analysis is required in this rule.
SCOPE LEVEL 
METHOD
SINCE 

v4.5
SECURITY RELEVANCE 

category: Weak Security Controls
category: Code Quality

Avoid granting more access than is required.
PARAMETERS 

-Ignore "protected" methods which can be made package-private
    -Default is "false"
    -If this parameter is enabled, the rule will only flag a violation on
     "protected" methods if they can be made "private".
-Check methods in anonymous classes (Default value is "false")
-Check methods in local classes (Default value is "false")
BENEFITS 

Following this rule makes the code more object-oriented and makes its
dependencies easier to understand.
REPAIR 

Change the method's accessibility or document the reason for the
excessive accessibility.
