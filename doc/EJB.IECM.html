Implement one or more 'ejbCreate()' methods in bean classes [EJB.IECM]



DESCRIPTION 


This rule identifies any "SessionBean" or "EntityBean" class that does not 
implement one or more 'ejbCreate()' methods. An error is reported for each 
occurrence.
SCOPE LEVEL 

CLASS
SINCE 

v4.5
BENEFITS 

For session beans, it is required that they implement one or more 'ejbCreate()'
methods by the Enterprise JavaBeans specification (version 1.1 and greater).
For entity beans, it is not required in the specification, but it is still
recommended.  The 'ejbCreate()' method is the standard way a bean object is 
created by the container, so each entity bean should define at least one 
'ejbCreate()' method.
EXAMPLE 

package examples.rules.ejb;

import javax.ejb.*;

public class IECM implements SessionBean {  //VIOLATION- There is no ejbCreate() 
                                            //method defined
    private String _id;
    
    //Stubs for SessionBean methods
    public void setSessionContext(SessionContext ctx) { }
    public void ejbRemove() { }
    public void ejbActivate() { }
    public void ejbPassivate() { }
}
REPAIR 

package examples.rules.ejb;

import javax.ejb.*;

public class IECMFixed implements SessionBean {
    private String _id;
    
    public void ejbCreate(String id) {  //FIXED
        _id= id;
    }
    
    //Stubs for SessionBean methods
    public void setSessionContext(SessionContext ctx) { }
    public void ejbRemove() { }
    public void ejbActivate() { }
    public void ejbPassivate() { }
}
