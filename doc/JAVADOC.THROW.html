Use the '@throws' or '@exception' Javadoc tag in methods [JAVADOC.THROW]



DESCRIPTION 


This rule identifies any method Javadoc comment whose number of 
'@throws' tags does not equal the number of exceptions in the "throws" list.
An error is reported for each occurrence.
SCOPE LEVEL 
METHOD
SINCE 

v4.5
PARAMETERS 

-check "public" methods (default as true)
-check "protected" methods (default as true)
-check "package-private" methods (default as true)
-check "private" methods (default as false)
-Use this tag for overriding and implementing methods: (default is '@see')
    -If an overriding or implementing method contains this tag, this method will 
     not be checked.
BENEFITS 

Commenting your code is a good way to document its behavior.
EXAMPLE 

package examples.rules.javadoc;

import java.io.*;

public class THROW { 
    /**
     * method()
     */
    public void method () throws IOException {  // VIOLATION
        System.in.read ();
    }
}
REPAIR 

Use the '@throws' Javadoc tag for each exception in the "throws" list.

package examples.rules.javadoc;

import java.io.*;

public class THROWFixed {
    /**
     * @throws IOException
     */
    public void method () throws IOException {  // FIXED
        System.in.read ();
    }
}
