Ensure that classes have a "public" constructor with zero arguments [XML.IVCC]



DESCRIPTION 


This rule identifies qualified names in XML elements that are used for class 
initialization but cannot be initialized. It violate for following cases:

    - It is an "interface"
    - It is an "abstract" class
    - It is not a "public" class
    - No "public" constructor with zero arguments in the class
    - Inner class is not a "public static" class
    - Class doesn't exist (see 'Ignore not existed classes' option)

Non-source classes are ignored unless some class from source code
extends or implement this class.
SCOPE LEVEL 
LINE
SINCE 

v8.3
PARAMETERS 

-XML elements which are used to store the class names
    - Rule will only check files that on the left-hand side of the parameter
      and their corresponding XML element paths on the right-hand side of the parameter.
    
- Ignore not existed classes
    - Default value is true. 
    When this option is disabled, this rule returns violations when classes don't exist 
    This option may cause false-positive results for non-source classes.
BENEFITS 

An unexpected runtime exception will be thrown if the class is not defined 
correctly (e.g., It is a non-"public" class, or there is no "public" constructor 
with zero arguments). This rule can detect these kind of problems.
EXAMPLE 

package examples.rules.xml;

import javax.servlet.*;
import javax.servlet.http.*;

public class IVCC extends HttpServlet {
    
    protected IVCC() {  // VIOLATION : no "public" constructor
    }
}

---- web.xml --------
<web-app>
   <servlet>
       <servlet-name>IVCCServlet</servlet-name>
       <servlet-class>examples.rules.xml.IVCC</servlet-class>
   </servlet>
</web-app>
REPAIR 

package examples.rules.xml;

import javax.servlet.*;
import javax.servlet.http.*;

public class IVCCFixed extends HttpServlet {
    
    public IVCCFixed() {  // FIXED
    }
}

---- web.xml --------
<web-app>
   <servlet>
       <servlet-name>IVCCServlet</servlet-name>
       <servlet-class>examples.rules.xml.IVCCFixed</servlet-class>
   </servlet>
</web-app>
