A5-Security Misconfiguration [OWASP2021.A5]

RULES

Do not call the 'printStackTrace()' method of "Throwable" objects [OWASP2021.A5.ACPST]
Avoid "try", "catch" and "finally" blocks with empty bodies [OWASP2021.A5.AECB]
Do not disable CSRF protection [OWASP2021.A5.DCSRFJAVA]
Do not disable CSRF protection [OWASP2021.A5.DCSRFXML]
Avoid using the DriverManagerDataSource class in production code [OWASP2021.A5.DMDS]
Disable XML external entity injection [OWASP2021.A5.DXXE]
Avoid debug information from Spring Security framework to logs [OWASP2021.A5.EWSSEC]
Restrict cross-origin resource sharing to secure origins [OWASP2021.A5.JXCORS]
Do not catch exception types which are too general or are unchecked exceptions [OWASP2021.A5.NCE]
Do not catch 'NullPointerException' [OWASP2021.A5.NCNPE]
Do not throw exception types which are too general or are unchecked exceptions [OWASP2021.A5.NTERR]
Avoid declaring methods to throw general or unchecked Exception types [OWASP2021.A5.NTX]
Password information should not be included in properties file in plaintext [OWASP2021.A5.PLAIN]
Do not declare entity beans as remote [OWASP2021.A5.RR]
Always specify error pages in web.xml [OWASP2021.A5.SEP]
Avoid calling print methods of 'System.err' or 'System.out' [OWASP2021.A5.SIO]
Ensure Session-ID Length is sufficient [OWASP2021.A5.SLID]
Avoid storing unencrypted Hibernate usernames and passwords in 'web.xml' files [OWASP2021.A5.UPWD]
