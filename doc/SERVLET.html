Servlets [SERVLET] 


These rules describe development best practice for Web applications
containing servlets, JavaServer files, and JDBC connections.
RULES

Do not use JDBC code in Servlet classes [SERVLET.AJDBC]
Do not use 'java.beans.Beans.instantiate()' [SERVLET.BINS]
Catch all exceptions which may be thrown within Servlet methods [SERVLET.CETS]
Avoid multiple '<servlet>' elements with the same '<servlet-name>' in a 'web.xml' file [SERVLET.DSN]
Do not define instance fields in Servlet classes [SERVLET.IF]
Avoid using collections without size limit in servlets [SERVLET.LML]
Declare a "public" constructor that takes no parameters [SERVLET.MDC]
Follow a limit for the number of mappings for each servlet in a 'web.xml' file [SERVLET.NMS]
Do not create static variables in a servlet [SERVLET.NSIS]
Do not use static variables in servlets without synchronization [SERVLET.NSSS]
Ensure that all servlets have a corresponding mapping in 'web.xml' files [SERVLET.SCM]
Specify an appropriate schema or DTD file for 'web.xml' files [SERVLET.SDTD]
Minimize usage of System.out and System.err in Servlets [SERVLET.SOP]
Ensure that all '<servlet>' elements contain a single non-empty '<servlet-name>' element in 'web.xml' files [SERVLET.SSN]
Do not use 'SingleThreadModel' in Servlet classes [SERVLET.STM]
Minimize synchronization in Servlets [SERVLET.SYN]
Use a Context Object to manage HTTP request parameters [SERVLET.UCO]
