Avoid misusing arguments in property values [PROPS.IARG]



DESCRIPTION 


This rule will identify the following potential mistakes in using arguments:

   1. Any argument that cannot be formatted appropriately in 'java.text.MessageFormat.format()', 
      for examples:
           i) "Class name: '{0}'"
           ii) "Can't find class : {0}"
   2. Any argument index that is skipped
      for examples:
          MessageFormat.format("{1}-{2}-{3}", new Object[]{"1", "2", "3"}); // expected result is "1-2-3", but actual result is "2-3-{3}"
   3. Any unmatched brace
SCOPE LEVEL 
LINE
SINCE 

v8.1
PARAMETERS 

-Ignore property file name (regular expression)
    -Allows you to provide a list of property files to ignore. If a property file 
     matches any of the regular expressions, it will be exempt from this rule.
BENEFITS 

This rule helps users detect all possible mistakes in using arguments.
EXAMPLE 

a='{0}'  // VIOLATION : argument {0} cannot be formatted appropriately
c={1}, {3} // VIOLATION : skipped {0} and {2}
b={{0}  // VIOLATION : unmatched brace
REPAIR 

a=''{0}''  // FIXED
c={0}, {1}, {2}, {3} // FIXED
b={0}  // FIXED
