Use the '@param' Javadoc tag for each parameter of methods [JAVADOC.PARAM]



DESCRIPTION 


This rule identifies any method that has a Javadoc comment but
does not have an '@param' tag for each parameter. An 
error is reported for each occurrence.
SCOPE LEVEL 
METHOD
SINCE 

v4.5
NOTES 

Jtest will not report a violation of this rule if a method does not have a Javadoc
comment.  Another rule (JAVADOC.MJDC) will report an error 
if there is no Javadoc comment.
PARAMETERS 

-check "public" methods (default as true)
-check "protected" methods (default as true)
-check "package-private" methods (default as true)
-check "private" methods (default as false)
-Use this tag for overriding and implementing methods: (default is '@see')
    -If an overriding or implementing method contains this tag, it does not have
     to have separate '@param' tags for each of the parameters.
BENEFITS 

Helps users of your methods understand what data you expect in each parameter.
DRAWBACKS 

Some parameters are obvious enough that they do not benefit from comments. There 
might not be anything to add to the '@param' text other than what is obvious from
the names of the parameter and the method.  The 'fixed' example below is an 
example of this problem.
EXAMPLE 

package examples.rules.javadoc;

public class PARAM {

    /**
     */  // VIOLATION
    public void setId (String id) {
        _id = id;
    }
    private String _id;
}
REPAIR 

Provide a '@param' tag for each "public" method's parameters.

package examples.rules.javadoc;

public class PARAMFixed {

    /** 
     * @param id is set as the new id.
     */  // FIXED
    public void setId (String id) {
        _id = id;
    }
    private String _id;
}
