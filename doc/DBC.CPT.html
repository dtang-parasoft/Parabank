Do not include a postcondition saying that "$result!=null" in methods which can return null [DBC.CPT]



DESCRIPTION 


This rule checks that, if there is a postcondition in a method saying that 
"$result != null", there is no "return null;" statement in the method's body.
SCOPE LEVEL 
LINE
SINCE 

v8.0
BENEFITS 

It is incorrect to have a postcondition stating "$result != null" if there is
a "return null;" statement in its body.  Fixing this may avoid a lot of 
confusion.
EXAMPLE 

package examples.rules.dbc;

public class CPT {
    /**
    * @post $result != null
    **/
    public CPT method() {
        return null;
    }
}
REPAIR 

Remove the misleading postcondition, or change it so that it is correct.

package examples.rules.dbc;

public class CPTFixed {
    /**
    * @post $result == null
    **/
    public CPTFixed method() {
        return null;
    }
}
