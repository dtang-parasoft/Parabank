Unsafe Environment Configuration [SECURITY.UEC]

RULES

Ensure that 'axis.development.system' is set to "false" in Axis 'server-config.wsdd' files [SECURITY.UEC.ADS]
Ensure that 'axis.enableListQuery' is set to "false" in Axis 'server-config.wsdd' files [SECURITY.UEC.AELQ]
Ensure that 'axis.disableServiceList' is set to "true" in Axis 'server-config.wsdd' files [SECURITY.UEC.DSL]
Avoid defining multiple security roles with the same name in 'web.xml' files [SECURITY.UEC.DSR]
Ensure that the 'Encrypt' directive is specified for each 'items' tag in Axis2 configuration files [SECURITY.UEC.EDAR]
Ensure that each filter mapped in a 'web.xml' file has a corresponding definition [SECURITY.UEC.FMCD]
Use 'https' instead of 'http' for the 'transportReceiver' and 'transportSender' in 'axis2.xml' configuration files [SECURITY.UEC.HTTPS]
Ensure that 'InflowSecurity' and 'OutflowSecurity' parameters are specified in Axis2 configuration files [SECURITY.UEC.ISOS]
Include an appropriate '<login-config>' element to specify the type of authentication to be performed in 'web.xml' files [SECURITY.UEC.LCA]
Avoid storing usernames and passwords in plain text in Castor 'jdo-conf.xml' files [SECURITY.UEC.PCCF]
Avoid using plain text passwords in Axis 'wsdd' files [SECURITY.UEC.PTPT]
Ensure that passwords are not stored as plaintext and are sufficiently long [SECURITY.UEC.PWDPROP]
Ensure that passwords are not stored as plaintext and are sufficiently long [SECURITY.UEC.PWDXML]
Restrict access to JSPs in 'web.xml' files by including a security constraint for '*.jsp' files [SECURITY.UEC.RAJ]
Ensure that "REST" is disabled in 'axis2.xml' configuration files [SECURITY.UEC.REST]
Ensure that the 'Signature' directive is specified for each 'items' tag in Axis2 configuration files [SECURITY.UEC.SDAR]
Always specify error pages in web.xml [SECURITY.UEC.SEP]
Ensure Session-ID Length is sufficient [SECURITY.UEC.SLID]
Avoid using the SOAP Monitor module [SECURITY.UEC.SMM]
Ensure that each security role referenced in a 'web.xml' file has a corresponding definition [SECURITY.UEC.SRCD]
Ensure that sessions are configured to time out in 'web.xml' files [SECURITY.UEC.STTL]
Ensure that the 'Timestamp' directive is specified for each 'items' tag in Axis2 configuration files [SECURITY.UEC.TDAR]
Ensure that all constrained resources are protected with a '<user-data-constraint>' element in 'web.xml' files [SECURITY.UEC.UDC]
Avoid using plain text passwords in Axis2 configuration files [SECURITY.UEC.UTAX]
Ensure SOAP messages are encrypted in WebSphere 'ibm-webservicesclient-ext.xmi' files [SECURITY.UEC.WCMC]
Ensure SOAP messages are digitally signed in WebSphere 'ibm-webservicesclient-ext.xmi' files [SECURITY.UEC.WCMI]
Avoid misconfiguring timestamps in WebSphere 'ibm-webservicesclient-ext.xmi' files [SECURITY.UEC.WCMT]
Ensure WS-Security is enabled in WebSphere 'ibm-webservicesclient-ext.xmi' files [SECURITY.UEC.WCMWS]
Avoid unencrypted passwords in WebSphere 'ibm-webservicesclient-ext.xmi' files [SECURITY.UEC.WCPWD]
Avoid unsigned timestamps in WebSphere 'ibm-webservicesclient-ext.xmi' files [SECURITY.UEC.WCUTS]
Ensure all web content directories have a "welcome file" [SECURITY.UEC.WELC]
Ensure SOAP messages are encrypted in WebSphere 'ibm-webservices-ext.xmi' files [SECURITY.UEC.WMC]
Ensure SOAP messages are digitally signed in WebSphere 'ibm-webservices-ext.xmi' files [SECURITY.UEC.WMI]
Avoid misconfiguring timestamps in WebSphere 'ibm-webservices-ext.xmi' files [SECURITY.UEC.WMT]
Ensure WS-Security is enabled in WebSphere 'ibm-webservices-ext.xmi' files [SECURITY.UEC.WMWS]
Avoid unencrypted passwords in WebSphere 'ibm-webservices-ext.xmi' files [SECURITY.UEC.WPWD]
Ensure that the Rampart WS-Security module is enabled in Axis2 configuration files [SECURITY.UEC.WSS]
Avoid unsigned timestamps in WebSphere 'ibm-webservices-ext.xmi' files [SECURITY.UEC.WUTS]
