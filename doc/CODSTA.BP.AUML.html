Avoid using multiple loggers, use logging levels instead [CODSTA.BP.AUML]



DESCRIPTION 


This rule flags classes with multiple loggers.
SCOPE LEVEL 
CLASS
SINCE 

v8.4
PARAMETERS 

-Logger Types to Check:
    -Default values are:
     java.util.logging.Logger
     org.apache.log4j.Logger
     
By default, this rule will only check the types "java.util.logging.Logger" and 
"org.apache.log4j.Logger".  These are two of the most commonly used loggers.  If
you use a logger other than these two, you should add the fully qualified name 
of that logger to the parameter list.
BENEFITS 

Similar functionality can be achieved by using logging levels, which is often
considered the preferred method. This can also aid in clarifying the code.
EXAMPLE 

package examples.rules.codsta.bp;

import java.util.logging.Logger;

public class AUML { //VIOLATION
    private final static Logger _logger1 = Logger.getAnonymousLogger();
    private final static Logger _logger2 = Logger.getAnonymousLogger();
    private final static Logger _logger3 = Logger.getAnonymousLogger();
}
REPAIR 

Remove additional loggers, and implement logging levels.

package examples.rules.codsta.bp;

import java.util.logging.Logger;

public class AUMLFixed { //FIXED
    private final static Logger _logger1 = Logger.getAnonymousLogger();
}
