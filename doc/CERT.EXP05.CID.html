Avoid using increment or decrement operators in nested expressions [CERT.EXP05.CID]



DESCRIPTION 


This rule identifies any increment(++) or decrement(--) operator
that is embedded in other expressions. An error is reported
for each occurrence.
SINCE 

v6.0
NOTES 

An increment or decrement operator will not be flagged as
an error if it is placed within a "for" loop condition, "do"
loop condition, "do-while" loop condition, or a super call in a
constructor.
BENEFITS 

Compounding increment or decrement operators into method
calls or mathematical operations could be difficult to  read
and understand.
EXAMPLE 

package examples.rules.codsta.read.cid;

public class CID {
    int _seed= 0;
    public double getNum() {
        return Math.random()* ++_seed; //VIOLATION
    }
}
REPAIR 

package examples.rules.codsta.read;

public class CID {
    int _seed= 0;
    public double getNum() {
        ++_seed;
        return Math.random()* _seed; //FIXED
    }
}
