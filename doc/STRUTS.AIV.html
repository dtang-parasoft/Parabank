Specify an @input attribute if '<action>' element has validation in 'struts-config.xml' files [STRUTS.AIV]



DESCRIPTION 


This rule checks 'struts-config.xml' files. A violation is flagged for each
'<action>' element with a @validate attribute set to "true" and no @input
attribute.
SINCE 

v8.4
SECURITY RELEVANCE 

category: Unsafe Environment Configuration

An @input attribute is required for '<action>' elements if the @validate
attribute is set to "true". Failure to specify an @input attribute may be a sign
of inconsistent and erroneous code. Furthermore, validation will be unable to
occur and errors may be displayed.
EXAMPLE 

The '<action>' element in the sample 'struts-config.xml' file has a @validate
attribute but does not specify an @input attribute.

struts-config.xml:

<?xml version="1.0" encoding="ISO-8859-1" ?>
<struts-config>
    <form-beans>
        <form-bean name="logonForm" type="app.LogonForm"/>
    </form-beans>
    <action-mappings>
        <!-- VIOLATION: Missing @input attribute  -->
        <action
            path="/LogonSubmit"
            type="app.LogonAction"
            name="logonForm"
            scope="request"
            validate="true"
        </action>
    </action-mappings>
</struts-config>
REPAIR 

Specify a valid value for the @input attribute or do not use the @validate
attribute.

struts-config.xml:
    
<?xml version="1.0" encoding="ISO-8859-1" ?>
<struts-config>
    <form-beans>
        <form-bean name="logonForm" type="app.LogonForm"/>
    </form-beans>
    <action-mappings>
        <!-- FIXED -->
        <action
            path="/LogonSubmit"
            type="app.LogonAction"
            name="logonForm"
            scope="request"
            validate="true"
            input="/pages/Logon.jsp">
            <forward name="success" path="/pages/Welcome.jsp"/>
            <forward name="failure" path="/pages/Logon.jsp"/>
        </action>
    </action-mappings>
</struts-config>
