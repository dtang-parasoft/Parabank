Exposing Sensitive Data [SECURITY.ESD]

RULES

Avoid writing to Consoles [SECURITY.ESD.ACW]
Do not log confidential or sensitive information [SECURITY.ESD.CONSEN]
Clear sensitive data after use [SECURITY.ESD.CSD]
Do not pass exception messages into output in order to prevent the application from leaking sensitive information [SECURITY.ESD.PEO]
Avoid storing sensitive data in plaintext in a cookie [SECURITY.ESD.PLC]
Avoid methods that might expose internal representations by returning arrays or other mutable fields [SECURITY.ESD.RA]
Store sensitive data in mutable objects [SECURITY.ESD.SDM]
Inspect instance fields of serializable objects to make sure they will not expose sensitive information [SECURITY.ESD.SIF]
Avoid calling print methods of 'System.err' or 'System.out' [SECURITY.ESD.SIO]
Do not expose data with a 'FileNotFound' exception [SECURITY.ESD.SNFD]
Do not interrogate or modify security policy information in a web component [SECURITY.ESD.SPI]
Declare "transient" fields "private" [SECURITY.ESD.TFP]
Avoid "transient" fields in serialPersistentFields array [SECURITY.ESD.TSPF]
Use secure communication with 'wss' (WebSocket Secure) instead of 'ws' [SECURITY.ESD.WS]
