Improper signature of the annotated method in EJB bean [EJB.EJB3.CMF]



DESCRIPTION 


This rule flags a violation if an annotated callback method has improper
formatting. The following callback annotations are checked: "@PostActivate",
"@PostConstruct", "@PreDestroy", and "@PrePassivate". These methods are checked
for Stateless, Stateful, and MessageDriven beans. A callback method satisfying
these conditions should have "void" return type and no parameters.
SINCE 

v8.4
EXAMPLE 

package examples.rules.ejb.ejb3;

@Stateful
public class CMF {
    @PostActivate
    @PostConstruct
    String startMethod() { //VIOLATION
        return "success";
    }

    @PreDestroy
    @PrePassivate
    void stopMethod(Object o) { //VIOLATION

    }
}
REPAIR 

If the wrong method is used, place the annotation on the correct method.
Otherwise, ensure the method does not return a value and has no parameters.

package examples.rules.ejb.ejb3;

@Stateful
public class CMF {
    @PostActivate
    @PostConstruct
    void startMethod() { //FIXED

    }

    @PreDestroy
    @PrePassivate
    void stopMethod() { //FIXED

    }
}
