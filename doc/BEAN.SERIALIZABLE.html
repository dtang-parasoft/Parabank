Ensure that JavaBean classes implement 'java.io.Serializable' [BEAN.SERIALIZABLE]



DESCRIPTION 



This rule identifies JavaBean classes that do not implement 'java.io.Serializable'.
All JavaBeans are required to be persistable. An error is reported for each
occurrence.
SCOPE LEVEL 

CLASS
SINCE 

v5.0
SECURITY RELEVANCE 

category: Erratic Application Behavior
category: Data Security

When implementing the Serializable interface, fields that contain confidential
or sensitive data (such as passwords) should be marked as transient or should use
a customized, encrypted serialization scheme. Otherwise, a potential attacker
could extract the confidential data from the serialization stream or from a
serialized object file.
BENEFITS 

Objects that support the Serializable interface can save and restore their 
state from disk. This is also a requirement for JavaBeans.
EXAMPLE 

package examples.rules.bean;

import java.io.Serializable;
import java.awt.Color;

public class SERIALIZABLEBean {  // VIOLATION

    private Color color = Color.green;

    //getter method
    public Color getColor() {
	return color;
    }

    //setter method
    public void setColor(Color newColor) {
	color = newColor;
	repaint();
    }
}
REPAIR 

Make the Bean class implement Serializable.

package examples.rules.bean;

import java.io.Serializable;
import java.awt.Color;

public class SERIALIZABLEFixedBean implements Serializable {  // FIXED

    private Color color = Color.green;

    //getter method
    public Color getColor() {
	return color;
    }

    //setter method
    public void setColor(Color newColor) {
	color = newColor;
	repaint();
    }
}
