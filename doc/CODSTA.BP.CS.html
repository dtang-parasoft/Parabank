Place constants on the appropriate side of comparisons [CODSTA.BP.CS]



DESCRIPTION 


This rule identifies constants that are not on the appropriate side of comparisons
(as defined in the rule parameters). With the default parameters, an error is
reported if a constant is not on the right-hand side of a comparison expression.
SCOPE LEVEL 
LINE
SINCE 

v6.0
PARAMETERS 

-Placement of constants:
   -Left-hand side 
   -Right-hand side (default option)
   
-Allow the constant "null" on either side of a comparison.
   -The default is off.  If this box is checked, the rule will ignore the 
    constant "null".
BENEFITS 

It is a good practice to place constants on the same side of
comparison expressions. 

Moreover, there is an added benefit to placing constants on the left-hand side 
of a comparison expression. A common typo while writing code is to use "=" 
instead of "==" in equality comparisons. If you have placed the constant on the 
left-hand side of the expression, the compiler will report an error message for 
this typo. For example, the javac compiler will report an error message for:

    if (false = var)

but not for:

    if (var = false)
EXAMPLE 

package examples.rules.codsta.bp;

public class CS {
    public void testMethod (int something) {
        if (something == 5) {}  // VIOLATION (as default setting)
    }
}
REPAIR 

package examples.rules.codsta.bp;

public class CSFixed {
    public void testMethod (int something) {
        if (5 == something) {}  // FIXED
    }
}
