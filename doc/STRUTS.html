Struts Framework [STRUTS] 

Rules related to the Struts framework.
RULES

Struts 2 [STRUTS.STRUTS2]
Avoid accessing a database from Action Classes [STRUTS.ACDA]
Avoid accessing a JSP page from Action Classes [STRUTS.ACJC]
Specify an @input attribute if '<action>' element has validation in 'struts-config.xml' files [STRUTS.AIV]
Specify a '<form-bean>' for each named '<action>' element in 'struts-config.xml' files [STRUTS.AMFB]
Avoid @prefix, @suffix, and @attribute for unnamed '<action>' elements in 'struts-config.xml' [STRUTS.APSN]
Avoid using relative paths for attributes in 'struts-config.xml' files [STRUTS.ARP]
Avoid duplicated forms in the 'validation.xml' [STRUTS.DFV]
Avoid local and global forwards with the same name in 'struts-config.xml' files [STRUTS.DLGF]
Ensure validators are enabled in the 'struts-config.xml' [STRUTS.EV]
Provide an appropriate getter and setter method for each field in a form bean [STRUTS.FIELDS]
Include only getter and setter methods in form beans [STRUTS.FORM]
Specify a non-empty name and path for each '<forward>' element in 'struts-config.xml' files [STRUTS.FWD]
Avoid calling methods of Action Classes [STRUTS.IACM]
Do not declare instance variables in Struts Action classes [STRUTS.INST]
Avoid defining multiple form beans with the same name in 'struts-config.xml' files [STRUTS.MFBSN]
Specify a valid 'maxlength' value for each form field in Struts validation files [STRUTS.MLFF]
Use the 'minlength' validator for password fields in 'validation.xml' [STRUTS.MLVP]
Specify a name and type for each form bean in 'struts-config.xml' files [STRUTS.NTFB]
Ensure that the @path attribute of '<action>' and '<forward>' elements begins with '/' in 'struts-config.xml' files [STRUTS.PFS]
Ensure Plugins are added in the 'struts-config.xml' [STRUTS.PLUGIN]
Avoid public methods in Action Classes [STRUTS.PMAC]
Specify a non-empty path and type for each '<exception>' element in 'struts-config.xml' files [STRUTS.PTE]
Ensure that the 'scope' attribute is set to either "request" or "session" for actions and exceptions in 'struts-config.xml' files [STRUTS.RSS]
Use a Struts DTD for validation in 'struts-config.xml' files [STRUTS.SCDTD]
Specify a valid 'type' attribute for each '<form-property>' element in 'struts-config.xml' files [STRUTS.TAFP]
Enable the Struts Validator plug-in in all 'struts-config.xml' files [STRUTS.VPI]
