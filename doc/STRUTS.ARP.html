Avoid using relative paths for attributes in 'struts-config.xml' files [STRUTS.ARP]



DESCRIPTION 


This rule checks 'struts-config.xml' files. A violation is flagged for each
attribute that makes use of a relative path. Attributes are checked for slashes
such as "//" and "\\".
SINCE 

v8.4
SECURITY RELEVANCE 

category: Unsafe Environment Configuration

An attribute using a relative path may cause a file to be unable to load if
directories are reorganized. It may also be possible for an incorrect file to be
used if similar names are used.
EXAMPLE 

The '<action>' element in the sample 'struts-config.xml' file uses a relative
path

struts-config.xml:

<?xml version="1.0" encoding="ISO-8859-1" ?>
<struts-config>
    <form-beans>
        <form-bean name="logonForm" type="app.LogonForm"/>
    </form-beans>
    <action-mappings>
        <!-- VIOLATION: @path is relative  -->
        <action
            path="/LogonSubmit"
            type="app.LogonAction"
            name="logonForm"
            scope="request"
        </action>
    </action-mappings>
</struts-config>
REPAIR 

struts-config.xml:
    
<?xml version="1.0" encoding="ISO-8859-1" ?>
<struts-config>
    <form-beans>
        <form-bean name="logonForm" type="app.LogonForm"/>
    </form-beans>
    <action-mappings>
        <!-- FIXED -->
        <action
            path="/forms/LogonSubmit"
            type="app.LogonAction"
            name="logonForm"
            scope="request"
        </action>
    </action-mappings>
</struts-config>
