Use a naming convention for classes [NAMING.NCL]



DESCRIPTION 


This rule identifies classes that do not follow the user-defined
naming convention (as specified in a regular expression). By default, the
rule requires that the class name starts with a capital first letter. An 
error is reported for each occurrence.
SCOPE LEVEL 
CLASS
SINCE 

v4.5
PARAMETERS 

-Top-level class
    -regular expression of top-level class
     (Class names should match the listed regular expression.
      Default is not set.)
    -check public top-level class. (default is true)
    -check package-private top-level class. (default is true)

-Member class
    -regular expression of member class
     (Class names should match the listed regular expression.
      Default is not set.)
    -check public member class. (default is true)
    -check protected member class. (default is true)
    -check package-private member class. (default is true)
    -check private member class. (default is true)


-Local class (class inside method)
    -regular expression of local class
     (Class names should match the listed regular expression.
      Default is not set.)
    -check package-private local class. (default is true)

-Negate regular expression (Option is available on all tabs.)
    -If this parameter is enabled, a violation will be flagged if the name of a
     class matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for class names which do not match the regular expression 
     specified.
-Ignore case (Option is available on all tabs.)
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of classes 
     must match the regular expression specified, including in case.
BENEFITS 

This helps enforce a naming convention.
EXAMPLE 

package examples.rules.naming;

public class NCL {  

    class nclinner {  // VIOLATION

    }
}
REPAIR 

Change the identifier to follow the convention.

package examples.rules.naming;

public class NCLFixed {  

    class NCLInnerFixed {  // FIXED 

    }
}
