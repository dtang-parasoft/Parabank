Avoid duplicated forms in the 'validation.xml' [APSC_DV.001460.DFV]



DESCRIPTION 


This rule identifies if more than one validation form has the same name in a
'validation.xml' file. An error is reported for each occurrence.
SINCE 

v8.3
SECURITY RELEVANCE 

If more than one validation form has the same name, the Struts validator will 
arbitrarily choose one of the forms. This will cause a security problem because 
some inputs may not be validated due to the arbitrary form.
EXAMPLE 

<?xml version="1.0" ?>
<form-validation>
    <formset>
        <form name="Login">
            <field property="username" depends="required">
                <msg name="required" key="error.username"/>
            </field>
        </form>
        <!-- VIOLATION : duplicated form name -->
        <form name="Login">
            <field property="user" depends="required">
                <msg name="required" key="error.user"/>
            </field>
        </form>
    </formset>
</form-validation>
REPAIR 

<?xml version="1.0" ?>
<form-validation>
    <formset>
        <form name="Login">
            <field property="username" depends="required">
                <msg name="required" key="error.username"/>
            </field>
        </form>
        <!-- FIXED : remove duplicated form name -->
    </formset>
</form-validation>
