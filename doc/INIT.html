Initialization [INIT] 

Rules that enforce the explicit
initialization of the variables.
RULES

Do not use array initializers [INIT.AAI]
Avoid explicitly initializing fields which have already been initialized by a superclass' constructor [INIT.ADI]
Ensure that the "if" check for lazy initialization uses the correct operator [INIT.AULI]
Do not put code other than initialization code in lazy initialization blocks [INIT.CLIB]
Explicitly initialize all fields [INIT.CSI]
Avoid double initialization of fields [INIT.DI]
Use diamond to invoke the constructor of a generic class [INIT.DIA]
Do not use initialization circularities for fields [INIT.IC]
Initialize all local variables explicitly at the declaration statement [INIT.LV]
Do not use non-final "static" fields during the initialization [INIT.NFS]
Use explicit initializations/Do not initialize "static" fields to default values [INIT.SF]
Do not initialize "static" "final" variables with non "final" "static" variables [INIT.SFA]
Do not use a "static" initializer that creates an instance of the current class before all "static final" fields are assigned [INIT.SICUI]
Avoid uninitialized reads of fields before or during constructor execution [INIT.UIRC]
