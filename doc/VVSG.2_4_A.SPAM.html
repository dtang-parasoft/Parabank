Declare a package-private method "final" if it is not overridden [VVSG.2_4_A.SPAM]



DESCRIPTION 


This rule identifies package-private methods that are not overridden and that
are not declared "final". An error is reported for each occurrence.

Global analysis is required in this rule.
SCOPE LEVEL 
METHOD
SINCE 

v4.5
SECURITY RELEVANCE 

"Exposing critical functionality essentially provides an attacker with
the privilege level of the exposed functionality. This could result in
the modification or exposure of sensitive data or possibly even execution
of arbitrary code." [CWE-749]
PARAMETERS 

-Check methods in abstract classes (Default value is "false")
-Check methods in anonymous classes (Default value is "false")
-Check methods in local classes (Default value is "false")
-Check unused methods (Default value is "false")
-Check static methods (Default value is "false")
BENEFITS 

1) It optimizes the code: because the compiler knows that nobody is
   overriding the method, it can generate optimized code.

2) It makes the code self documenting: somebody looking at the
   method knows that no other method overrides it.
REPAIR 

Make the method "final" or document the reason for the
method not being final.
