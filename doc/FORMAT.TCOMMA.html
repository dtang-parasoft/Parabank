Avoid or enforce the use of trailing commas in array initializers [FORMAT.TCOMMA]



DESCRIPTION 


This parameterized rule identifies either 1) code that uses a trailing comma in the 
array initializer, or 2) code that does not use a trailing comma in the 
array initializer. An error is reported for each occurrence. 

Note that this rule is a switch logic rule.
SCOPE LEVEL 
LINE
SINCE 

v6.0
PARAMETERS 

-Enforce/Avoid trailing comma
The default parameter is set as avoid trailing comma.
BENEFITS 

The usage of the trailing comma makes it easier to switch array 
elements. However, on the other hand, those trailing commas are meaningless
to program semantics and could be removed. Whether you decide to enforce or
avoid trailing comma usage is a coding preference; either way, it is unlikely
to affect the program's correctness.
EXAMPLE 

package examples.rules.format;

public class TCOMMA {
    int[] a1= new int[] { 1, 3, 5, 7 } //VIOLATION when enforcing trailing comma
    int[] a2= new int[] { 2, 4, 6, 8, } //VIOLATION when avoiding trailing comma                  
}
REPAIR 

package examples.rules.format;

public class TCOMMAFixed {
    int[] a1= new int[] { 1, 3, 5, 7, } //FIXED when enforcing trailing comma
    int[] a2= new int[] { 2, 4, 6, 8 } //FIXED when avoiding trailing comma                  
}
