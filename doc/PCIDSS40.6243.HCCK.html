Avoid using hard-coded cryptographic keys [PCIDSS40.6243.HCCK]



DESCRIPTION 


This rule identifies hard-coded cryptographic keys. A violation is reported for
each occurrence.
SINCE 

v9.0
NOTES 

This rule only checks cryptographic keys composed of hexadecimal values.
SECURITY RELEVANCE 

category: Weak Security Controls

If a hard-coded cryptographic key is used the application becomes more
vulnerable to brute force attacks.

Enforcing this rule will help to protect against some of the OWASP Top 10 
application vulnerabilities including "A6-Sensitive Data Exposure".
PARAMETERS 

-Check keys whose length is greater than (default 9)
BENEFITS 

See SECURITY RELEVANCE.
EXAMPLE 

package examples.rules.security.wsc;

public class HCCK {
    private static final String _KEY = "295e7a2379624b6e2b3f28222f"; //VIOLATION

    public boolean isValid(String checkValid) {
        return _KEY.equals(checkValid);
    }
}
REPAIR 

Remove the hard-coded cryptographic key and consider an alternative.
