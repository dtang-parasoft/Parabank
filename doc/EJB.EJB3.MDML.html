Ensure @MessageDriven beans specify a MessageListener [EJB.EJB3.MDML]



DESCRIPTION 


This rule identifies MessageDriven beans which do not specify a message
listener.
The following locations are checked:
    1) @MessageDriven annotation member "messageListenerInterface"
    2) "implements" keyword of MessageDriven bean class
    3) "messaging-type" in the deployment descriptor
SINCE 

v8.4
PARAMETERS 

Check the following:
    -Check annotations
    -Check annotations and deployment descriptors (default)
        Check deployment descriptors in the following location:
            -Check project (default)
            -Check package

Specify message listeners using qualified names:
    -"javax.jms.MessageListener" (default)
    Message listeners specified using this parameter are used to verify
    those implemented by the "implements" keyword or specified in the
    annotation.
EXAMPLE 

package examples.rules.ejb.ejb3;

@MessageDriven
public class MDML { //VIOLATION
    ...
}
REPAIR 

Specify a message listener for the MessageDriven bean.

package examples.rules.ejb.ejb3;

@MessageDriven(
    messageListenerInterface="javax.jms.MessageListener")
public class MDMLFixed { //FIXED
    ...
}

Debu Panda, Reza Rahman, and Derek Lane. EJB3 in Action.
