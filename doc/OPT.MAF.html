Make getter and setter methods for instance fields "final" [OPT.MAF]



DESCRIPTION 


This rule identifies getter and setter methods for instance fields that are not "final".
An error is reported for each occurrence.
SCOPE LEVEL 

METHOD
SINCE 

v4.5
NOTES 

If you are using Hotspot VM and Sun's Java 2 SDK1.3, this rule does not improve 
performance.

You should be careful when using the quickfix for this rule because the quickfix may 
cause compile errors.  The quickfix is to declare the getter or setter method "final".  
While this is OK in most cases, it will cause a compile error if the getter or setter
method is overridden.  Since getter and setter methods are generally not overridden, 
this case should be rare and may indicate a design flaw in the code.  Because the 
quickfix may cause compile errors, the "Quickfix All" option is not available for this 
rule.  You should review each violation and make sure that applying the quickfix will 
not cause a compile error.
BENEFITS 

Mobile platforms have reduced resources, optimizations are especially important.

Making getter and setter methods for instance fields "final" tells the compiler that 
the method cannot be overridden and thus can be inlined.
EXAMPLE 

package examples.rules.opt;

public class MAF {
    public void setSize (int size) {  // VIOLATION
         _size = size;
    }
    private int _size;
}
REPAIR 

Make the accessor "final".

package examples.rules.opt;

public class MAFFixed {
    final public void setSize (int size) {  // FIXED
         _size = size;
    }
    private int _size;
}
