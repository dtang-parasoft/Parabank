Object Oriented Programming [OOP] 

Rules that enforce good Object
Oriented Programming practices.
RULES

Do not extend concrete classes in "abstract" classes [OOP.ACECC]
Avoid casts to types which are more specific than necessary [OOP.ACST]
Avoid "public"/"protected"/package-private fields [OOP.AF]
Do not hide inherited fields [OOP.AHF]
Do not hide inherited "static" member methods [OOP.AHSM]
Add the "@Override" annotation to overriding methods [OOP.AOA]
Always override certain methods when extending certain types [OOP.AOCM]
Redeclare a class with only "abstract" methods and "static final" fields as an "interface" [OOP.ASFI]
Do not implement or extend annotations [OOP.AUASI]
Do not use an object to access "static" fields or methods [OOP.AUO]
Do not declare a method in an interface which conflicts with a 'protected' method of 'Object' [OOP.CIMOM]
Ensure methods are either a command(change state) or a query(get state) [OOP.CQS]
Do not hide fields and local variables declared in enclosing scopes [OOP.HIF]
Do not give method local variables and parameters the same name as class fields [OOP.HMF]
Avoid inner classes in interfaces [OOP.ICIF]
Avoid declaring methods inherited from class "Object" in an interface [OOP.IDOM]
Avoid implementing interfaces only to access constants [OOP.IIAC]
Implement interface methods non-trivially or declare them "abstract" [OOP.IIN]
Use "instanceof" only on interfaces [OOP.INSOF]
Avoid more than two levels of nested inner classes [OOP.LEVEL]
Give "finalize()" methods "protected" access [OOP.MFP]
Avoid interfaces with no fields or methods (marker interfaces) [OOP.MI]
Provide mutable classes with copy functionality [OOP.MUCOP]
Avoid "abstract" classes without "abstract" methods [OOP.NAM]
Declare methods in "abstract" classes with empty bodies "abstract" [OOP.NOMA]
Do not declare "public" or package-private constructors in "abstract" classes [OOP.NPAC]
Do not override an instance "private" method [OOP.OPM]
Do not use "protected" access for members of "final" classes [OOP.PIFC]
Ensure 'public' classes have at least one 'public' or 'protected' member [OOP.PMPC]
Do not declare "private" or "static" methods as "final" [OOP.PSDF]
Do not declare a class as implementing an interface if a superclass already implements that interface [OOP.RI]
Avoid referencing a class' subclasses from itself [OOP.RSFC]
Use a naming convention for singleton classes [OOP.SNGL]
Specify which version of potentially ambiguous methods you wish to call for method calls in inner classes [OOP.SVHM]
Use getContextClassLoader() instead of getClassLoader() [OOP.THRECL]
