A2-Cryptographic Failures [OWASP2021.A2]

RULES

Avoid using insecure cryptographic algorithms for data encryption with Spring [OWASP2021.A2.AISSAJAVA]
Avoid using insecure cryptographic algorithms in Spring XML configurations [OWASP2021.A2.AISSAXML]
Avoid using the javax.crypto.NullCipher class in non-test classes [OWASP2021.A2.AUNC]
Do not log confidential or sensitive information [OWASP2021.A2.CONSEN]
Ensure arguments passed to certain methods come from predefined methods list [OWASP2021.A2.ENPP]
Avoid using hard-coded cryptographic keys [OWASP2021.A2.HCCK]
Do not hard-code IP addresses and port numbers [OWASP2021.A2.HCNA]
Use 'https' instead of 'http' for the 'transportReceiver' and 'transportSender' in 'axis2.xml' configuration files [OWASP2021.A2.HTTPS]
Ensure the HostnameVerifier.verify() method validates the certificate [OWASP2021.A2.HV]
Avoid using insecure algorithms for cryptography [OWASP2021.A2.ICA]
Avoid non-random "byte[]" when using IvParameterSpec [OWASP2021.A2.IVR]
MessageDigest objects must process the data with the 'update' method [OWASP2021.A2.MCMDU]
Use hash functions with a salt [OWASP2021.A2.MDSALT]
Do not pass exception messages into output in order to prevent the application from leaking sensitive information [OWASP2021.A2.PEO]
Prevent exposure of sensitive data [OWASP2021.A2.SENS]
Initialize KeyGenerator instances [OWASP2021.A2.SIKG]
Use 'java.security.SecureRandom' instead of 'java.util.Random' or 'Math.random()' [OWASP2021.A2.SRD]
Safely serialize sensitive data [OWASP2021.A2.SSSD]
Use the SSL-enabled version of classes when possible [OWASP2021.A2.USC]
Properly validate server identity [OWASP2021.A2.VSI]
