Ensure that XML files are well-formatted [XML.WF]



DESCRIPTION 


This rule identifies the following illegal XML format problems:
   - Byte-order mark(BOM) character in the XML prolog
   - Content in the XML prolog
   - Missing end tag of XML elements
   - Missing quotes in attribute
   - Missing end quote in attribute
   - Unknown attribute
   
An error is reported for each occurrence.

XML prolog consists of two parts: the XML declaration and the Document Type 
Declaration (DTD). The DTD is an optional component of the XML document which 
appears before the root element.
SCOPE LEVEL 
LINE
SINCE 

v8.3
PARAMETERS 

-Check xHTML files (default false) 
    -When enabled rule is triggering errors on xHTML files
BENEFITS 

Illegal XML formats will cause SAXParseExceptions when parsing the XML file. 
Fixing all the illegal XML formats can avoid this problem.
EXAMPLE 
<?xml version="1.0" ?>
<xml>
  <body>
    <rules>
        <!-- VIOLATION : missing end tag -->
        <rule id="XML.WF">
    </rules>
  </body>
</xml>
REPAIR 
<?xml version="1.0" ?>
<xml>
  <body>
    <rules>
        <!-- FIXED : add a "/" to close the tag -->
        <rule id="XML.WF" />
    </rules>
  </body>
</xml>
REFERENCES 

Extensible Markup Language (XML) 1.1 (Second Edition)
