A3-Sensitive Data Exposure [OWASP2017.A3]

RULES

Avoid using insecure cryptographic algorithms for data encryption with Spring [OWASP2017.A3.AISSAJAVA]
Avoid using insecure cryptographic algorithms in Spring XML configurations [OWASP2017.A3.AISSAXML]
Avoid using the javax.crypto.NullCipher class in non-test classes [OWASP2017.A3.AUNC]
Do not log confidential or sensitive information [OWASP2017.A3.CONSEN]
Ensure arguments passed to certain methods come from predefined methods list [OWASP2017.A3.ENPP]
Avoid using hard-coded cryptographic keys [OWASP2017.A3.HCCK]
Do not hard-code IP addresses and port numbers [OWASP2017.A3.HCNA]
Use 'https' instead of 'http' for the 'transportReceiver' and 'transportSender' in 'axis2.xml' configuration files [OWASP2017.A3.HTTPS]
Ensure the HostnameVerifier.verify() method validates the certificate [OWASP2017.A3.HV]
Avoid using insecure algorithms for cryptography [OWASP2017.A3.ICA]
Avoid non-random "byte[]" when using IvParameterSpec [OWASP2017.A3.IVR]
MessageDigest objects must process the data with the 'update' method [OWASP2017.A3.MCMDU]
Use hash functions with a salt [OWASP2017.A3.MDSALT]
Do not pass exception messages into output in order to prevent the application from leaking sensitive information [OWASP2017.A3.PEO]
Ensure that passwords are not stored as plaintext and are sufficiently long [OWASP2017.A3.PWDPROP]
Prevent exposure of sensitive data [OWASP2017.A3.SENS]
Initialize KeyGenerator instances [OWASP2017.A3.SIKG]
Use 'java.security.SecureRandom' instead of 'java.util.Random' or 'Math.random()' [OWASP2017.A3.SRD]
Safely serialize sensitive data [OWASP2017.A3.SSSD]
Use the SSL-enabled version of classes when possible [OWASP2017.A3.USC]
Always verify JarFile signatures [OWASP2017.A3.VJFS]
Properly validate server identity [OWASP2017.A3.VSI]
