Do not use variables of the unsafe type 'java.lang.ThreadGroup' [VVSG.2_5_B.AUTG]



DESCRIPTION 



This rule identifies the use of 'java.lang.ThreadGroup' variables. An error
is reported for each occurrence.
SINCE 

v4.5
BENEFITS 

ThreadGroup has safety flaws and should be avoided because it is unnecessary.
The 'activeCount ()' method, for example, is not thread safe, yet by definition,
it will be used in a threaded environment.
REPAIR 

Store 'Thread' references in an array or collection.
REFERENCES 

Joshua Bloch: "Effective Java - Programming Language Guide"
Addison Wesley, 2001, pp. 211

SEI CERT Oracle Coding Standard for Java
THI01-J. Do not invoke ThreadGroup methods
