Avoid constant interface anti-pattern [CODSTA.POD.ACIAP]



DESCRIPTION 


This rule identifies the constant interface anti-pattern.
SINCE 

v8.4
NOTES 

This rule is similar to CODSTA.POD.ISACF, which also flags interfaces containing
constants. However, CODSTA.POD.ACIAP specifically checks for interfaces with 
constants and no methods or inner types. CODSTA.POD.ACIAP is less strict than
CODSTA.POD.ISACF because CODSTA.POD.ACIAP allows some constants to be defined in 
an interface as long as there is at least one method or type in the interface. 
They were made separate rules in case the user has a preference for one or the 
other.
BENEFITS 

Use of interfaces for constants can lead to confusion, as they may be visible in
a class' public API or visible in the Javadoc.
EXAMPLE 

package examples.rules.codsta.pod;

interface ACIAP {
    Integer _zero = new Integer(0);
    Integer _one = new Integer(1);
}
REPAIR 

Add additional functionality to the class, such as methods, or use a class with
a private constructor to prevent instantiation of the class.

package examples.rules.codsta.pod;

public class ACIAPFixed {
    private ACIAPFixed() {}
    
    public static final Integer _zero = new Integer(0);
    public static final Integer _one = new Integer(1);
}
