Error-Prone Coding [CODSTA.EPC]

RULES

Do not make assignments to method parameters [CODSTA.EPC.AFP]
Avoid conversions from parameterized types to raw types [CODSTA.EPC.AGBPT]
Do not use constructors in the 'clone()' method [CODSTA.EPC.CLNC]
Avoid using the conditional operator with mismatched numeric types [CODSTA.EPC.COMT]
Use "int" instead of "byte" or "short" and "double" instead of "float" for variable declarations [CODSTA.EPC.IBS]
Do not use too many non-"final" "static" fields [CODSTA.EPC.MSF]
Do not call an "abstract" method from a constructor in an "abstract" class [CODSTA.EPC.NCAC]
Do not perform bitwise and arithmetic operations on the same data [CODSTA.EPC.NCBAV]
Do not catch exception types which are too general or are unchecked exceptions [CODSTA.EPC.NCE]
Do not call methods that might cause unexpected NullPointerExceptions during constructor execution [CODSTA.EPC.NCNFC]
Use overloading judiciously [CODSTA.EPC.OVERLOAD]
Call 'super.clone()' in all 'clone()' methods [CODSTA.EPC.SCLONE]
Avoid using the same generic type variable for multiple method arguments [CODSTA.EPC.STA]
Use 'StringTokenizer' instead of 'indexOf()' and 'substring()' for String parsing [CODSTA.EPC.UST]
Do not write to static fields from non-static methods [CODSTA.EPC.WSIM]
