Ensure parameters and return type of remote business methods are Serializable [EJB.EJB3.SRBM]



DESCRIPTION 


This rule checks methods declared in remote business interfaces. A violation is
flagged for each method that has a non-Serializable return type or parameters.
As remote business methods may be invoked via RMI, the parameters and return
type must be Serializable.
SINCE 

v8.4
NOTES 

This rule considers an interface a remote business interface if it is has the
"@Remote" annotation.
EXAMPLE 

package examples.rules.ejb.ejb3;

@Remote
public interface SRBM {
    int getInt();
    void setInt(int i);
    Object getName(); //VIOLATION
    void setName(Object o); //VIOLATION
    ...
}
REPAIR 

If possible, reimplement the method using Serializable types.

package examples.rules.ejb.ejb3;

@Remote
public interface SRBMFixed {
    int getInt();
    void setInt(int i);
    String getName(); //FIXED
    void setName(String o); //FIXED
    ...
}
REFERENCES 

Debu Panda, Reza Rahman, and Derek Lane. EJB3 in Action.
Greenwich: Manning, 2007. pp. 78, 88.
