Avoid ignored or invalid annotations [EJB.EJB3.AIA]



DESCRIPTION 


This rule flags a violation if an annotation is used which is either ignored or
invalid in the context in which it is used.
SINCE 

v8.4
NOTES 

This rule checks the following situations:
    1) @PrePassivate used with Stateless or MessageDriven bean
           -Only Stateful beans are passivated

    2) @PostActivate used with Stateless or MessageDriven bean
           -Only Stateful beans are passivated

    3) @WebService used with Stateful bean
           -SOAP-based services are stateless in nature

    4) @TransactionAttribute used with TransactionManagementType.BEAN
           -@TransactionAttribute is only for container managed transactions

    5) @TransactionAttribute used without TransactionAttributeType.REQUIRED or
       TransactionAttributeType.NOT_SUPPORTED for MessageDriven bean
           -Clients do not invoke MessageDriven beans directly, so there are no
           transactions to suspend or join.
EXAMPLE 

package examples.rules.ejb.ejb3;

@Stateless
public class AIA {
    @PrePassivate //VIOLATION
    @PostActivate //VIOLATION
    public void update() {
        ...
    }
    ...
}
REPAIR 

Remove the annotation or replace with the intended one.

package examples.rules.ejb.ejb3;

@Stateless
public class AIAFixed {
    public void update() { //FIXED
        ...
    }
    ...
}
