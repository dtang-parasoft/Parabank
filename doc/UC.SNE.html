Avoid empty "synchronized" statements [UC.SNE]



DESCRIPTION 


This rule identifies "synchronized" statements with empty bodies.
An error is reported for each occurrence.
SINCE 

v6.0
SECURITY RELEVANCE 

"An empty synchronized block will wait until nobody else is using
the synchronizer being specified. While this may be part of the desired
behavior, because you haven't protected the subsequent code by placing it
inside the synchronized block, nothing is stopping somebody else from
modifying whatever it was you were waiting for while you run the subsequent
code." [CWE-585]
BENEFITS 

"synchronized" statements are supposed to enclose statements 
that need to be synchronized. Consequently, empty "synchronized"
statements are usually unnecessary and should be removed.
EXAMPLE 

package examples.rules.uc;

public class SNE
{
    public void myMethod () {
        //...
        synchronized (this) {
        //VIOLATION
        }
    }
}
REPAIR 

package examples.rules.uc;

public class SNEFixed
{
    public void myMethod () {
    //...
        synchronized (this) {
            //...
            System.out.println("Synchronizing"); //FIXED
	}
    }
}
