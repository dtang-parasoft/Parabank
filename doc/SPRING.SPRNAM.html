Use naming conventions for spring beans [SPRING.SPRNAM]



DESCRIPTION 


This rule checks whether the bean id attribute value is the same as the bean class name--
with the first letter changed from capital to small.  
A violation is reported for each bean in a Spring xml configuration file
that does not follow this naming convention for ids.
SINCE 

v9.2
NOTES 

This rule ignores beans that have multiple names or do not have a name or id.
BENEFITS 
Following this rule makes code easier to understand and maintain.
EXAMPLE 

<?xml version="1.0" encoding="UTF-8"?>

<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd"

    <bean id="personSetterByName"	<!-- VIOLATION -->
        class="org.springbyexample.springindepth.chapter03.autowire.PersonContainer" />
</beans>
REPAIR 

<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd"

    <bean id="personContainer"	<!-- FIXED -->
        class="org.springbyexample.springindepth.chapter03.autowire.PersonContainer" />
</beans>
