Order Javadoc tags appropriately [OWASP2019.API9.ORDER]



DESCRIPTION 


This rule identifies classes whose Javadoc tags are not ordered according to
Sun's recommendation. By default, an error is reported if Javadoc tags are not in the 
following order:

    1.  @author
    2.  @version
    3.  @concurrency
    4.  @param
    5.  @return
    6.  @exception
    7.  @throws
    8.  @see
    9.  @since
    10. @serial
    11. @serialdata
    12. @serialfield
    13. @deprecated
    14. other tags (e.g DbC tags : @pre, @post, @invariant)
SCOPE LEVEL 
LINE
SINCE 

v5.0
PARAMETERS 

- a Javadoc tag table which user can change the javadoc tag order and add
  their own tags.
  
To give several tags the same order priority, add them as a new comma separated list to 
the rule parameters.

For example, to allow @author and @version to be used in any order before the rest of 
the tags:
    1. View/Change Rule parameters
    2. Uncheck @author and @version
    3. Add a new element '@author, @version'
    4. Move '@author, @version' to the top.
BENEFITS 

If everyone follows the same Javadoc tag ordering, it will help others to
read the code.
EXAMPLE 

package examples.rules.javadoc;

import java.sql.*;

// VIOLATION
/**
 * this is an example class
 *
 * @deprecated
 * @version 1.0
 * @author jtest
 */
public class ORDER {

    // VIOLATION
    /**
     * this method will return a Database Connection
     *
     * @pre url != null
     * @pre db != null
     * @post $result != null
     * @param url URL
     * @param db database name
     * @exception SQLException
     * @exception Exception
     * @return Connection
     */
    public Connection getConnection(String url, String db) 
        throws SQLException, Exception
    {
    Connection con = null;
    // create db connection
    return con;
    }
}
REPAIR 

Re-order the Javadoc tags to follow the convention.

package examples.rules.javadoc;

import java.sql.*;

// OK 
/**
 * this is an example class
 *
 * @author jtest
 * @version 1.0
 * @deprecated
 */
public class ORDERFixed {

    // OK 
    /**
     * this method will return a Database Connection
     *
     * @param url URL
     * @param db database name
     * @return Connection
     * @exception SQLException
     * @exception Exception
     * @pre url != null
     * @pre db != null
     * @post $result != null
     */
    public Connection getConnection(String url, String db) 
        throws SQLException, Exception
    {
    Connection con = null;
    // create db connection
    return con;
    }
}
