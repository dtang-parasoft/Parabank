Do not declare a method in an interface which conflicts with a 'protected' method of 'Object' [OOP.CIMOM]



DESCRIPTION 


This rule will flag a violation for any method declared in an interface which 
conflicts with one of the 'protected' methods of the class 'java.lang.Object'
(either 'clone()' or 'finalize()').  A violation will be flagged for each 
'clone()' method defined in an interface which does not have the return type
'Object' and for every 'finalize()' method defined in an interface which does
not have 'void' for its return type.
SINCE 

v8.0
BENEFITS 

If there is a conflict between a method declared in an interface and a method
of 'Object', the interface cannot be implemented.  Thus, these conflicts should
be resolved.
EXAMPLE 

package examples.rules.oop;

public interface CIMOM {
    int clone();  //VIOLATION
}
REPAIR 

Remove the conflicting method declaration, or change the return type so that it
matches the return type of the method in 'java.lang.Object'.

package examples.rules.oop;

public interface CIMOMFixed {
    Object clone();  //FIXED
}
