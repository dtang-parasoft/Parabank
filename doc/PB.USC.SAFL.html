Avoid self assignments/initializations to fields and/or local variables [PB.USC.SAFL]



DESCRIPTION 


This rule identifies self assignments/initializations to fields and/or local 
variables. An error is reported if a field and/or local variable is assigned
to itself.
SCOPE LEVEL 
LINE
SINCE 

v6.0
PARAMETERS 

You can choose to enable or disable the option of checking:
-fields
-local variables and parameters

Both options are enabled by default.
BENEFITS 

An assignment with no effect should be removed since it is meaningless
to the code. However, such a self-assignment could also be an 
indication of a misspelled parameter.
EXAMPLE 

package examples.rules.pb.usc;

public class SAFL {
    int num;
    public SAFL(int mum) { //misspelled parameter
        this.num= num; //VIOLATION
    }
}
REPAIR 

package examples.rules.pb.usc;

public class SAFL_Fixed {
    int num;
    public SAFL(int num) { //FIXED
       this.num= num; //FIXED
    }
}
