Use the '@return' Javadoc tag in method Javadoc comments [JAVADOC.MRDC]



DESCRIPTION 


This rule identifies non-"void" methods that lack a '@return' tag
An error is reported for each occurrence.
SCOPE LEVEL 
METHOD
SINCE 

v4.5
PARAMETERS 

-check "public" methods (default as true)
-check "protected" methods (default as true)
-check "package-private" methods (default as true)
-check "private" methods (default as false)
-Use this tag for overriding and implementing methods: (default is '@see')
    -If an overriding or implementing method contains this tag, it does not have
     to have an '@return' tag.
BENEFITS 

Commenting your code is a good way to document its behavior.
EXAMPLE 

package examples.rules.javadoc;

public class MRDC {
    /**
     * getStr method
     */
    public String getStr () {  // VIOLATION
        return _str; 
    }
    private String _str;
}
REPAIR 

Provide the '@return' tag in the Javadoc comments.

package examples.rules.javadoc;

public class MRDCFixed {
    /**
     * @return string  // FIXED
     */
    public String getStr () {
        return _str;
    }
    private String _str;
}
