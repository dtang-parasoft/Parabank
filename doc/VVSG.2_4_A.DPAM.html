Declare package-private methods as inaccessible as possible [VVSG.2_4_A.DPAM]



DESCRIPTION 


This rule identifies package-private methods that are not as inaccessible as
possible. An error is reported for each occurrence.

Global analysis is required in this rule.
SCOPE LEVEL 
METHOD
SINCE 

v4.5
SECURITY RELEVANCE 

"Exposing critical functionality essentially provides an attacker with
the privilege level of the exposed functionality. This could result in
the modification or exposure of sensitive data or possibly even execution
of arbitrary code." [CWE-749]
PARAMETERS 

-Check methods in anonymous classes (Default value is "false")
-Check methods in local classes (Default value is "false")
BENEFITS 

Following this rule makes the code more object-oriented and makes its
dependencies easier to understand.
REPAIR 

Change the method's accessibility or document the reason for the
excessive accessibility.
