Avoid loop variables which are not used in the condition of the loop [PB.LOGIC.AULV]



DESCRIPTION 


This rule identifies unused loop variables. An error is
reported if a loop variable is declared but unused in
the loop condition.
SCOPE LEVEL 
LINE
SINCE 

v6.0
PARAMETERS 

-Do not report any violations for loops containing a "break" 
 or "return" statement.  (Default is "true")
BENEFITS 

Variables declared in the initialization part of the for
loop are supposed to be the loop variables. This only
makes sense if they are actually used in the condition part
of the for loop. An unused loop variable not only indicates
useless loop variables, but also might indicate that loop
variables were mistakenly mixed up within multi-level loops.
EXAMPLE 

package examples.rules.pb.logic;

public class AULV {
    public void myMethod() {
    for (int i= 0; i< 10; i++) {
            for (int unusedJ= 0; i< 10; unusedJ++) { //VIOLATION
                System.out.print(i* unusedJ+ " ");
            }
            System.out.println();
        }
    }
}
REPAIR 

package examples.rules.pb.logic;

public class AULVFixed {
    public void myMethod() {
    for (int i= 0; i< 10; i++) {
            for (int j= 0; j< 10; j++) { //FIXED
                System.out.print(i* j+ " ");
            }
            System.out.println();
        }
    }
}
