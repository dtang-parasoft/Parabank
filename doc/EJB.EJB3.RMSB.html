Specify a @Remove method for @Stateful beans [EJB.EJB3.RMSB]



DESCRIPTION 


This rule identifies "@Stateful" beans which do not specify a "@Remove" method.
SINCE 

v8.4
PARAMETERS 

Check the following:
    -Check annotations
    -Check annotations and deployment descriptors (default)
        Check deployment descriptors in the following location:
            -Check project (default)
            -Check package
BENEFITS 

Beans without a "@Remove" method have no way to inform the container that the
bean is no longer needed. This may cause the bean to remain until the session
ends or times out. Adding a "@Remove" method will prevent unused beans from
tying up resources.
EXAMPLE 

package examples.rules.ejb.ejb3;

@Stateful
public class RMSB implements RMSBInterface { //VIOLATION
    ...
}
REPAIR 

Ensure the bean has a "@Remove" method.

package examples.rules.ejb.ejb3;

@Stateful
public class RMSBFixed implements RMSBFixedInterface { //FIXED
    ...

    @Remove
    void removeMethod() {
        ...
    }
}
REFERENCES 

Debu Panda, Reza Rahman, and Derek Lane. EJB3 in Action.
Greenwich: Manning, 2007. pp. 101-102.
