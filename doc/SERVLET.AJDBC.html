Do not use JDBC code in Servlet classes [SERVLET.AJDBC]



DESCRIPTION 


This rule identifies Servlet classes that use JDBC code. An
error is reported for each occurrence.
SCOPE LEVEL 

CLASS
SINCE 

v5.1
BENEFITS 

Avoiding JDBC code in Servlet classes creates cleaner abstractions, increases
cohesion, and reduces coupling, thus improving modularity and reusability.
REFERENCES 

"Core J2EE Patterns - Best Practices and Design Strategies"
by Deepak Alur, John Cupri and Dan Malks. pp. 102
