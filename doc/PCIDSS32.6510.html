6.5.10 Broken authentication and session management [PCIDSS32.6510]

RULES

Avoid DNS lookups for decision making [PCIDSS32.6510.DNSL]
Avoid using the 'getRequestedSessionId' method from the 'HttpServletRequest' class [PCIDSS32.6510.HGRSI]
Always call 'HttpSession.invalidate()' before 'LoginContext.login()' [PCIDSS32.6510.ISL]
Avoid storing usernames and passwords in plain text in Castor 'jdo-conf.xml' files [PCIDSS32.6510.PCCF]
Avoid using plain text passwords in Axis 'wsdd' files [PCIDSS32.6510.PTPT]
Ensure that passwords are not stored as plaintext and are sufficiently long [PCIDSS32.6510.PWDXML]
Ensure proper session expiration [PCIDSS32.6510.RUIM]
Ensure that sessions are configured to time out in 'web.xml' files [PCIDSS32.6510.STTL]
Protect against using unprotected credentials [PCIDSS32.6510.TDPASSWD]
Use the ''getSecure()'' and ''setSecure()'' methods to enforce the use of secure cookies [PCIDSS32.6510.UOSC]
Avoid storing unencrypted Hibernate usernames and passwords in 'web.xml' files [PCIDSS32.6510.UPWD]
Avoid using plain text passwords in Axis2 configuration files [PCIDSS32.6510.UTAX]
Avoid unencrypted passwords in WebSphere 'ibm-webservicesclient-ext.xmi' files [PCIDSS32.6510.WCPWD]
Avoid unencrypted passwords in WebSphere 'ibm-webservices-ext.xmi' files [PCIDSS32.6510.WPWD]
