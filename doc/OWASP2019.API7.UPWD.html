Avoid storing unencrypted Hibernate usernames and passwords in 'web.xml' files [OWASP2019.API7.UPWD]



DESCRIPTION 


This rule checks for Hibernate usernames and passwords which are stored in plain
text in "web.xml" files.   Hibernate usernames and passwords should always be 
encrypted if they are to be stored in XML files.  Since there are many different 
kinds of encryption algorithms in the market, in this rule, we assume that an 
encrypted password is a combination of uppercase letters, lowercase letters, 
digits and symbols.  A violation will be flagged for each Hibernate username or
password which is stored in a "web.xml" file and which does not contain an 
uppercase letter, lowercase letter, digit or symbol.
SINCE 

v8.4
SECURITY RELEVANCE 

category: Unsafe Environment Configuration

If a user stores a plain text username or password in an XML Hibernate 
configuration file, the username or password can easily be stolen by anyone with 
access to that file.  This makes it very easy for hackers to steal the username
or password.

This rule helps to enforce the PCI DSS (Payment Card Industry Data Security 
Standard) Requirement #6: "Develop and maintain secure systems and 
applications".  Specifically, this rule helps to test for Issue 6.3.6: 
"Removal of custom application accounts, user IDs, and passwords before 
applications become active or are released to customers".

This rule also helps to enforce the PCI DSS Requirement #8: "Assign a unique ID 
to each person with computer access."  Specifically, this rule helps to test for 
Issue 8.4: "Render all passwords unreadable during transmission and storage on 
all system components using strong cryptography" and Issue 8.5.11: "Use 
passwords containing both numeric and alphabetic characters."

Enforcing this rule will help to protect against some of the OWASP Top 10 
application vulnerabilities including "A6-Sensitive Data Exposure".
EXAMPLE 

web.xml:

<hibernate-configuration>
    <session-factory>  
        <property name="hibernate.connection.username">User</property>  <!-- VIOLATION: Username not encrypted -->
        <property name="hibernate.connection.password">Pass</property>  <!-- VIOLATION: Password not encrypted -->
    </session-factory>
</hibernate-configuration>
REPAIR 

Use an encryption algorithm to encrypt the username and password.  The choice of
an encryption algorithm and how to encrypt and decrypt the username and password
is up to you.

web.xml:

<hibernate-configuration>
    <session-factory>  
        <property name="hibernate.connection.username">jI$^!nv&j`9n7c@U1i</property>  <!-- FIXED -->
        <property name="hibernate.connection.password">SnRlc3Q6bnVsbA==</property>  <!-- FIXED -->
    </session-factory>
</hibernate-configuration>
