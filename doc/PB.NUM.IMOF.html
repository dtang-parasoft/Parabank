Avoid casting the result of an integer multiplication to "long" [PB.NUM.IMOF]



DESCRIPTION 


This rule identifies that if an integer multiplication expression assigns to
a variable which type is "long" or returns as a "long" value. An error is 
reported for each occurrence.
SINCE 

v8.2
SECURITY RELEVANCE 

category: Erratic Application Behavior

Unexpected numeric results may cause unexpected behavior and create security
holes. See BENEFITS for more information.
BENEFITS 

Integer multiplication expression may cause overflow. If the integer expression 
value will assign to a "long" type variable eventually, we should convert the 
expression to "long" expression before doing multiplication in order to avoid
overflow problem.
EXAMPLE 

package examples.rules.pb.num;

public class IMOF {

    long toLong(int num) { 
        return 1000*2000*3000*num;  // VIOLATION
    } 
}
REPAIR 

Change one of the integer value to "long"

package examples.rules.pb.num;

public class IMOFFixed {

    long toLong(int num) { 
        return 1000L*2000*3000*num;  // FIXED
    } 
}
