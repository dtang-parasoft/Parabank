Avoid unmatched property entries in resources with different locales [PROPS.MENTRY]



DESCRIPTION 


This rule identifies any property name that is not matched in two resources with different locales.
This means that the property name is found in one resource, but not in the other.
An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v8.1
PARAMETERS 

-Ignore property file name (regular expression)
    -Allows you to provide a list of property files to ignore. If a property file 
     matches any of the regular expressions, it will be exempt from this rule.
BENEFITS 

It is a possible bug if property names in two resources with different locale are not matched.
EXAMPLE 

------------ message.properties ----------------
a=A
b=B
c=C

------------ messsage_ja.properties --------------
// VIOLATION : missing property name 'a'
b=B
c=C
d=D  // VIOLATION : extra property
REPAIR 

------------ message.properties ----------------
a=A
b=B
c=C

------------ messsage_ja.properties --------------
a=A  // FIXED
b=B
c=C
// FIXED : remove extra property 'd'
