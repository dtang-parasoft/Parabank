Avoid improper use of @Id annotation [EJB.EJB3.IDA]



DESCRIPTION 


This rule flags a violation if an @Id notation is used improperly. The following
cases are checked:
    1) If an @Id notation is used with a value of type double, Double, float, or
       Float. Some precision may be lost when storing a value into a database.
       It then becomes possible for two different @Id objects to resolve to the
       same database entry.
    2) If @EmbeddedId is used in conjunction with @Id.
SINCE 

v8.4
EXAMPLE 

package examples.rules.ejb.ejb3;

public class IDA {
    @Id
    protected double d; //VIOLATION
}
REPAIR 

Use another data type if possible, or create a separate value for the id.

package examples.rules.ejb.ejb3;

public class IDAFixed {
    @Id
    protected int i;
    protected double d; //FIXED
}
