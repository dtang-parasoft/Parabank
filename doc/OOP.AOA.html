Add the "@Override" annotation to overriding methods [OOP.AOA]



DESCRIPTION 


This rule will flag a violation for any method declaration which overrides or implements
a method but does not have an "@Override" annotation.  A violation will be flagged 
for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v8.0
BENEFITS 

Adding the "@Override" annotation makes it more obvious to the reader that a 
method declaration is an "overriding" or "implementing" method rather than a "new" method.
EXAMPLE 

package examples.rules.oop;

public class AOA {
    public String toString() { //VIOLATION
        return "";
    }
}
REPAIR 

Add an "@Override" annotation to the method declaration.

package examples.rules.oop;

public class AOAFixed {
    @Override 
    public String toString() { //FIXED
        return "";
    }
}
