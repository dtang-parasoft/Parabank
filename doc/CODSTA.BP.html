Bad Practice [CODSTA.BP]

RULES

Avoid "break" and/or "continue" with labels [CODSTA.BP.ABCL]
Avoid declaring methods that return 'Object' [CODSTA.BP.AMRO]
Avoid using reflection methods [CODSTA.BP.ARM]
Avoid returning "null" for arrays and certain types [CODSTA.BP.ARN]
Avoid label statements [CODSTA.BP.AULS]
Avoid using multiple loggers, use logging levels instead [CODSTA.BP.AUML]
Avoid using wildcards in method return types [CODSTA.BP.AWRT]
Provide a '{}' block for conditional statements [CODSTA.BP.BLK]
Avoid unnecessary nested blocks [CODSTA.BP.BLOCK]
Declare fields with uppercase character names as "final" [CODSTA.BP.CFNF]
Avoid 'public' or 'protected' constructors for immutable classes [CODSTA.BP.CMUTA]
Do not use "break" and/or "continue" statements [CODSTA.BP.CONTINUE]
Place constants on the appropriate side of comparisons [CODSTA.BP.CS]
Declare loggers as "static final" fields [CODSTA.BP.DLSF]
Do not call methods which terminates Java Virtual Machine [CODSTA.BP.EXIT]
Declare all formal parameters as "final" [CODSTA.BP.FPF]
Reference interface constants with their declaring interface names [CODSTA.BP.FQNIC]
Avoid using 'Hashtable' and 'Vector' [CODSTA.BP.HTV]
Avoid or enforce usage of '*' form of import statements [CODSTA.BP.IMPTD]
Avoid passing non-reifiable types to varargs methods [CODSTA.BP.NRVA]
Avoid declaring methods to throw general or unchecked Exception types [CODSTA.BP.NTX]
Ensure overloaded constructors and methods share the same accessibility [CODSTA.BP.OCMA]
Ensure that a class which has only "private" constructors is declared as "final" [CODSTA.BP.PCF]
Do not define "public" or "protected" members in anonymous classes [CODSTA.BP.PPAC]
