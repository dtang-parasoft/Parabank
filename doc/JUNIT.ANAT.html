Prefer assertNull and assertNotNull over assertTrue and assertFalse [JUNIT.ANAT]



DESCRIPTION 


It is recommended to use the assertNull() and assertNotNull() methods
rather than assertTrue() and assertFalse() to assert equality to "null" in
JUnit tests (see BENEFITS).

This rule detects when the assertTrue() and assertFalse() methods are
used with comparison to "null" in method parameters.
SINCE 

v9.0
BENEFITS 

This rule helps you improve test code readability and obtain error
messages that offer better insight into why a test failed.
EXAMPLE 

package examples.rules.junit;

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class ANATTest {
    @Test
    public void testMethod() {
        Object o = new Object();
        assertFalse(o == null); //VIOLATION
    }
}
REPAIR 

Use assertNotNull() instead of assertFalse().

package examples.rules.junit;

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class ANATFixedTest {
    @Test
    public void testMethod() {
        Object o = new Object();
        assertNotNull(o); //FIXED
    }
}
