A6-Security Misconfiguration [OWASP2017.A6]

RULES

Do not call the 'printStackTrace()' method of "Throwable" objects [OWASP2017.A6.ACPST]
Avoid "try", "catch" and "finally" blocks with empty bodies [OWASP2017.A6.AECB]
Do not disable CSRF protection [OWASP2017.A6.DCSRFJAVA]
Do not disable CSRF protection [OWASP2017.A6.DCSRFXML]
Avoid using the DriverManagerDataSource class in production code [OWASP2017.A6.DMDS]
Avoid debug information from Spring Security framework to logs [OWASP2017.A6.EWSSEC]
Restrict cross-origin resource sharing to secure origins [OWASP2017.A6.JXCORS]
Do not catch exception types which are too general or are unchecked exceptions [OWASP2017.A6.NCE]
Do not catch 'NullPointerException' [OWASP2017.A6.NCNPE]
Do not throw exception types which are too general or are unchecked exceptions [OWASP2017.A6.NTERR]
Avoid declaring methods to throw general or unchecked Exception types [OWASP2017.A6.NTX]
Password information should not be included in properties file in plaintext [OWASP2017.A6.PLAIN]
Ensure that passwords are not stored as plaintext and are sufficiently long [OWASP2017.A6.PWDXML]
Do not declare entity beans as remote [OWASP2017.A6.RR]
Always specify error pages in web.xml [OWASP2017.A6.SEP]
Avoid calling print methods of 'System.err' or 'System.out' [OWASP2017.A6.SIO]
Ensure Session-ID Length is sufficient [OWASP2017.A6.SLID]
Avoid storing unencrypted Hibernate usernames and passwords in 'web.xml' files [OWASP2017.A6.UPWD]
