Avoid out of range comparisons [VVSG.2_5_4_E.OOR]



DESCRIPTION 


This rule identifies comparisons that have one side as char/byte/short/int
variable and the other side as a constant. If the constant is not within the 
corresponding range of possible values, then the comparison is flagged.
SCOPE LEVEL 
LINE
SINCE 

v6.0
PARAMETERS 

-Check "char" comparisons (default true)
-Check "byte" comparisons (default false)
-Check "short" comparisons (default false)
-Check "int" comparisons (default false)
BENEFITS 

It does not make sense to compare a char/byte/short/int variable to a constant that 
is either less than or greater than the bounds of the type, as the comparison
would always return "true", or always return "false" regardless of the
variable's value.
EXAMPLE 

package examples.rules.pb.logic;

public class UIF {
    public static String getAsciiString(String inString) {
        StringBuffer outStringBuffer= new StringBuffer();
        for (int i= 0; i< inString.length(); i++) {
            char currentChar= inString.charAt(i);
            if (currentChar> -1 && currentChar< 256) { //VIOLATION, -1 is not a possible value of "char"
                outStringBuffer.append(currentChar);
            }
        }
        return outStringBuffer.toString();
    }
}
REPAIR 

package examples.rules.pb.logic;

public class UIFFixed {
    public static String getAsciiString(String inString) {
        StringBuffer outStringBuffer= new StringBuffer();
        for (int i= 0; i< inString.length(); i++) {
            char currentChar= inString.charAt(i);

                outStringBuffer.append(currentChar);
            }
        }

    }
