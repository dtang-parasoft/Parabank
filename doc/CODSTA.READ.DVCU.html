Declare variables as close as possible to where they are used [CODSTA.READ.DVCU]



DESCRIPTION 


Reports an error if the local variable declaration is not close to where the
variable is used.
SCOPE LEVEL 
LINE
SINCE 

v5.1
BENEFITS 

In large classes, it could be difficult to find the variable declaration. By
keeping a variable declaration close to its usage, readability can be improved.
EXAMPLE 

package examples.rules.codsta.read;

public class DVCU {

    public void sort2(int[] array) {

        int temp;  // VIOLATION
        if (array[0] > array[1]) {
            temp = array[1];
            array[1] = array[0];
            array[0] = temp;
        }
    }

}
REPAIR 

move the "temp" variable declaration inside "if" statement

package examples.rules.codsta.read;

public class DVCUFixed {

    public void sort2(int[] array) {

        if (array[0] > array[1]) {
            int temp;  // FIXED
            temp = array[1];
            array[1] = array[0];
            array[0] = temp;
        }
    }

}
