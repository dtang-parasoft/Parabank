Do not increment or decrement on the same variable over multiple nested "for" loop statements [PB.LOGIC.JI]



DESCRIPTION 


This rule identifies code where the same loop variable 
gets incremented or decremented over multiple nested loops. 
An error is reported for each occurrence.
SINCE 

v6.0
BENEFITS 

Loop variables that are incremented/decremented over multiple nested
for loops are most likely mistakenly placed incrementers, and 
they would result in unexpected looping behaviors. Even if they 
are intentional, they could still be confusing and potentially 
result in logical errors.
EXAMPLE 

package examples.rules.pb.logic;

public class JI {
    public void myMethod() {
        for (int i= 0; i< 10; i++) {
            for (int j= 9; j>= 0; i--) { //VIOLATION
                //INFINITE LOOP
            }
        }
    }
}
REPAIR 

package examples.rules.pb.logic;

public class JIFixed {
    public void myMethod() {
        for (int i= 0; i< 10; i++) {
            for (int j= 9; j>= 0; j--) { //FIXED
                //LOOPING 100 TIMES
            }
        }
    }
}
