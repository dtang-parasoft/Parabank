CWE-352: Cross-Site Request Forgery (CSRF) [CWE.352]

RULES

Do not disable CSRF protection [CWE.352.DCSRFJAVA]
Do not disable CSRF protection [CWE.352.DCSRFXML]
Encapsulate arguments of dangerous methods with a validation method [CWE.352.EACM]
Ensure that methods annotated with @RequestMapping specify the HTTP request method they call [CWE.352.REQMAP]
Protect against HTTP response splitting [CWE.352.TDRESP]
Protect against XSS vulnerabilities [CWE.352.TDXSS]
Use the ''getSecure()'' and ''setSecure()'' methods to enforce the use of secure cookies [CWE.352.UOSC]
Validate all dangerous data [CWE.352.VPPD]
