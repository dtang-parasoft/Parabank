Do not hide fields and local variables declared in enclosing scopes [OOP.HIF]



DESCRIPTION 


A field, local variable, or parameter that is defined in the nested scope
should not have the same name as a field, variable, or parameter defined
in the outer scope. Using identical names makes them easy to confuse and may 
prevent access to to the field, variable, or parameter defined in the outer
scope.  

This rule identifies declarations of fields, local variables, and parameters
that hide or shadow fields, variables, and parameters defined in the outer
scope.
SCOPE LEVEL 
LINE
SINCE 

v7.5
BENEFITS 

This rule helps you prevent confusion of fields, variables, and parameters 
from the nested and outer scopes, as well as ensure that variables in 
the outer scope can be accessed.
EXAMPLE 
In below example method setName() doesn't works as expected because Java overwrites parameter value instead of field 'name'

public class HIFExample {
    private String name = "test";

    public String getName() {
        return name;
    }

    public void setName(String name) { // VIOLATION
        name = name;
    }
}
REPAIR 

public class HIFExample {
    private String name = "test";

    public String getName() {
        return name;
    }

    public void setName(String sName) { // VIOLATION
        name = sName;
    }
}
