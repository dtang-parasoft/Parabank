Do not define direct or indirect subclasses of 'RuntimeException' [PB.RE.NXRE]



DESCRIPTION 


This rule identifies direct and indirect subclassing of RuntimeException.
An error is reported for each occurrence.
SCOPE LEVEL 

CLASS
SINCE 

v4.5
SECURITY RELEVANCE 

category: Erratic Application Behavior
category: Error Handling

Potential erroneous exception handling. See BENEFITS for more information.
BENEFITS 

Exceptions in java.lang.RuntimeException and its subclasses are used for
avoidable exceptions. The Java compiler does not check the correct handling of
these exceptions, so erroneous exception handling may not be noticed.
EXAMPLE 
         
package examples.rules.pb.re;
         
public class NXRE extends RuntimeException {  // VIOLATION         
    public NXRE (String s) {
        super(s);
    }
}
REPAIR 

Change the class to extend Exception.

package examples.rules.pb.re;
        
public class NXREFixed extends Exception {  // FIXED
    public NXREFixed (String s) {
        super(s);
    }
}
