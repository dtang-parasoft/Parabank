Avoid unnecessary modifiers in an "interface" [VVSG.14_2_G.AAI]



DESCRIPTION 


This rule identifies the use of unnecessary modifiers in "interface" members. 
A parameter allows checking of the "interface" declaration. An error is reported
for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 
-Check for "abstract" in interface declaration (default false)
    Interface declarations are always "abstract," so the keyword is unnecessary
BENEFITS 

Interface methods are always "public" and "abstract". Interface fields are
always "public", "static" and "final".  Methods defined in interfaces are always 
"public" and "abstract".  Types defined in interfaces are always "public" and 
"static".  Interface declarations are always "abstract". These modifiers are 
automatically applied, so there is no need to write them.
EXAMPLE 

package examples.rules.uc;

public interface AAI {
    public void method ();  // VIOLATION: useless "public"
    abstract int getSize ();  // VIOLATION: useless "abstract"
    static int SIZE = 100;  // VIOLATION: useless "static"
}
REPAIR 

Remove the unnecessary modifiers.

package examples.rules.uc;

public interface AAIFixed {
    void method ();  // FIXED
    int getSize ();  // FIXED
    int SIZE = 100;  // FIXED
}
