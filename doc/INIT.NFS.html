Do not use non-final "static" fields during the initialization [INIT.NFS]



DESCRIPTION 


This rule identifies usage of non-final "static" fields in a field's initialization.
An error is reported for each occurrence.
SINCE 

v4.5
BENEFITS 

"Static" values can be modified by other users of a class.  Therefore, 
the "static" field may contain a different value than what users might
expect.
EXAMPLE 

package examples.rules.init;

public class NFS {
    static int max = 10;
    int size = max;  // VIOLATION
}
REPAIR 

Use a "static final" field instead.

package examples.rules.init;

public class NFSFixed {
    static final int MAX = 10;  // FIXED
    static int max = 10;
    int size = NFSFixed.MAX;  // FIXED
}
