Ensure that property entries are localized correctly [PROPS.TENTRY]



DESCRIPTION 


This rule identifies instances where two resources with different locales in the same 
package have the same property names with the same values. An error is reported for 
each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v8.1
PARAMETERS 

-Ignore property file name (regular expression)
    -Allows you to provide a list of property files to ignore. If a property file 
     matches any of the regular expressions, it will be exempt from this rule.
BENEFITS 

If two resources with different locales have the same property names with the same values, 
it means that the value is not localized yet.
EXAMPLE 

------------ message.properties ----------------
internal.error=Internal error

------------ messsage_ja.properties --------------
internal.error=Internal error  // VIOLATION
REPAIR 

------------ message.properties ----------------
internal.error=Internal error

------------ messsage_ja.properties --------------
internal.error=\u5185\u90e8\u30a8\u30e9\u30fc  // FIXED
