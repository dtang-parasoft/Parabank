Runtime Exceptions [PB.RE]

RULES

Do not add a collection to itself [PB.RE.ACTI]
Avoid modifying fixed-size Collections [PB.RE.AMFSL]
Avoid calling the 'add()' method of a Queue [PB.RE.AQA]
Avoid possible ArrayStoreExceptions [PB.RE.ASE]
Always check parameters before use in array access [PB.RE.CAI]
Check the return value of calls to 'String.indexOf()' before passing this value to other "String" methods [PB.RE.CIOR]
Ensure 'Iterator.next()' method implementations throw 'NoSuchElementException' [PB.RE.CTNSE]
Check whether or not "readLine()" returned null before dereferencing the return value [PB.RE.IDRL]
Make sure the IllegalArgumentException message matches the code logic [PB.RE.IIAE]
Use the "fromIndex" argument of 'indexOf()' when necessary [PB.RE.IOSS]
Do not call 'String.equals(constant)' or 'String.equalsIgnoreCase(constant)' [PB.RE.ISEM]
Avoid using methods add() from JLayer class [PB.RE.JLAY]
Ensure that dereferenced variables match variables which were previously checked for "null" [PB.RE.NCMD]
Do not define direct or indirect subclasses of 'RuntimeException' [PB.RE.NXRE]
Avoid dereferencing null objects [PB.RE.PNPD]
When used HttpURLConnection always check return value from the getResponseCode() methods before call getInputStream() [PB.RE.RCODE]
Avoid methods returning "null" value [PB.RE.VRNULL]
