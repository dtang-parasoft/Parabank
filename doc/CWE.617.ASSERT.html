Do not use assertions in production code [CWE.617.ASSERT]



DESCRIPTION 


"The product contains an assert() or similar statement that can be triggered
by an attacker, which leads to an application exit or other behavior that is
more severe than necessary." [CWE-617]

This rule detects when assertion statements are used in production code.
SCOPE LEVEL 

LINE
SINCE 

v10.4
PARAMETERS 

-Check only public methods
(enabled by default)
BENEFITS 

This rule helps you ensure that assertions are not used in your production
code. Assertions should be only be used for testing purposes during
the development phase. In production environment, it is safer to replace
assertions with appropriate exceptions.
EXAMPLE 

The following code triggers a violation:

package examples.rules.codsta.org;

public class ASSERT {
    public String doSomething(int amount) {
        assert amount > 0; //VIOLATION
        ...
    }
}
REPAIR 

package examples.rules.codsta.org;

public class ASSERTFixed {
    public String doSomething(double amount)     {
        if (amount <= 0) { // FIXED
            throw new IllegalArgumentException("Value must be greater than 0!");
        }
        ...
    }
}
