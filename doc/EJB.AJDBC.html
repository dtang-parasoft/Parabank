Do not use JDBC code inside of EJB classes [EJB.AJDBC]



DESCRIPTION 


This rule identifies JDBC code inside EJB classes. An error is reported
for each occurrence.
SINCE 

v5.1
BENEFITS 

Separating data access code from the EJB creates cleaner abstractions,
increases cohesion, and reduces coupling, thus improving modularity 
and reusability.
REFERENCES 

"Core J2EE Patterns - Best Practices and Design Strategies"
by Deepak Alur, John Cupri and Dan Malks. 2001. pp. 113-115.
