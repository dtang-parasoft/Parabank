Avoid using relative ''../'' path reference to parent files [SPRING.IMPORTSRES]



DESCRIPTION 


This rule identifies usage of the relative path to parent in import tags within Spring xml configuration files.
SINCE 

v9.2
NOTES 

It is not recommended to import files from parent directories using a relative "../" path.
BENEFITS 

From Spring Framework Documentation:

section: 3.2.2.1. Composing XML-based configuration metadata

"It is possible to reference files in parent directories using a relative "../" path. 
However, this is not recommended because it creates 
a dependency on a file that is outside the current application. 
This is in particular not recommended for "classpath:" 
URLs (e.g. "classpath:../services.xml") where the runtime resolution 
process will pick the "nearest" classpath root and then look into 
its parent directory. This is fragile since classpath configuration 
changes may lead to a different directory being picked."
EXAMPLE 

<import resource="classpath:../services.xml"/>
REPAIR 

Move services.xml to a directory below and change the configuration to:
<import resource="classpath:./services.xml"/>
