Ensure threads are named [TRS.NAME]



DESCRIPTION 


This rule ensures that threads (or a subclass of Thread) are initialized with a 
name. It also ensures that constructors for subclasses of Thread call the super 
constructor to initialize a name. The rule also flags instantiations of threads
where the constructor gives the thread a default name.
SINCE 

v8.4
PARAMETERS 

-Do not allow constant String literals for thread names (default false)
BENEFITS 

Having thread names is important for debugging.
EXAMPLE 

package examples.rules.trs;

public class Name {
    Thread t = new Thread(); //VIOLATION
    
    class subThread extends Thread {
        subThread() { //VIOLATION
        }
    }
}
REPAIR 

Initialize threads with a name and call a super constructor that initializes
name.

package examples.rules.trs;

public class NameFixed {
    Thread t = new Thread("thread"); //FIXED
    
    class subThread extends Thread {
        subThread() { //FIXED
            super("name");
        }
    }
}
