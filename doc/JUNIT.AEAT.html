Prefer assertEquals and assertNotEquals over assertTrue and assertFalse [JUNIT.AEAT]



DESCRIPTION 


It is recommended to use the assertEquals() and assertNotEquals() methods
rather than assertTrue() and assertFalse() to assert equality in JUnit tests
(see BENEFITS).

This rule detects when the assertTrue() and assertFalse() methods are
used with "==" or "!=" in method parameters.
SINCE 

v9.0
NOTES 

This rule does not trigger if method parameters contain comparison
to "null". See the JUNIT.ANAT rule.
BENEFITS 

This rule helps you improve test code readability and obtain error
messages that offer better insight into why a test failed.
EXAMPLE 

package examples.rules.junit;

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class AEATTest {
    @Test
    public void testMethod() {
        int o = 5;
        assertFalse(o == 1); //VIOLATION
    }
}
REPAIR 

Use assertNotEquals() instead of assertFalse().


package examples.rules.junit;

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class AEATFixedTest {
    @Test
    public void testMethod() {
        int o = 5;
        assertNotEquals(o,1); //FIXED
    }
}
