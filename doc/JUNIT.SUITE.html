Make 'suite()' methods "public" and "static" [JUNIT.SUITE]



DESCRIPTION 


This rule identifies JUnit test case 'suite()' methods that are not "public" and 
"static". An error is reported for each occurrence.
SCOPE LEVEL 
METHOD
SINCE 

v5.0
NOTES 

This rule will only test classes which meet one of the following conditions:

1. The class extends "junit.framework.TestCase".
2. The class has at least one method with the "org.junit.Test" annotation.  This
only applies to JUnit 4.0 and later.
BENEFITS 

JUnit will not recognize the 'suite()' method if it is not "public" and 
"static".
EXAMPLE 

package examples.rules.junit;

import junit.framework.*;

public class SUITE extends TestCase {  

    private void suite() { // VIOLATION

    }
}
REPAIR 

Change the 'suite()' method to "public" and "static".

package examples.rules.junit;

import junit.framework.*;

public class SUITEFixed extends TestCase {

    public static void suite() { // FIXED

    }
}
