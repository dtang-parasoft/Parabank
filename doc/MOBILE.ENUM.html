Avoid using enums [MOBILE.ENUM]



DESCRIPTION 


This rule reports violations when any enums are defined.  There is a lot
of hidden overhead involved with creating enums, so it is ideal to use 
static variables when compile time checking is not a necessity.
SCOPE LEVEL 
LINE
SINCE 

v9.0
NOTES 
When using enums, it can be less expensive to use the ordinal() method than 
to compare objects directly.
BENEFITS 

This rule can help optimize code.
EXAMPLE 

package examples.rules.mobile;

public class ENUM {
	enum Direction {
		NORTH, // gets ordinal = 0
		SOUTH, // gets ordinal = 1
		EAST, // gets ordinal = 2
		WEST; // gets ordinal = 3
	}
	public void test(){
		int heading = (Direction.SOUTH);
	}
}
REPAIR 

Use static variables instead of the enum.


package examples.rules.mobile;

public class ENUM {
	private static final int NORTH = 0;
	private static final int SOUTH = 1;
	private static final int EAST = 2;
	private static final int WEST = 3;

	public void test(){
		int heading = SOUTH;
	}
}
