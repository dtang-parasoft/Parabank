Enforce number of space character(s) between a conditional statement and the opening "(" parenthesis [FORMAT.SC]



DESCRIPTION 


This rule identifies any keyword for conditional, loop, and exception statement
that is not followed by one whitespace(by default) character (unless it is at 
the end of a line or followed by comment). 

This rule applies to the keywords "if", "else", "switch", "for", "while",
and "catch". If such a keyword is followed by an opening
parenthesis "(", there must be no more than one space(by default) after the
parenthesis. The keyword "synchronized" is not subject to rule FORMAT.SC.
SCOPE LEVEL 
LINE
SINCE 

v8.0
PARAMETERS 

-Number of spaces in between (default value is 1)
BENEFITS 

If all team members follow the same formatting style, it will be easier for each
team member to read and understand the code.
EXAMPLE 

package examples.rules.format;

public class SC {
    public void method (int i) {
        if(  i < 0) {  // VIOLATION
            System.out.println("Less than zero");
        }
    }
}
REPAIR 

Place a single space before the "(" and place no more than one space after
the parenthesis.

package examples.rules.format;

public class SCFixed {
    public void method (int i) {
        if (i < 0) {  // FIXED
      //if ( i < 0) { // Would also be acceptable  
            System.out.println("Less than zero");
        }
    }
}
