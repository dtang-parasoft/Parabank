Use a naming convention for "static" methods [NAMING.NSM]



DESCRIPTION 


This rule identifies "static" methods that do not follow the user-defined naming 
convention (as specified in a regular expression).  The rule does not enforce a 
default requirement. An error is reported for each occurrence.
SCOPE LEVEL 
METHOD
SINCE 

v4.5
PARAMETERS 

-regular expression
    -"static" method names should match the listed regular expression.
    -There is no default regular expression.
-check public method. (default is true)
-check protected method. (default is true)
-check package-private method. (default is true)
-check private method. (default is true)
-Negate regular expression
    -If this parameter is enabled, a violation will be flagged if the name of a
     "static" method matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for "static" method names which do not match the regular expression
     specified.
-Ignore case
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of "static" 
     methods must match the regular expression specified, including in case.
BENEFITS 

This helps enforce a naming convention.
