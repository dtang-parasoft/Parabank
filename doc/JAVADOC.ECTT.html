Enforce custom Javadoc tags for types [JAVADOC.ECTT]



DESCRIPTION 


This rule identifies Javadoc comment of classes, interfaces, enums and annotations
that lack the Javadoc tag the users specified. An error is reported
for each missing tag.
SCOPE LEVEL 
LINE
SINCE 

v8.0
NOTES 

This rule would not report any violation if the Javadoc is not present,
since another rule JAVADOC.PJDC would report missing Javadoc.
PARAMETERS 

-check "public" types (default as true)
-check "protected" types (default as false)
-check "package-private" types (default as false)
-check "private" types (default as false)

-do not check classes with the following string(default as internal)
If the string is in a single line comment directly before the top level class
declaration, the c-style block comments will have no effect.
        //internal
        public class NOCHECK
         /*internal*/
        public class CHECKNORMALLY
-do not check files within the following packages(default as .*internal.*)
Files will be ignored if they are in packages that match the regular expression.

-custom Javadoc tags to be enforced
Users can enter a list of tags they wish to enforce for the classes and
interfaces. By default, @since is in the list.
BENEFITS 

Commenting your code is a good way to document its behavior.
EXAMPLE 

package examples.rules.javadoc;

/**
 * Class PJDC
 */
public class PJDC { //VIOLATION, missing @since
}
REPAIR 

package examples.rules.javadoc;

/**
 * Class PJDC
 * @since v1.0
 */
public class PJDCFixed { //FIXED
}
