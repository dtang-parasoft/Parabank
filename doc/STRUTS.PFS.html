Ensure that the @path attribute of '<action>' and '<forward>' elements begins with '/' in 'struts-config.xml' files [STRUTS.PFS]



DESCRIPTION 


This rule checks 'struts-config.xml' files. A violation is flagged for each
'<action>' or '<forward>' element with a @path attribute that does not begin
with '/'.
SINCE 

v8.4
SECURITY RELEVANCE 

category: Unsafe Environment Configuration

A @path attribute is intended to be context relative. Failure to begin a path
with '/' may cause an incorrect page to load or none at all.
EXAMPLE 

The '<action>' element in the sample 'struts-config.xml' file has a @path
attribute which does not begin with '/'

struts-config.xml:

<?xml version="1.0" encoding="ISO-8859-1" ?>
<struts-config>
    <form-beans>
        <form-bean name="logonForm" type="app.LogonForm"/>
    </form-beans>
    <action-mappings>
        <!-- VIOLATION: @path does not begin with '/'  -->
        <action
            path="LogonSubmit"
            type="app.LogonAction"
            name="logonForm"
            scope="request"
        </action>
    </action-mappings>
</struts-config>
REPAIR 

Begin all @path attributes with '/'

struts-config.xml:
    
<?xml version="1.0" encoding="ISO-8859-1" ?>
<struts-config>
    <form-beans>
        <form-bean name="logonForm" type="app.LogonForm"/>
    </form-beans>
    <action-mappings>
        <!-- FIXED -->
        <action
            path="/LogonSubmit"
            type="app.LogonAction"
            name="logonForm"
            scope="request"
        </action>
    </action-mappings>
</struts-config>
