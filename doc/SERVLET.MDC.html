Declare a "public" constructor that takes no parameters [SERVLET.MDC]



DESCRIPTION 


This rule identifies Servlet classes that lack a "public" constructor
that takes no parameters.  An error is reported for each occurrence.
Note that if a Servlet class does not define any constructor, it would
not be flagged, since such a class would have an implicit default constructor 
that takes no parameter.
SCOPE LEVEL 
CLASS
SINCE 

v4.5
BENEFITS 

The Container invokes no-argument constructor to create an 
instance of the Servlet class. If such a constructor is not present,
the Container would be unable to create the Servlet instance.
EXAMPLE 

package examples.rules.servlet;

import javax.servlet.*;
import javax.servlet.http.*;

public class MDC extends HttpServlet {

    // VIOLATION : missing default constructor
    public MDC(String name) { 
    // do something
    }
}
REPAIR 

package examples.rules.servlet;

import javax.servlet.*;
import javax.servlet.http.*;

public class MDCFixed extends HttpServlet {

    public MDCFixed(String name) { 
    // do something
    }

    public MDCFixed() { // FIXED
    // do something
    }
}
