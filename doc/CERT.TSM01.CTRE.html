Do not let "this" reference escape during construction [CERT.TSM01.CTRE]



DESCRIPTION 


This rule identifies cases where an anonymous class is published from within a
constructor. This allows a "this" reference to escape, and in an environment
with multiple threads, it may be possible to view an incompletely constructed
object.
SINCE 

v8.4
SECURITY RELEVANCE 

category: Concurrency

Viewing of an incompletely constructed object can allow the program state to
become inconsistent or unexpected, which can lead to confusion and erroneous
behavior.
EXAMPLE 

package examples.rules.trs;

public class CTRE {
    public CTRE(EventSource source) {
        source.registerListener( //VIOLATION
            new EventListener() {
                public void onEvent(Event e) {
                    doSomething(e);
                }
            });
    }
}
REPAIR 

Do not publish until the constructor has safely completed. One method is to
create a separate initializing method, which calls the constructor, and then
publishes afterwards.

package examples.rules.trs;

public class CTREFixed {
    private final EventListener listener;

    private CTREFixed () {
        listener = new EventListener() { 
            public void onEvent(Event e) {
                doSomething(e);
            }
        };
    }

    public static CTREFixed newInstance(EventSource source) {
        CTREFixed safe = new CTREFixed();
        source.registerListener(safe.listener); //FIXED
        return safe;
    }
}
