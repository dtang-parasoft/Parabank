Do not use initialization circularities for fields [INIT.IC]



DESCRIPTION 


This rule identifies initialization circularities of "static" fields. An error 
is reported for each occurrence. 

Initialization circularities occur for initializations of "static" fields 
(cross classes/files) that depend on each other's initialized values.
SCOPE LEVEL 
LINE
SINCE 

v6.0
SECURITY RELEVANCE 

category: Erratic Application Behavior
category: Code Quality

Obfuscating and confusing code is error-prone and hard to maintain, therefore 
increases the likelihood of security problems. See BENEFITS for more
information.
BENEFITS 

Because of the class loading mechanism, the initialized values of static fields
might be different depending on which class is loaded first. Such an uncertainty
could be a potential bug.
EXAMPLE 

package examples.rules.init;

public class IC {
    public static void main(String args[]) {
        IC2 ref2= new IC2(); //If the order is switched, the result would be different
        IC1 ref1= new IC1();  
        System.out.println(IC1.j + " "+ IC2.i);
    }
}
class IC1 {
    static int j= IC2.i+ 1; //VIOLATION
}
class IC2 {
    static int i= IC1.j;  //VIOLATION
}
REPAIR 

package examples.rules.init;

public class ICFixed {
    public static void main(String args[]) {
        IC2 ref2= new IC2(); 
        IC1 ref1= new IC1();  
        System.out.println(IC1.j + " "+ IC2.i);
    }
}
class IC1 {
    static int j= IC2.i+ 1; //FIXED
}
class IC2 {
    static int i= 0; //FIXED  
}
