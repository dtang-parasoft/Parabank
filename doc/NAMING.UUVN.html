Do not use variables with identical names or names that differ only in case [NAMING.UUVN]



DESCRIPTION 


This rule checks to make sure that local variables and parameters are not 
declared with names that are identical to the name of another local variable
or parameter in the same scope except in capitalization.  A violation will be 
flagged for each local variable or parameter which violates this rule.
SCOPE LEVEL 
LINE
SINCE 

v8.0
BENEFITS 

While it is not a compiler error to have two variables with names that differ
only in case in the same scope, it is very confusing to the person reading the
code.  Unique variable names should be used.
EXAMPLE 

package examples.rules.naming;

public class UUVN {
    public int sum(int foo){
        int Foo= 9; //VIOLATION- already a parameter named "foo"
        return foo+5;
    }
}
REPAIR 

package examples.rules.naming;

public class UUVNFixed {
    public int sum(int foo){
        int bar= 9; //FIXED
        return foo+5;
    }
}
