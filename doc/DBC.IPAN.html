Include a '@pre != null' tag for each parameter that is dereferenced before being checked for null [DBC.IPAN]



DESCRIPTION 


This rule flags parameters which are dereferenced before being checked for
"null". Parameters which have a "@pre" javadoc tag denoting they are never null
are ignored.
SINCE 

v8.4
NOTES 

If a parameter is dereferenced by assignment, field access, or method invocation
then the user may know that the parameter is never null. Otherwise, a
NullPointerException would be thrown. If this is the case a "@pre" javadoc tag
should be added denoting this fact. Also, the user may have mistakingly
forgotten to check for "null"
PARAMETERS 

-Check "public" methods (default true)
-Check "protected" methods (default true)
-Check "package-private" methods (default true)
-Check "private" methods (default true)
EXAMPLE 

package examples.rules.dbc;

public class IPAN {
    /**
     * @pre s != null
     */
    public String concat(Object s, Object t) {
        return s.toString() + t.toString(); //VIOLATION
    }
}
REPAIR 

Remove the misleading postcondition, or change it so that it is correct.

package examples.rules.dbc;

public class IPANFixed {
    /**
     * @pre s != null
     * @pre t != null
     */
    public String concat(Object s, Object t) {
        return s.toString() + t.toString(); //FIXED
    }
}
