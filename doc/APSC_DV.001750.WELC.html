Ensure all web content directories have a "welcome file" [APSC_DV.001750.WELC]



DESCRIPTION 


This rule identifies if web content directories didn't have a "welcome file" which
the "welcome files" are specified in 'web.xml'. An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v9.0
SECURITY RELEVANCE 

Enforcing this rule will help to protect against some of the OWASP Top 10 2013
application vulnerabilities, including:

A5-Security Misconfiguration

For example, if a web content directory, '/admin' didn't have a welcome file, hackers
will be able to see and download all the files in '/admin' directory by 
typing url 'http://notsafe.com/admin'
PARAMETERS 

-Select which folders(web content directroies) will be used in web applications:
    [x] All folders will be used (selected by default)
    [ ] List of folders will be used (Glob expression)
EXAMPLE 

<!--- web.xml -->
<web-app>
   ......
   <welcome-file-list>
        <welcome-file>index.jsp</welcome-file>
   </welcome-file-list>
   .....
</web-app>
   
<!--- web application tree structure -->

    + /
    |
    |--+ admin     <------ VIOLATION : missing welcome file
       |
       |--+ login.jsp
          + account.jsp
    |
    |--+ other
       |
       |+  ........
    |
    |--+ WEB-INF
       |
       |+ .......
REPAIR 

Add index.jsp in /admin web content directory

<!--- web.xml -->
<web-app>
   ......
   <welcome-file-list>
        <welcome-file>index.jsp</welcome-file>
   </welcome-file-list>
   .....
</web-app>
   
<!--- web application tree structure -->

    + /
    |
    |--+ admin     <------ FIXED : add welcome file 'index.jsp'
       |
       |--+ index.jsp
          + login.jsp
          + account.jsp
    |
    |--+ other
       |
       |+  ........
    |
    |--+ WEB-INF
       |
       |+ .......
