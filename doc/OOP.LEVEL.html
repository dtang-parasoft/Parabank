Avoid more than two levels of nested inner classes [OOP.LEVEL]



DESCRIPTION 


Requires that no more than two levels of nested inner classes be used.
SCOPE LEVEL 

CLASS
SINCE 

v4.5
SECURITY RELEVANCE 

category: Erratic Application Behavior
category: Code Quality

Complicated code is error-prone and hard to maintain, 
therefore increases the likelihood of security problems.
BENEFITS 

Nesting more than two levels deep can be hard to follow.
EXAMPLE 

package examples.rules.oop;

public class LEVEL {
    class Level1 {
        class Level2 {
            class Level3 {  // VIOLATION
                private boolean _isClosed = false;
            }
            private int _count = 0;
        }
        private int _size = 0;
    }
    private int _length = 0;
}
REPAIR 

Move the code from the nested classes into separate classes in their own files.

First file 'LEVELFixed.java':

package examples.rules.oop;

public class LEVELFixed {
    class Level1 {
        class Level2 {
            private int _count = 0;
        }
        private int _size = 0;
    }
    private int _length = 0;
}

class Level3Fixed {
    private boolean _isClosed = false;
}
