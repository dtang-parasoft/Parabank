Avoid constructor injection [SPRING.ACARG]



DESCRIPTION 


This rule flags each use of the <constructor-arg> tag in Spring XML configuration files.
SINCE 

v9.2
NOTES 

Referring to constructors in Spring XML configurations may lead to ambiguity when the class
declares multiple constructors. Property injection is recommended instead of constructor injection.
BENEFITS 

Setter injection is more flexible and manageable, especially when the class has multiple
properties and some of them are optional.
EXAMPLE 

<bean id="myBean"
        class="examples.MyClass0"
        <constructor-arg value="123"/>
    </bean>
REPAIR 

<bean id="myBean"
         class="examples.MyClass0"
        <property name="maxCapacity"
            value="123">
    </bean>
REFERENCES 

1. Twelve Best Practices For Spring XML Configurations by Jason Zhicheng Li (O'Reilly ONJava.com)
http://onjava.com/pub/a/onjava/2006/01/25/spring-xml-configuration-best-practices.html
