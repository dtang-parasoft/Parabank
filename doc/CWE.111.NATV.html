Do not use user-defined "native" methods [CWE.111.NATV]



DESCRIPTION 



This rule identifies user-defined native methods. An error is
reported for each occurrence.
SCOPE LEVEL 

METHOD
SINCE 

v4.5
SECURITY RELEVANCE 

category: Input-Based Attacks
category: Malicious Code

Native code can cause security problems such as buffer 
overflows and format string vulnerabilities.

This rule helps to enforce the PCI DSS (Payment Card Industry Data Security 
Standard) Requirement #6: "Develop and maintain secure systems and applications".
Specifically, this rule tests for Issue 6.5.5: "Buffer overflows".  Since native 
code is often vulnerable to buffer overflows, enforcing this rule will help to
enforce this requirement.
BENEFITS 

User-defined native methods are not portable because all native methods 
must be ported to each platform.
DRAWBACKS 

Some features must be implemented at the native level to use native libraries
or implement a performance optimization.
EXAMPLE 

package examples.rules.port;

public class NATV {
    native void method (String s);  // VIOLATION
}
REPAIR 

Find a non-native way to perform the operation.
