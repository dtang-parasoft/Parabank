Follow a limit for the number of mappings for each servlet in a 'web.xml' file [SERVLET.NMS]



DESCRIPTION 


Servlets should not be mapped to multiple URL patterns in a 'web.xml' file.  By 
default, this rule will flag a violation for each servlet name in a 'web.xml' 
file which has more than one mapping.
SINCE 

v8.4
SECURITY RELEVANCE 

category: Unsafe Environment Configuration

If you map multiple URL patterns to a single servlet name, it may be a mistake.  
You may have intended to specify a different servlet for at least one of those 
'<servlet-mapping>' elements.  This mistake is common when carelessly copying 
and pasting code.  This mistake can lead to a denial of access to the servlet 
you intended to specify in the mapping or other problems.
PARAMETERS 

-Maximum number of mappings for each servlet
    -The default value is "1".
    -With the default value of "1", the rule will flag a violation if there are 
     more than 1 '<servlet-mapping>' elements for the same servlet in a 
     'web.xml' file.  If you would like to allow multiple mappings for a single
     servlet, you should increase the value of this parameter.
BENEFITS 

Mapping a single servlet to multiple URL patterns is confusing and unnecessary.  
You should try to only have one mapping defined for each servlet.
EXAMPLE 

The servlet named "ExampleServlet" is mapped to 5 different URL patterns in this
example 'web.xml' file.

web.xml:

<web-app xmlns="http://java.sun.com/xml/ns/j2ee"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xsi:schemaLocation="http://java.sun.com/xml/ns/j2ee http://java.sun.com/xml/ns/j2ee/web-app_2_4.xsd"
        version="2.4">    
     
    <servlet>
        <servlet-name>ExampleServlet</servlet-name>
        <servlet-class>examples.rules.servlet.ExampleServlet</servlet-class>
        <load-on-startup>1</load-on-startup>
    </servlet>       
                     
    <servlet-mapping>
        <servlet-name>ExampleServlet</servlet-name>  <!-- VIOLATION: The servlet 
                             "ExampleServlet" is mapped to 5 different URL patterns. -->
        <url-pattern>/ExampleServlet</url-pattern>
    </servlet-mapping>
    
    <servlet-mapping>
        <servlet-name>ExampleServlet</servlet-name>  <!-- VIOLATION: The servlet 
                             "ExampleServlet" is mapped to 5 different URL patterns. -->
        <url-pattern>/Foo*</url-pattern>
    </servlet-mapping>
    
    <servlet-mapping>
        <servlet-name>ExampleServlet</servlet-name>  <!-- VIOLATION: The servlet 
                             "ExampleServlet" is mapped to 5 different URL patterns. -->
        <url-pattern>/Bar</url-pattern>
    </servlet-mapping>
    
    <servlet-mapping>
        <servlet-name>ExampleServlet</servlet-name>  <!-- VIOLATION: The servlet 
                             "ExampleServlet" is mapped to 5 different URL patterns. -->
        <url-pattern>/Baz</url-pattern>
    </servlet-mapping>
    
    <servlet-mapping>
        <servlet-name>ExampleServlet</servlet-name>  <!-- VIOLATION: The servlet 
                             "ExampleServlet" is mapped to 5 different URL patterns. -->
        <url-pattern>/*</url-pattern>
    </servlet-mapping>

</web-app>
REPAIR 

Map each servlet to a single URL pattern.

web.xml:

<web-app xmlns="http://java.sun.com/xml/ns/j2ee"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xsi:schemaLocation="http://java.sun.com/xml/ns/j2ee http://java.sun.com/xml/ns/j2ee/web-app_2_4.xsd"
        version="2.4">    
     
    <servlet>
        <servlet-name>ExampleServlet</servlet-name>
        <servlet-class>examples.rules.servlet.ExampleServlet</servlet-class>
        <load-on-startup>1</load-on-startup>
    </servlet>       
                     
    <servlet-mapping>
        <servlet-name>ExampleServlet</servlet-name>  <!-- FIXED: Map the servlet 
                                          "ExampleServlet" to only 1 URL pattern. -->
        <url-pattern>/ExampleServlet</url-pattern>
    </servlet-mapping>

</web-app>
