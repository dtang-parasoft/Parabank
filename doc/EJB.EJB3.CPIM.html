Call 'InvocationContext.proceed()' from @AroundInvoke methods [EJB.EJB3.CPIM]



DESCRIPTION 


This rule flags a violation if a method with the "@AroundInvoke" annotation does
not call "InvocationContext.proceed()" with the parameter. If "proceed()" is not
called, the method which was intercepted will never be called.
SINCE 

v8.4
NOTES 

A violation is only flagged if the "@AroundInvoke" method matches the signature:
"Object <METHOD>(InvocationContext) throws Exception"
EXAMPLE 

package examples.rules.ejb.ejb3;

public class CPIM {
    @AroundInvoke
    Object aroundInvokeMethod(InvocationContext ic) throws Exception { //VIOLATION
        _logger.entering("CPIM", "interceptedMethod");
        return null;
    }
}
REPAIR 

Add a call to "InvocationContext.proceed()"

package examples.rules.ejb.ejb3;

public class CPIMFixed {
    @AroundInvoke
    Object aroundInvokeMethod(InvocationContext ic) throws Exception { //FIXED
        _logger.entering("CPIMFixed", "interceptedMethod");
        return ic.proceed();
    }
}
