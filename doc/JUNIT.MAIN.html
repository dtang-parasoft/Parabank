Ensure that JUnit classes have a main() allowing them to be executed in isolation [JUNIT.MAIN]



DESCRIPTION 


This rule identifies any JUnit test class that does not have a main() method
that allows it to be executed in isolation. An error
is reported for each occurrence.
SCOPE LEVEL 

CLASS
SINCE 

v4.5
NOTES 

This rule will only test classes which meet one of the following conditions:

1. The class extends "junit.framework.TestCase".
2. The class has at least one method with the "org.junit.Test" annotation.  This
only applies to JUnit 4.0 and later.
BENEFITS 

Although unit tests are often grouped into a suite of tests,
developers should be allowed to run individual tests in isolation.
EXAMPLE 

package examples.rules.junit;

import junit.framework.*;

public class MAINTest extends TestCase {
  
   // VIOLATION : missing main method
}
REPAIR 

Add a main () method.

package examples.rules.junit;

import junit.framework.*;

public class MAINFixedTest extends TestCase {
  
    public static void main(String[] args) {  // FIXED

    }
}
