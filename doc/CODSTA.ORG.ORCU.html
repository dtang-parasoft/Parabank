Order compilation unit elements appropriately [CODSTA.ORG.ORCU]



DESCRIPTION 


This rule identifies inappropriately-ordered compilation units.
An error is reported if a compilation unit is not ordered as follows:

1) File header.
2) "package" statement.
3) "import" statements.
4) "public" type definition.
5) Non-"public" types definitions.

Note that this rule verifies whether the elements are properly
ordered; it does not verify whether the elements actually exist.
This rule can produce spurious warning messages for source files
that precede imports or groups of imports with additional comments
("import comments").
SCOPE LEVEL 
FILE
SINCE 

v5.0
BENEFITS 

Makes code easier to read and less error-prone.
EXAMPLE 

package examples.rules.codsta.org;  // VIOLATION

/*
 * File header.
 */

/**
 * Top class javadoc header.
 */

public class ORCU {
}
REPAIR 

Reorder the compilation unit.

/*
 * File header.
 */

package examples.rules.codsta.org;  // FIXED

/**
 * Top class javadoc header.
 */

public class ORCUFixed {
}
