Do not define instance fields in Servlet classes [SERVLET.IF]



DESCRIPTION 


This rule identifies instance fields which are defined in Servlet 
classes.  This rule will flag a violation for each instance field 
declared in a class which implements the "javax.servlet.Servlet" 
interface.
SINCE 

v8.0
BENEFITS 

The use of instance member variables in Servlet classes is discouraged 
because only one instance of a Servlet class is created by the J2EE 
framework and used in a multithreaded way.  Thus, it is probably incorrect
to use instance member variables.  Local variables should be used instead
of instance variables.
EXAMPLE 

package examples.rules.servlet;

import javax.servlet.http.*;

public class IF extends HttpServlet {
    private Object o;  // VIOLATION
}
REPAIR 

Change the instance variable to a local variable or a static field, or 
remove it if it is not necessary.

package examples.rules.servlet;

import javax.servlet.http.*;

public class IFFixed extends HttpServlet {
    //FIXED- instance variable removed
}
