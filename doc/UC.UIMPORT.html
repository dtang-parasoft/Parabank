Avoid unused "import" statements [UC.UIMPORT]



DESCRIPTION 


This rule identifies unused import statements. An error is reported for all unused import
statements-- even when types are declared using the fully qualified name.
SCOPE LEVEL 
FILE
SINCE 

v5.0
PARAMETERS 

-Ignore unused "import" statements if the fully-qualified type name is used. (default is false)
    If enabled, this rule would act like the former Jtest rule UC.UIMPORT2, which is deprecated.
-Ignore "import" statements if the "import" statement is used in the Javadoc. (default is true)
    If enabled, the rule will count usages of "import" statements in the Javadoc
    comments of a file as valid usages of the "import" statement.
BENEFITS 

If you do not follow this rule and are not precise about what you are importing,
readers will have a difficult time understanding your classes' context and dependencies.
This makes code more difficult to read and more error prone.
EXAMPLE 

package examples.rules.uc;

import java.io.*;  // VIOLATION
import java.util.Vector;  // VIOLATION

public class UIMPORT {

    private java.util.Vector v;

}
REPAIR 

Remove the import statement.

package examples.rules.uc;

// FIXED : remove the java.io.* statement
// FIXED : remove the java.util.Vector statement

public class UIMPORTFixed {

    private java.util.Vector v;

}
LOG 

@deprecated UC.UIMPORT2
