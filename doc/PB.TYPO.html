Typographical Errors [PB.TYPO]

RULES

Avoid "try", "catch" and "finally" blocks with empty bodies [PB.TYPO.AECB]
Avoid assignment within a condition [PB.TYPO.ASI]
Avoid unnecessary self computation [PB.TYPO.BW]
Ensure that the correct "super" method is invoked [PB.TYPO.CSI]
Avoid assigning same variable in the fall-through switch case [PB.TYPO.DAV]
Avoid control statements with empty bodies [PB.TYPO.EB]
Ensure the overriding method name does not have a typo [PB.TYPO.IMO]
Do not give methods and fields the same name as the enclosing class or each other [PB.TYPO.NAMING]
Avoid having more than one "getter" or "setter" method for the same field [PB.TYPO.RSK]
Ensure "switch" statements do not contain typos [PB.TYPO.TLS]
Do not use octal integer literals [PB.TYPO.UOL]
Ensure that arguments passed to Java wrapper classes do not contain typos [PB.TYPO.WT]
