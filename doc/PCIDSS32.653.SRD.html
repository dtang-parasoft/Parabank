Use 'java.security.SecureRandom' instead of 'java.util.Random' or 'Math.random()' [PCIDSS32.653.SRD]



DESCRIPTION 


This rule identifies declarations of the 'java.util.Random'
object or method invocations of 'Math.random()'. An error
is reported for each occurrence.
SINCE 

v6.0
SECURITY RELEVANCE 

category: Weak Security Controls
category: Cryptography

Enforcing this rule will help to protect against the OWASP Top 10 2021
application vulnerability "A02:2021 – Cryptographic Failures".

Avoid weak random number generators. See BENEFITS for more
information.
BENEFITS 

'java.security.SecureRandom' is more secure than
'java.util.Random' and 'Math.random()'. The 'SecureRandom'
class provides a cryptographically strong pseudo-random
number generator (PRNG). A cryptographically strong
pseudo-random number minimally complies with the
statistical random number generator tests specified in FIPS
140-2, Security Requirements for Cryptographic Modules,
section 4.9.1. Additionally, SecureRandom must produce
non-deterministic output; thus, it is required that
the seed material be unpredictable and that output of
SecureRandom be cryptographically strong sequences as
described in RFC 1750:
Randomness Recommendations for Security.
EXAMPLE 

package examples.rules.security.wsc;

public class SRD {
    public static void main(String[] args) {
        java.util.Random r = new java.util.Random(); //VIOLATION
    }
}
REPAIR 

package examples.rules.security.wsc;

public class SRDFixed {
    public static void main(String[] args) {
        java.security.SecureRandom r = new java.security.SecureRandom(); //FIXED
    }
}
