DEPRECATED: Number of comment lines in a "class" or "interface" [METRICS.NOTCML]



DESCRIPTION 


Rules from this category are deprecated - see METRIC category.

The number of comments in a class indicates whether or not the code is
documented. This rule includes: //, /* */, and /** */ format comments that
are not part of other statements.
NOTES 

Jtest will report an error if the value is greater or equal 500.
BENEFITS 

Every class, interface, method and field should have a Javadoc comment
associated with it. Documentation helps users of your code to understand it
better.
EXAMPLE 

package examples.rules.metrics;

public class CCNL {
    /** javadoc for example method
     */
    public void example (boolean isDone) {
        if (isDone) {  /* this is a comment */
            System.out.println ("this is a statement");
            someMethod ();
        } // end of if
    }  // end of example.

    void someMethod () {
    }
}

//This class' comments: 5
//This class' statements: 7
//This class' total lines: 13
