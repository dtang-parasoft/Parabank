Avoid splitting property names into multiple lines [PROPS.MLN]



DESCRIPTION 


This rule identifies property name strings which are separated into multiple 
lines.  An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v8.1
PARAMETERS 

-Ignore property file names (regular expression)
    -Allows you to provide a list of property files to ignore. If a property file 
     matches any of the regular expressions, it will be exempt from this rule.
BENEFITS 

Separating property names into multiple lines will make it harder for the user 
to read. This separation is unnecessary.
EXAMPLE 

a\
b=AB  // VIOLATION
REPAIR 

ab=AB  // FIXED
