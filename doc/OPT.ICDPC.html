When calling "Math.ceil()", do not pass in an integer that has been cast to a floating point type [OPT.ICDPC]



DESCRIPTION 


This rule checks to make sure that an integer type (short, int, or long) is not 
cast to a floating point type (float or double) and passed to the "Math.ceil()" 
method.  This operation is a no-op since converting an integer to a floating 
point should give a number with no fractional part.  A violation will be flagged
for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v8.0
BENEFITS 

Mobile platforms have reduced resources, optimizations are especially important.

Calling "Math.ceil()" with a number that has no fractional part as the argument
is unnecessary and is likely indicative of a bug in the logic of the code.  
Perhaps the argument to "Math.ceil()" was supposed to be something that
EXAMPLE 

package examples.rules.opt;

public class ICDPC{
    public static void main(String[] args){
        System.err.println("The ceiling of 3/4 is: "+
                Math.ceil((double)(3/4))); //VIOLATION
    }
}
REPAIR 

package examples.rules.opt;

public class ICDPCFixed{
    public static void main(String[] args){
        System.err.println("The ceiling of 3/4 is: "+ (3/4)); // FIXED : removed unnecessary Math.ceil() call
    }
}
