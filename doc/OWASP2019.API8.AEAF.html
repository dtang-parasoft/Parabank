Do not extend from the Struts classes 'ActionForm' and 'DynaActionForm' [OWASP2019.API8.AEAF]



DESCRIPTION 



This rule makes sure that classes do not extend from the class "org.apache.struts.action.
ActionForm" or from the class "org.apache.struts.action.DynaActionForm". A violation will be 
flagged for each class extending either ActionForm or DynaActionForm.
SINCE 

v8.0
SECURITY RELEVANCE 

category: Input-Based Attacks

If an ActionForm is not validated, the values passed to the system can cause various types of 
security errors based on unvalidated inputs such as SQL injection, cross-site scripting, and 
other injection vulnerabilities.
BENEFITS 

Instead of extending ActionForm or DynaActionForm, Struts ActionForms should extend from one
of the following:
    org.apache.struts.validator.ValidatorForm
    org.apache.struts.validator.ValidatorActionForm
    org.apache.struts.validator.DynaValidatorForm
    org.apache.struts.validator.DynaValidatorActionForm
Extending ActionForm or DynaActionForm causes the Struts validation framework to be bypassed.
EXAMPLE 

package example.rules.security.iba;

import org.apache.struts.action.ActionForm;

public class AEAF extends ActionForm  {
    public AEAF() {
        super();        
    }
}
REPAIR 

Extend from ValidatorForm, ValidatorActionForm, DynaValidatorForm, or DynaValidatorActionForm
instead of ActionForm or DynaActionForm.

package example.rules.security.iba;

import org.apache.struts.validator.ValidatorForm;

public class AEAFFixed extends ValidatorForm  {
    public AEAFFixed() {
        super();        
    }
}
