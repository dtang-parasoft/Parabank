Make sure implementation of Object.equals(Object) is reflexive [CERT.MET08.EQREFL]



DESCRIPTION 

This rule detects Object.equals(Object) methods that return false when called with the same object
for which the method is invoked as a parameter. This is inconsistent with the equals method rule,
which states that for any non-null reference value x, x.equals(x) should return true.

EXAMPLE 
Here is an example that triggers a violation:

public class Example
{
    int a;

    public boolean equals(Object obj)
    {
        if (obj instanceof Example) {
            return ((Example)obj).a < a; // VIOLATION
        }
        return false;
    }
}
REPAIR 

public class Repair
{
    int a;

    public boolean equals(Object obj)
    {
        if (obj instanceof Repair) {
            return ((Repair)obj).a == a;
        }
        return false;
    } // NO VIOLATION
}
