Avoid using potential Java keywords as identifiers [NAMING.ID]



DESCRIPTION 


This rule identifies a variable that is considered to be a java keyword
in other version of JDK. An error is reported for each occurrence.
SINCE 

v8.2
NOTES 

Considerable variable names are:
  - enum
  - assert
BENEFITS 

Using potential java keyword name in variable will cause compilation error
when the JDK version is changed. Therefore, developers should avoid using those
names in their variables.
EXAMPLE 

package examples.rules.naming;

public class ID {

    private String enum;  // VIOLATION
    private int assert;   // VIOLATION
}
REPAIR 

Rename the fields

package examples.rules.naming;

public class IDFixed {
 
    private String myEnum;  // FIXED
    private int _myAssert;  // FIXED

}
