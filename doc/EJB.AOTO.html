Avoid one-to-one mapping between session beans and entity beans [EJB.AOTO]



DESCRIPTION 


This rule identifies one-to-one mapping between session beans 
and entity beans. An error is reported if the session bean only
wraps one entity bean.
SCOPE LEVEL 

CLASS
SINCE 

v5.1
BENEFITS 

There is no benefit to a one-to-one mapping of a session bean to
an entity bean. Such mapping only introduces a layer of session beans
that act as proxies.
REFERENCES 

"Core J2EE Patterns - Best Practices and Design Strategies"
by Deepak Alur, John Cupri and Dan Malks. 2001. pp. 108-109.
