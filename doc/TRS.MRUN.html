Give subclasses of Thread a 'run()' method so they can run as separate threads [TRS.MRUN]



DESCRIPTION 


This rule identifies classes that extend "Thread" but do not have a 'run()' 
method. An error is reported for each occurrence.
SCOPE LEVEL 
CLASS
SINCE 

v4.5
NOTES 

This rule will not flag on "abstract" classes which extend "Thread".  For 
"abstract" Thread classes, it is acceptable if the subclasses define their own
'run()' methods.
BENEFITS 

Classes that extend 'Thread' should always have a 'run()' method.  If a 'run()' 
method is not implemented, the class will not run as a separate thread.
EXAMPLE 

package examples.rules.trs;

public class MRUN extends Thread{
    public MRUN () {}

    // VIOLATION, no run method
}
REPAIR 

Provide a 'run()' method for this class.

package examples.rules.trs;

public class MRUNFixed extends Thread {
    public void run () {}  // FIXED
}
