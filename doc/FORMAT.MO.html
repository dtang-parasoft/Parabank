Enforce the order of annotations and modifiers [FORMAT.MO]



DESCRIPTION 


This rule identifies code that does not properly order modifiers and 
annotations. With the default parameters, an error is reported if a declaration 
does not order its modifiers as follows (from first to last):

annotations
public/protected/private
abstract
static
final
transient
volatile
synchronized
native
strictfp

Note that, by default, annotations must come before any other modifiers.
SCOPE LEVEL 
LINE
SINCE 

v6.0
PARAMETERS 

-Sort modifiers based on modifier types
    You can modify the order of the modifiers.
-Order of Annotations    
    Specify whether the annotations should come before or after any other 
    modifiers. (Default is "before".)
BENEFITS 

Following coding conventions can improve the integrity and 
consistency of the code.
EXAMPLE 

package examples.rules.format;

public class MO {
    static public void main(String args[]) {} //VIOLATION
}
REPAIR 

package examples.rules.format;

public class MOFixed {
    public static void main(String args[]) {} //FIXED
}
