Include at least one test method in each 'TestCase' class [JUNIT.TCWNT]



DESCRIPTION 


This rule identifies non-abstract classes which extend from 'junit.
framework.TestCase' but do not contain any test methods.  A violation 
will be flagged for each occurrence.
SINCE 

v8.0
NOTES 

This rule defines a test method as any method which has a name beginning with
"test".  ("test" is not case sensitive for purposes of this rule.)
BENEFITS 

If a class is a JUnit 'TestCase' class, it should contain at least one test
method.  Otherwise, the class does not serve any purpose for testing.
EXAMPLE 

package examples.rules.junit;

import junit.framework.TestCase;

public class TCWNT extends TestCase { //VIOLATION- TestCase class with no test
                                      //method
    public void method() { 
    }
}
REPAIR 

Add at least one test method.

package examples.rules.junit;

import junit.framework.TestCase;

public class TCWNTFixed extends TestCase {
    public void testMethod() {	//FIXED
        assertFalse(true);  
    }
}
