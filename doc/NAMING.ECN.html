Ensure class names reflect classes which they extend [NAMING.ECN]



DESCRIPTION 


This rule flags a violation if a class name suggests a specific class is
extended (directly or indirectly), but it is not. For example, a violation will
be flagged if a class name ends with "Exception" but does not extend
"java.lang.Exception". See PARAMETERS.
SINCE 

v8.0
PARAMETERS 

-Check the following types and names:
    Name of class (regular expression)    Extended class (qualified name)
    "^.*Exception$"                       "java.lang.Exception" (default)
    "^.*Throwable$"                       "java.lang.Throwable" (default)

    For example, the first table entry will cause class names which match the
    regular expression "^.*Exception$" (end with "Exception") to flag a
    violation if they do not also extend, directly or indirectly,
    "java.lang.Exception".
BENEFITS 

Fixing this violation can improve readability and prevent confusion.
EXAMPLE 

package examples.rules.naming;

public class ECNException { //VIOLATION
}
REPAIR 

If the type was supposed to extend an Exception type, add the missing 'extends' 
statement to the declaration. Otherwise, change the name of the type so that 
it does not end in 'Exception'.

package examples.rules.naming;

public class ECNFixedException extends Exception { //FIXED
}
