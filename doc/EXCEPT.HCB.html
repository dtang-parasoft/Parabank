Do not hide 'catch' blocks [EXCEPT.HCB]



DESCRIPTION 


This rule looks for hidden 'catch' blocks.  It will flag an error for any 
'catch' block which cannot be reached because all exceptions of the type 
specified in the block will be caught by an earlier 'catch' block or because
the specified type cannot be thrown in the 'try' block.
SCOPE LEVEL 
LINE
SINCE 

v8.0
NOTES 

This rule will not check 'catch' blocks where the exception is of the type 
'java.lang.Exception', 'java.lang.RuntimeException', 'java.lang.Throwable' or 
any subclass of 'java.lang.RuntimeException' to make sure that the exception 
can be thrown in the corresponding 'try' block.  This is because unexpected 
runtime exceptions can be caught by these blocks.  Since it is difficult to 
predict when these exceptions can happen, the rule will not check that these 
exception types can be thrown in the 'try' block.  The rule will, however, check 
that these 'catch' blocks are not hidden by earlier 'catch' blocks.
BENEFITS 

Hidden 'catch' blocks are useless.  They may indicate a misunderstanding of how 
the code in the corresponding 'try' block works, or they may simply be a 
mistake.  In either case, they are confusing and should be removed.
EXAMPLE 

package examples.rules.except;

public class HCB{
    public void test() {
        try {
            throw new java.io.CharConversionException();
        } catch (java.io.CharConversionException e) {
            //some code
        } catch (java.io.IOException e) {  //VIOLATION- 'catch' block is hidden
            //some code
        }
    }
}
REPAIR 

Remove the useless 'catch' block.

package examples.rules.except;

public class HCBFixed {
    public void test() {
        try {
            throw new java.io.CharConversionException();
        } catch (java.io.CharConversionException e) {  //FIXED
            //some code
        }
    }
}
