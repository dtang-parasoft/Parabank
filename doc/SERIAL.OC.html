Ensure outer class is serializable if its inner class is serializable [SERIAL.OC]



DESCRIPTION 


This rule identifies when the non-static inner class is serializable but the
outer class is not serializable. An error is reported for each occurrence.
SINCE 

v8.2
NOTES 

By default, the rule will flag cases only where Serializable is explicitly
implemented. If it is not explicitly implemented then there may be uncertainty
as to whether the class was intended to be used in a Serializable manner or
simply intended to extend or implement a particular interface/class.
PARAMETERS 

-Check inner classes indirectly implementing Serializable (default false)
    Enabling this parameter treats the following as implementing Serializable:
        1) Implementing Serializable
        2) Implementing an interface that extends Serializable
        3) Extending a class that implements Serializable
    With the parameter disabled, only the first option is counted. See NOTES
    for more information.
BENEFITS 

If the non-static inner is serializable and when it is in serialization. It
will attempt to serialize its class and attempt to serialize its outer class
which is associated with it. However, because the outer is non-serializable, it
will cause runtime error.
EXAMPLE 

package examples.rules.serial;

import java.io.*;

public class OC {

    public class Inner implements Serializable {  // VIOLATION
    }
}
REPAIR 

Change inner class to 'static'

package examples.rules.serial;

import java.io.*;

public class OCFixed implements Serializable {

    public static class Inner implements Serializable {  // FIXED
    }

}
