APSC_DV.002290 The application must use the Federal Information Processing Standard (FIPS) 140-2-validated cryptographic modules and random number generator if the application implements encryption, key exchange, digital signature, and hash functionality. [APSC_DV.002290]

RULES

Avoid using insecure cryptographic algorithms for data encryption with Spring [APSC_DV.002290.AISSAJAVA]
Use hash functions with a salt [APSC_DV.002290.MDSALT]
Use 'java.security.SecureRandom' instead of 'java.util.Random' or 'Math.random()' [APSC_DV.002290.SRD]
