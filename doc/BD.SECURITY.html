Security [BD.SECURITY] 

Core rules that identify coding constructs that may
introduce security vulnerabilities into the application.

These rules do not require an optional security license.
RULES

Avoid unvalidated input in array indexes [BD.SECURITY.ARRAY]
Do not expose data wrapped by a buffer to untrusted code [BD.SECURITY.BUFEXP]
Path should be standardized before validation [BD.SECURITY.CANPATH]
Prevent security vulnerability (custom rule) [BD.SECURITY.CUSTOM]
Encapsulate arguments of dangerous methods with a validation method [BD.SECURITY.EACM]
Check floating-point inputs for exceptional values [BD.SECURITY.FPEXC]
Avoid passing unvalidated binary data to log methods [BD.SECURITY.LOG]
Avoid operating on tainted data in privileged blocks [BD.SECURITY.PRIVIL]
Remove temporary files before termination [BD.SECURITY.REMTMP]
Prevent exposure of sensitive data [BD.SECURITY.SENS]
Avoid passing sensitive data to functions that write to log files [BD.SECURITY.SENSLOG]
Do not rely on the default automatic signature verification provided by URLClassLoader and java.util.jar [BD.SECURITY.SIGCLASS]
Safely serialize sensitive data [BD.SECURITY.SSSD]
Validate potentially tainted data before it is used to determine the size of memory allocation [BD.SECURITY.TDALLOC]
Protect against Command injection [BD.SECURITY.TDCMD]
Validate potentially tainted data before it is used in methods that generate code [BD.SECURITY.TDCODE]
Protect against Jakarta Digester injection [BD.SECURITY.TDDIG]
Protect against Environment injection [BD.SECURITY.TDENV]
Protect against File contents injection [BD.SECURITY.TDFILES]
Protect against File names injection [BD.SECURITY.TDFNAMES]
Exclude unsanitized user input from format strings [BD.SECURITY.TDINPUT]
Protect against JXPath injection [BD.SECURITY.TDJXPATH]
Protect against LDAP injection [BD.SECURITY.TDLDAP]
Protect against Library injection [BD.SECURITY.TDLIB]
Protect against log forging [BD.SECURITY.TDLOG]
Protect against network resource injection [BD.SECURITY.TDNET]
Protect against using unprotected credentials [BD.SECURITY.TDPASSWD]
Protect against HTTP response splitting [BD.SECURITY.TDRESP]
Protect against Reflection injection [BD.SECURITY.TDRFL]
Do not store untrusted data in HTTP session [BD.SECURITY.TDSESSION]
Protect against SQL injection [BD.SECURITY.TDSQL]
Protect against XML data injection [BD.SECURITY.TDXML]
Protect against XPath injection [BD.SECURITY.TDXPATH]
Protect against XSS vulnerabilities [BD.SECURITY.TDXSS]
Validate all dangerous data [BD.SECURITY.VPPD]
Validate untrusted XML using schema or DTD before reading [BD.SECURITY.XMLVAL]
