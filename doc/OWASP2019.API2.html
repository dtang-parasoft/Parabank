API2-Broken User Authentication [OWASP2019.API2]

RULES

Avoid hard-coding the arguments to certain methods [OWASP2019.API2.AHCA]
Avoid using insecure cryptographic algorithms for data encryption with Spring [OWASP2019.API2.AISSAJAVA]
Avoid using insecure cryptographic algorithms in Spring XML configurations [OWASP2019.API2.AISSAXML]
Avoid using the javax.crypto.NullCipher class in non-test classes [OWASP2019.API2.AUNC]
Keep all authentication methods centralized to enforce consistency [OWASP2019.API2.CAM]
Avoid using cryptographic keys which are too short [OWASP2019.API2.CKTS]
Clear sensitive data after use [OWASP2019.API2.CSD]
Avoid granting access permission for EJB methods to the 'ANYONE' role [OWASP2019.API2.DPANY]
Avoid using hard-coded cryptographic keys [OWASP2019.API2.HCCK]
Avoid passing hardcoded usernames/passwords/URLs to database connection methods [OWASP2019.API2.HCCS]
Avoid using the 'getRequestedSessionId' method from the 'HttpServletRequest' class [OWASP2019.API2.HGRSI]
Avoid using insecure algorithms for cryptography [OWASP2019.API2.ICA]
Use hash functions with a salt [OWASP2019.API2.MDSALT]
Use the 'minlength' validator for password fields in 'validation.xml' [OWASP2019.API2.MLVP]
Call authentication methods to enforce consistency [OWASP2019.API2.PAC]
Avoid storing usernames and passwords in plain text in Castor 'jdo-conf.xml' files [OWASP2019.API2.PCCF]
Avoid EJB 3 methods without security annotations [OWASP2019.API2.PERMIT]
Password information should not be included in properties file in plaintext [OWASP2019.API2.PLAIN]
Avoid storing sensitive data in plaintext in a cookie [OWASP2019.API2.PLC]
Avoid using plain text passwords in Axis 'wsdd' files [OWASP2019.API2.PTPT]
Mark cookies as HttpOnly [OWASP2019.API2.SCHTTP]
Protect against using unprotected credentials [OWASP2019.API2.TDPASSWD]
Avoid using plain text passwords in Axis2 configuration files [OWASP2019.API2.UTAX]
