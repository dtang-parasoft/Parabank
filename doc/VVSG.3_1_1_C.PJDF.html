Provide Javadoc comments and descriptions for fields [VVSG.3_1_1_C.PJDF]



DESCRIPTION 


This rule identifies fields that lack Javadoc comments
or descriptions within the Javadoc comments.
An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v6.0
NOTES 

The "Quick Fix All" feature is not always available for this rule because some
of the Quick Fixes require user input.  These cases must be Quick Fixed
individually.
PARAMETERS 

-Check if javadoc comment is missing (default as true)
-Check if javadoc description is missing (default as false)
    Javadoc should give a brief description of the commenting field.

-check "public" fields (default as true)
-check "protected" fields (default as false)
-check "package-private" fields (default as false)
-check "private" fields (default as false)

-Ignore fields with names (in regular expression):
    Allows you to provide a list of fields to ignore. If a
    field name matches any of the regular expressions, it will be exempt from
    this rule. For example, if "^serialId$" is added to the list, then fields
    with name "serialID" would not identified as errors-- even if they do not
    have Javadoc.
BENEFITS 

Commenting your code is a good way to document its behavior.
EXAMPLE 

package examples.rules.javadoc;

/** Comments about this class
 */
public class PJDF {

    public int index; // VIOLATION
}
REPAIR 

Provide Javadoc comments for each field.

package examples.rules.javadoc;

/** Comments about this class
 */
public class PJDFFixed {

     /** Comments about this field
      */
    public int index; // FIXED

}
