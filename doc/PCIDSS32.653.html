6.5.3 Insecure cryptographic storage [PCIDSS32.653]

RULES

Avoid using custom MessageDigest implementations [PCIDSS32.653.ACMD]
Avoid using insecure cryptographic algorithms for data encryption with Spring [PCIDSS32.653.AISSAJAVA]
Avoid using insecure cryptographic algorithms in Spring XML configurations [PCIDSS32.653.AISSAXML]
Avoid using the javax.crypto.NullCipher class in non-test classes [PCIDSS32.653.AUNC]
Avoid using cryptographic keys which are too short [PCIDSS32.653.CKTS]
Avoid using insecure algorithms for cryptography [PCIDSS32.653.ICA]
Use hash functions with a salt [PCIDSS32.653.MDSALT]
Password information should not be included in properties file in plaintext [PCIDSS32.653.PLAIN]
Avoid storing sensitive data in plaintext in a cookie [PCIDSS32.653.PLC]
Ensure that passwords are not stored as plaintext and are sufficiently long [PCIDSS32.653.PWDPROP]
Use 'java.security.SecureRandom' instead of 'java.util.Random' or 'Math.random()' [PCIDSS32.653.SRD]
