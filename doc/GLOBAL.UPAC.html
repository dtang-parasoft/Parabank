Avoid globally unused package-private types [GLOBAL.UPAC]



DESCRIPTION 


This rule identifies unused package-private types. An error is reported for each
occurrence.

Global analysis is required in this rule.
SCOPE LEVEL 
CLASS
SINCE 

v4.5
NOTES 

The "Quick Fix All" feature is not available for this rule because the user
should determine whether each class should be removed separately instead of
automatically removing them all.
SECURITY RELEVANCE 

category: Weak Security Controls
category: Code Quality

The unused code might implement security features. Security enhancement will be
missed if the code is not used.
BENEFITS 

These unused entities usually point to either:

1) Old code that is no longer needed.  This makes the class more difficult to
   understand.

2) A logical flaw (if the entity needs to be used, but other classes incorrectly
   avoid using it).
REPAIR 

Remove the unused type or document the reason for its existence.
