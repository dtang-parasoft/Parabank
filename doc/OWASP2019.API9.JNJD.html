Do not insert non-Javadoc comments between Javadoc comments and declarations [OWASP2019.API9.JNJD]



DESCRIPTION 


This rule identifies non-Javadoc comments between Javadoc comments and declarations.
An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v6.0
BENEFITS 

Inserting a non-Javadoc comment on top of a declaration which already has a 
Javadoc comment could be confusing because these non-Javadoc comments could easily be
misinterpreted as a part of the Javadoc comment. However, these comments 
would not be included when auto-generating the API documents.
EXAMPLE 

package examples.rules.javadoc;

public class JNJD {

    /**
     * setter method.
     * @param id1 
     * @author ParaSoft
     */ 
     // Note: .... <---- VIOLATION
    public void setId (String id) {
        _id = id;
    }
    private String _id;
}
REPAIR 

package exmaples.rules.javadoc;

public class JNJDFixed {

    /** 
     * setter method.
     * @param id is set as the new id.
     * @author ParaSoft
     *
     * Note: .... <---- FIXED
     */ 
    public void setId (String id) {
        _id = id;
    }
    private String _id;
}
