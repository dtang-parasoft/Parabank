Place the type that has the same name as the file as the first type [FORMAT.FCN]



DESCRIPTION 



This rule identifies any type(class/interface/enum/annotation) that has the same name as the file,
but is not the first type(class/interface/enum/annotation) in the file.
An error is reported for each occurrence.
SCOPE LEVEL 

FILE
SINCE 

v6.0
NOTES 

Although inner classes could have same name as the file, they are not
reported as errors.
BENEFITS 

It is good practice to make the first type(class/interface/enum/annotation) in the file
be the type(class/interface/enum/annotation) with the same name as the file.
EXAMPLE 

//with file name FCN.java

package examples.rules.format;

class myClass
{}

public class FCN
{}    //VIOLATION
REPAIR 

//with filename FCNFixed.java

package examples.rules.format; 

public class FCNFixed //FIXED
{}

class myClass
{}
