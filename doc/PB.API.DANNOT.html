Add the "@Deprecated" annotation to the declarations of deprecated members [PB.API.DANNOT]



DESCRIPTION 


When using JDK 1.5 or higher, the "@Deprecated" annotation should be used for
deprecated members.  This rule checks for deprecated types, methods, and fields 
which are not declared using the "@Deprecated" annotation.
SCOPE LEVEL 
LINE
SINCE 

v8.0
BENEFITS 

When the "@Deprecated" annotation is used, the compiler can enforce that the
member declared as deprecated is actually deprecated.  For instance, using the
"-Xlint:deprecation" flag, the compiler will issue a warning if a deprecated 
method is overridden when the "@Deprecated" annotation is used.
EXAMPLE 

package examples.rules.pb.api;

public class DANNOT {
    /**
     * @deprecated
     */
    public void deprecatedMethod()  //VIOLATION
    {
    }
}
REPAIR 

Add the "@Deprecated" annotation to the declaration of deprecated members.

package examples.rules.pb.api;

public class DANNOTFixed {
    /**
     * @deprecated
     */
    @Deprecated                     //FIXED
    public void deprecatedMethod()
    {
    }
}
