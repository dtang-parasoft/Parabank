Follow a naming convention for getter methods [NAMING.GETA]



DESCRIPTION 


This rule identifies getter methods whose names do not follow the user-defined
naming convention (as specified in a regular expression).  By default, the rule 
requires that the name of each getter method begin with "get".  An error is 
reported for each occurrence.
SCOPE LEVEL 
METHOD
SINCE 

v4.5
PARAMETERS 

-regular expression
    -Getter method names should match this regular expression.  
    -The default is "^get.", which enforces that the names of getter methods 
     begin with "get".)
-check public method. (default is true)
-check protected method. (default is true)
-check package-private method. (default is true)
-check private method. (default is false)
-Negate regular expression (default is false)
    -If this parameter is enabled, a violation will be flagged if the name of a
     getter method matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for getter method names which do not match the regular expression
     specified.
-Ignore case (default is false)
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of getter 
     methods must match the regular expression specified, including in case.
BENEFITS 

Using the 'get' prefix makes it clear that the method is an accessor that 
returns the value of the field.
DRAWBACKS 

Adding the extra 'get' requires extra typing and makes calling the accessor take 
extra screen space.
EXAMPLE 

package examples.rules.naming;

public class GETA {
    public int method () {  // VIOLATION
        return _count;
    }
    private int _count = 100;
}
REPAIR 

Change the getter method's name to 'get<Field name>'.

package examples.rules.naming;

public class GETAFixed {
    public int getCount () {
        return _count;
    }
    private int _count = 2;
}
