Avoid or enforce usage of '*' form of import statements [VVSG.14_2_I.IMPTD]



DESCRIPTION 


This rule identifies code that uses wild cards (*) in "import" statements.
With the default parameter, an error is reported if the wild card is 
used when importing a class. The rule can also be configured to report
an error if the code uses a wild card other than when importing a class.
SCOPE LEVEL 
LINE
SINCE 

v6.0
PARAMETERS 

-Rule options:
    -Do not use wild cards when importing a class (default option)
    -Use wild cards when importing a class
    -Use customized setting
          -Use wild card import when number of imported types of same package is greater than 20 (default is 20)
          -Use single type import when number of used types of same package is less than 2 (default is 2)
BENEFITS 

Do not use wild cards when importing a class:
If you are not precise about what you are importing, readers might have a difficult
time understanding your classes' context and dependencies.  Some people 
prefer not using "import" at all (thus requiring that every class reference 
is fully qualified), which prevents all possible ambiguity and reduces the source
code changes required if the package names change.

Use wild cards when importing a class:
The import statement allows the use of wildcards when indicating class 
names.  For example, the statement import java.awt.*; makes all 
of the classes in the java.awt package available.  The wildcard does not 
cause code bloat; only the classes you used from the package are compiled 
into the resulting bytecodes.

By using customized settings, you can fine-tune the parameters to eliminate excessive 
single type import statements while preserving single type imports (if they 
are within a reasonable number).
DRAWBACKS 

Do not use wild cards when importing a class:
You need to modify your import statements whenever you start or stop
using an imported class.
EXAMPLE 

package examples.rules.codsta.bp;

import java.io.*;  // VIOLATION 

public class IMPTD {
    void method (InputStream in) {
        if (in == null) return;
    }
}
REPAIR 

package examples.rules.codsta.bp;

import java.io.InputStream;  // FIXED

public class IMPTDFixed {
    void method (InputStream in) {
        if (in == null) return;
    }
}
