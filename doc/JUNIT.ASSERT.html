Include a message string in JUnit assertions [JUNIT.ASSERT]



DESCRIPTION 


This rule identifies JUnit assertions that do not include error message
strings. An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v5.0
NOTES 

This rule will only test classes which meet one of the following conditions:

1. The class extends "junit.framework.TestCase".
2. The class has at least one method with the "org.junit.Test" annotation.  This
only applies to JUnit 4.0 and later.
PARAMETERS 

-Ignore 'assertEquals()' (default is true)
BENEFITS 

When JUnit assertions include error message strings, the results for JUnit test
case failures contain information about the failure.
EXAMPLE 

package examples.rules.junit;

import junit.framework.*;

public class ASSERT extends TestCase {  

    private void testMethod() { // VIOLATION
        fail();
    }
}
REPAIR 

Add an error message string in 'fail()'.

package examples.rules.junit;

import junit.framework.*;

public class ASSERTFixed extends TestCase {

    private void testMethod() { // FIXED 
        fail("fail message here");
    }
}
