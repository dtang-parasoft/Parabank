Do not use fully qualified type names [CODSTA.READ.AFQN]



DESCRIPTION 


Instead of using the fully qualified name for types (the package name and type 
name), just the name of the type should be used.  Using just the simple type 
name will require the type to be imported.  This rule will flag a violation for 
any case where a fully qualified type name is used in a situation in which the 
simple name of the class could have been used instead.
SCOPE LEVEL 
LINE
SINCE 

v8.0
NOTES 

There are some exceptions to the general rule that simple type names should be 
used instead of fully qualified names.  For instance, fully qualified type names
may be necessary to avoid ambiguity when there are two types with the same name
used within a file.  The rule will not flag a violation in cases like this.
PARAMETERS 

-Ignore qualified names in Javadoc (default true)
    Enabling this parameter will ignore qualified names in Javadoc.
    For example, the following would be ignored with this parameter enabled.
    /**
     * {@link java.util.Collection}
     */
    void method() {
    }
BENEFITS 

Using simple type names instead of the fully qualified names for types makes the
code easier to read and less cluttered.
DRAWBACKS 

If you follow this rule, you will need more import statements.  Importing an  
additional type may be unnecessary overhead if the type is used only sparingly.
EXAMPLE 

package examples.rules.codsta.read;

public class AFQN {
    public void createDate() {
        java.util.Date date = new java.util.Date(System.currentTimeMillis());  // 2 VIOLATIONS- The
                                               // fully qualified type name "java.util.Date" is used
                                               // instead of the simple type name "Date".
    }
}
REPAIR 

Import the type and use the type's simple name (just the class name) instead of
its fully qualified name (package and class name)

package examples.rules.codsta.read;

import java.util.Date;

public class AFQNFixed {
    public void method() {
        Date date = new Date(System.currentTimeMillis());  //FIXED
    }
}
