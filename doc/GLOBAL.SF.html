Declare non-private fields "final" if they are read-only [GLOBAL.SF]



DESCRIPTION 


This rule identifies read-only non-private fields that are not declared "final". 
An error is reported for each occurrence.

Global analysis is required in this rule.
SINCE 

v8.4
NOTES 

A non-private field is considered read-only if the field has no write accesses
in the application.
SECURITY RELEVANCE 

category: Code Quality

Declaring a field "final" helps to prevent its value from being unexpectedly
changed.
PARAMETERS 

-Check fields with the following accessibilities:
    -public (Default value is "true")
    -protected (Default value is "true")
    -package-private (Default value is "true")
-Check "static" fields only (Default value is "false")
-Check unused fields (Default value is "false")
-Ignore the following classes: (specify with Glob expressions)
    -Default "org.eclipse.osgi.util.NLS" "*"
EXAMPLE 

package examples.rules.global;

public class SF {

    public static boolean FLAG = true;  // VIOLATION
    
    public void run() {
        if (FLAG) {
            // ....
        }
    }
}
REPAIR 

Make the non-private field "final".

public class SFFixed {

    public static final boolean FLAG = true; // FIXED
    
    public void run() {
        if (FLAG) {
            // ....
        }
    }
}
