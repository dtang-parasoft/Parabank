Do not initialize "static" "final" variables with non "final" "static" variables [INIT.SFA]



DESCRIPTION 


This rule identifies initializations of static final fields with 
static non-final fields. An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v6.0
NOTES 

Even if a non-final field is passed into a method invocation which
returns and assigns value to the final field, it would still be reported
as an error because the return value is likely to be based on a non-
final field.
PARAMETERS 

-Ignore fields of type "String" in initializations. (Default is true.)
    -If this parameter is enabled, a violation will not be flagged for any 
     "String" variables used in the initialization.
BENEFITS 

It could be confusing to assign a non-constant (non-final)
field to a constant (final) field. Constant (final) fields
are supposed to be fed by constant literals to avoid 
confusion.
EXAMPLE 

package examples.rules.init;

public class SFA {
    static int a = 0;
    static final int b = a + 1; //VIOLATION
}
REPAIR 

package examples.rules.init;

public class SFAFixed {
    static int a = 0;
    static final int b = 1; //FIXED
}
