Do not throw exceptions from constructors of "public" non-"final" classes [CERT.OBJ11.EPNFC]



DESCRIPTION 


This rule flags a violation if a constructor declaration of a "public"
non-"final" class throws an exception. An error is reported for each occurrence.
SINCE 

v8.4
SECURITY RELEVANCE 

category: Erratic Application Behavior

If an exception is thrown from a constructor of a "public" non-"final" class an
attacker may be able to access a partially constructed object. One alternative
is to use a flag to signify that an error occurred during initialization and
consult the flag before any action the object takes.
EXAMPLE 

package examples.rules.except;

public class EPNFC {
    int _i;

    EPNFC(int i) throws Exception {
        if (i == 0) {
            throw new Exception();
        }
        _i = i;
    }
}
REPAIR 

Avoid throwing an exception from the constructor. If the exception is to declare
an error occurred it may be possible to instead set a flag which is checked at
necessary stages.

package examples.rules.except;

public class EPNFCFixed {
    int _i;
    volatile boolean _isValid = false;

    EPNFC(int i) throws Exception {
        if (i != 0) {
            _i = i;
            isValid = true;
        }
    }
}
