Declare package-private fields as inaccessible as possible [CERT.OBJ51.DPAF]



DESCRIPTION 


This rule identifies package-private fields that are not as inaccessible as
possible. An error is reported for each occurrence.

Global analysis is required in this rule.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-Ignore "final" fields (default is true)
 "final" modifier could imply the declared field is a constant
 for later use. Such a constant should be ignored.
-Ignore fields declared in "abstract" classes (default is false)
BENEFITS 

Following this rule makes the code more object-oriented and makes its
dependencies easier to understand.
REPAIR 

Change the field's accessibility or document the reason for the
excessive accessibility.
