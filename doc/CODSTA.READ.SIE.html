Use 'isEmpty()' for Collections and Maps instead of comparing 'size()' to 0 [CODSTA.READ.SIE]



DESCRIPTION 


This rule identifies cases where a Collection or Map calls 'size()' and compares
the result to '0'.  These calls can generally be replaced with a call to
'isEmpty()'. See NOTES and BENEFITS for more information.
SINCE 

v9.0
NOTES 

Assuming a List 'list', this rule checks the following comparisons:
    1) list.size() != 0, 0 != list.size()
    2) list.size() == 0,  0 == list.size()
    3) list.size() > 0
    4) 0 < list.size()
BENEFITS 

Using 'Collection.isEmpty()' instead of 'Collection.size() == 0' may improve
readability as the method name 'isEmpty' provides information about the check
that is being performed. Also, comparing 'size()' to '0' is just repeating
functionality that is already present in the 'isEmpty()' method.
EXAMPLE 

package examples.rules.codsta.read;

import java.util.List;

public class SIE {
    public int sum(List<Integer> list) {
        int sum = 0;
        if (list.size() > 0) { //VIOLATION
            for (Integer integer : list) {
                sum += integer;
            }
        }
        return sum; 
    }
}
REPAIR 

package examples.rules.codsta.read;

import java.util.List;

public class SIEFixed {
    public int sum(List<Integer> list) {
        int sum = 0;
        if (!list.isEmpty()) { //FIXED
            for (Integer integer : list) {
                sum += integer;
            }
        }
        return sum; 
    }
}
