Do not declare SerialVersionUID in classes that do not implement Serializable [SERIAL.SRLZ]



DESCRIPTION 


Reports error if serialVersionUID is declared, but the class is not Serializable
SINCE 

v9.0
NOTES 

A given class will be serializable if it implements serializable itself, or if
a parent class or a class that the given class extends is serializable.
BENEFITS 

The serialVersionUID field being declared indicates that the user intended to 
create a serializable class, which they failed to do.  This rule alerts them to
the fact.
EXAMPLE 

package examples.rules.SERIAL;

import java.io.Serializable;

public class SRLZ_2 implements SRLZ_Interface{ //VIOLATION
    private static final long serialVersionUID = 13;

    public int compareTo(Object o) {
        return 0;
    }

    private class SRLZ_3 implements Serializable{
        private static final long serialVersionUID = 13;
    }
}

interface SRLZ_Interface extends Comparable { //VIOLATION
    public static final long serialVersionUID = 13;
}
REPAIR 

EitherSerializable or remove the reference to serialVersionUID

package examples.rules.SERIAL;

import java.io.Serializable;

public class SRLZ_2 implements SRLZ_Interface, Serializable { //FIXED
    private static final long serialVersionUID = 13;

    public int compareTo(Object o) {
        return 0;
    }

    private class SRLZ_3 implements Serializable{
        private static final long serialVersionUID = 13;
    }
}

interface SRLZ_Interface extends Comparable { //FIXED
}
