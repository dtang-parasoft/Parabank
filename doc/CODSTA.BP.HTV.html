Avoid using 'Hashtable' and 'Vector' [CODSTA.BP.HTV]



DESCRIPTION 


This rule identifies instantiations of 'java.util.Hashtable' and
'java.util.Vector'. A violation is reported for each occurrence.
SINCE 

v9.0
PARAMETERS 

-Check Hashtable (default true)
-Check Vector (default true)
BENEFITS 

Hashtable and Vector provide synchronization while other Collections and Maps
do not. This can lead to confusion as to whether or not synchronization was
intended and required. In cases where synchronization is not required, HashMap
or ArrayList usually be used as a drop-in replacement. Unsynchronized
Collections or Maps can also improve performance. In cases where synchronization
is required, the 'java.util.Collections#synchronizedXXX()' methods can be used
to wrap a Collection or Map. A call to one of these methods has the benefit of
explicitly stating in the code that any synchronization is intended.
EXAMPLE 

package examples.rules.codsta.bp;

import java.util.*;

public class HTV {
    private Hashtable _table;

    public HTV() {
        _table = new HashTable(); //VIOLATION
        _table.put("Hello","World");
    }
    ...
}
REPAIR 

package examples.rules.codsta.bp;

import java.util.*;

public class HTVFixed {
    private Map _table;

    public HTVFixed() {
        _table = new HashMap(); //FIXED
        _table.put("Hello","World");
    }
    ...    
}
