Ensure overloaded constructors and methods share the same accessibility [CODSTA.BP.OCMA]



DESCRIPTION 


This rule identifies overloaded constructors and methods of a class with
different accessibilities.  Each method that is overloaded will be flagged.
SINCE 

v8.3
PARAMETERS 
-Check constructors (default true)
-Check methods (default true)
BENEFITS 

Inconsistent accessibilities is usually unintended. In some cases an overloaded
method/constructor may be called such that the code is able to compile.
However, at  runtime an overloaded method/constructor with different
accessibility may be called instead, which could lead to an error.
EXAMPLE 

package examples.rules.codsta.bp;

class OCMA {
    public OCMA(){};
    protected OCMA(int i){}; //VIOLATION

    protected void method1(Object o){};
    private void method1(String s){}; //VIOLATION
}
REPAIR 

Change the method/constructor with differing visibility to match.

package examples.rules.codsta.bp;

class OCMAFixed {
    public OCMAFixed(){};
    public OCMAFixed(int i){}; //FIXED
    
    protected void method1(Object o){};
    protected void method1(String s){}; //FIXED
}
