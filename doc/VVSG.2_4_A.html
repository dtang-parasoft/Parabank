2.4-A: Modularity [VVSG.2_4_A]

RULES

Avoid constructors and overriding methods which are more accessible than those of their super classes [VVSG.2_4_A.AMA]
Keep all access control methods centralized to enforce consistency [VVSG.2_4_A.CACM]
Declare "package-private" types as inaccessible as possible [VVSG.2_4_A.DPAC]
Declare package-private fields as inaccessible as possible [VVSG.2_4_A.DPAF]
Declare package-private methods as inaccessible as possible [VVSG.2_4_A.DPAM]
Declare "public/protected" types as inaccessible as possible [VVSG.2_4_A.DPPC]
Declare "public/protected" fields as inaccessible as possible [VVSG.2_4_A.DPPF]
Declare "public/protected" methods as inaccessible as possible [VVSG.2_4_A.DPPM]
Do not use "protected" access for members of "final" classes [VVSG.2_4_A.PIFC]
Declare a package-private method "final" if it is not overridden [VVSG.2_4_A.SPAM]
Use a Session Facade to manage access to entity beans [VVSG.2_4_A.USF]
