Ensure 'public' classes have at least one 'public' or 'protected' member [OOP.PMPC]



DESCRIPTION 


This rule identifies any "public" class that does not contain a
"public" or "protected" member. An error is reported for each
occurrence.
SCOPE LEVEL 
LINE
SINCE 

v7.5

EXAMPLES

package examples.rules.oop;

public class PMPC {  // VIOLATION

    PMPC() {
    }

    String getClassName() {
        return getClass().getName();
    }
}
BENEFITS 

If a class does not contain a "public" or "protected" member,
there is no reason to make the class "public".  Since none of 
the members are accessible to the outside world or to 
subclasses, there is no point in making the class "public".  
When this happens, the programmer probably meant to give the 
class a more restrictive access level or to make one of the 
members "public" or "protected".
REPAIR 

Change the class to "package-private" (the default accessibility).

package examples.rules.oop;

class PMPCFixed {  // FIXED

    PMPCFixed() {  
    }

    String getClassName() {  
        return getClass().getName();
    }
}
