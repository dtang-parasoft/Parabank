Avoid exporting "internal" packages in MANIFEST.MF [ECLIPSE.IPMF]



DESCRIPTION 


This rule identifies "internal" packages that are exported in MANIFEST.MF.
An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v8.1
BENEFITS 

The Export-Package manifest attribute in an Eclipse plug-in specifies which packages of Java code
are accessible at runtime from other plug-ins.  It applies to Eclipse version 3.1 and later.

Packages with "internal" in the package name should not be listed in the Export-Package section of
Eclipse plugin MANIFEST.MF files unless they have the ";x-internal:=true" or ";x-friends:=..."
suffix.
EXAMPLE 

Export-Package: com.acme.eclipse.plugin,
 com.acme.eclipse.plugin.internal, # VIOLATION An internal package is exported
 com.acme.eclipse.plugin.internal.constants # VIOLATION An internal package is exported
REPAIR 

Export-Package: com.acme.eclipse.plugin,
 com.acme.eclipse.plugin.internal;x-internal:=true,
 com.acme.eclipse.plugin.constants;x-friends:="com.acme.eclipse.friend.plugin"
