Javadoc Comments [JAVADOC] 

Rules related to Javadoc comments.
RULES

Avoid unsupported '@' tags and other tags that should not be used in Javadoc comments [JAVADOC.BT]
Always include a description of whether or not a method can return null in the Javadoc [JAVADOC.CRN]
Avoid unused Javadoc tags [JAVADOC.DPMT]
Enforce custom Javadoc tags for fields [JAVADOC.ECTF]
Enforce custom Javadoc tags for methods/annotation member types [JAVADOC.ECTM]
Enforce custom Javadoc tags for types [JAVADOC.ECTT]
Do not insert non-Javadoc comments between Javadoc comments and declarations [JAVADOC.JNJD]
Use the '@author' Javadoc tag in declaration Javadoc comments [JAVADOC.MAJDT]
Include a meaningful description in Javadoc tags [JAVADOC.MDJT]
Avoid misformatted Javadoc tags [JAVADOC.MISFORMAT]
Use the '@return' Javadoc tag in method Javadoc comments [JAVADOC.MRDC]
Use the '@version' tag in type Javadoc comments [JAVADOC.MVJDT]
Order Javadoc tags appropriately [JAVADOC.ORDER]
Use the '@param' Javadoc tag for each parameter of methods [JAVADOC.PARAM]
Provide Javadoc comments and descriptions for types [JAVADOC.PJDC]
Provide Javadoc comments and descriptions for fields [JAVADOC.PJDF]
Provide Javadoc comments and descriptions for methods [JAVADOC.PJDM]
Enforce restraint on number of lines used for Javadoc comments [JAVADOC.SINGLE]
Use the '@concurrency' Javadoc tag on "synchronized" methods and blocks [JAVADOC.SMJT]
Avoid misspelling words in Javadoc comments and string literals [JAVADOC.SPELL]
Specify 'RUNTIME' as the retention policy when using the '@Documented' annotation [JAVADOC.SRRP]
Use the '@throws' or '@exception' Javadoc tag in methods [JAVADOC.THROW]
Provide a Javadoc comment for 'toString()' methods [JAVADOC.TSMJT]
Ensure that certain Javadoc tags only span one line [JAVADOC.TSOL]
Avoid using the '@return' Javadoc tag on "void" methods [JAVADOC.VMCR]
