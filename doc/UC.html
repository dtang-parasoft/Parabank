Unused Code [UC] 

Rules that check for unused code.
RULES

Avoid unnecessary modifiers in an "interface" [UC.AAI]
Remove commented out Java code [UC.ACC]
Ensure that classes do not explicitly extend 'java.lang.Object' [UC.AEEO]
Avoid empty static and non-static initializers [UC.AESTAT]
Do not override "abstract" methods of a parent class with "abstract" methods [UC.AMAMI]
Avoid redundant throw clauses [UC.ARTD]
Avoid unused labels [UC.AUL]
Avoid collection objects that are never read [UC.AURCO]
Avoid local variables that are never read [UC.AURV]
Avoid unnecessary "boolean" comparisons [UC.BCMP]
Avoid duplicate code in 'if' branches [UC.DIEB]
Do not explicitly "import" the java.lang.* "package" [UC.DIL]
Avoid empty "finalize()" methods [UC.EF]
Avoid redundant 'finalize()' methods which only call the superclass' 'finalize()' method [UC.FCSF]
Avoid redundant "final" keywords in method declarations in "final" classes [UC.FMFC]
Avoid unused "private" fields [UC.PF]
Do not import classes from the package that contains the current class [UC.PIMPORT]
Avoid unused "private" methods [UC.PM]
Avoid redundant 'static' keywords in enum type declarations [UC.RSKE]
Avoid empty "synchronized" statements [UC.SNE]
Avoid methods that only call the overridden implementation (superclass implementation) [UC.SO]
Use a caught exception in the "catch" block [UC.UCATCH]
Avoid unnecessary 'if' statements [UC.UCIF]
Avoid unnecessary "else" statements [UC.UES]
Avoid unused "import" statements [UC.UIMPORT]
Avoid unused parameters [UC.UP]
Avoid unused "private" classes or interfaces [UC.UPC]
Avoid unnecessary "return" statement at the end of "void" methods [UC.VR]
