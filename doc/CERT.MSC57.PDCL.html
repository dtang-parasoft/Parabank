Place "default" as the last case of the "switch" statement [CERT.MSC57.PDCL]



DESCRIPTION 


This rule identifies any "default" case that is not the last case of
the "switch" statement. An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v5.1
BENEFITS 

It is a best practice for the "default" case to be the last case of the 
"switch" statement. If the "default" case is not the last case, you must 
ensure that the "default" case has a "break" statement. Otherwise, it will
fall through to the next case.
EXAMPLE 

package examples.rules.pb;

public class PDCL {

    public void method() {
        switch(3) {
        case 1 : System.out.println("case 1");
        default : System.out.println("default case");  // VIOLATION
        case 2 : System.out.println("case 2");
        }
        // output: 
        // default case
        // case 2
    }
}
REPAIR 

Place the "default" case last.

public class PDCLFixed {

    public void method() {
        switch(3) {
        case 1 : System.out.println("case 1");
        case 2 : System.out.println("case 2");
        default : System.out.println("default case");  // FIXED
        }
        // output:
        // default case
    }
}
