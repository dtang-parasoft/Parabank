Ensure all types have a non default package name [CODSTA.ORG.UNDPN]



DESCRIPTION 


This rule identifies types(classes, interfaces, enum, annotation) that do not have
a non default package name. An error is reported at the beginning
of the file if the type inside that file belong to the 
default package.
SCOPE LEVEL 

FILE
SINCE 

v6.0
BENEFITS 

As the number of files (classes) scales up, it is easier to 
manage files (classes) if they are assigned to packages in
an organized way. Enforcing this rule could prevent files
(classes) from dangling in a large file system without a 
categorized identity.
EXAMPLE 

//VIOLATION: no package statement

public class UNDPN {
    //...
}
REPAIR 

package examples.rules.codsta.org; //FIXED

public class UNDPNFixed {
    //...
}
