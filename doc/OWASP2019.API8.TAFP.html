Specify a valid 'type' attribute for each '<form-property>' element in 'struts-config.xml' files [OWASP2019.API8.TAFP]



DESCRIPTION 


'<form-property>' elements for form beans in 'struts-config.xml' files must 
specify a 'type' attribute which is a valid Java type name.  If the 'type' 
attribute is missing or not a valid Java type name, the input checking will not
work correctly.  This rule will flag a violation for '<form-property>' elements
in 'struts-config.xml' files which contain any of the following errors:

1.  The 'type' attribute is missing.
2.  The value of the 'type' attribute is empty or contains whitespace 
characters.
3.  The 'type' attribute is not a valid Java type name (e.g., The attribute 
value contains a character which is not legal in a Java identifier.)

A violation will be flagged for each occurrence.
SINCE 

v8.4
SECURITY RELEVANCE 

category: Unsafe Environment Configuration

'<form-property>' elements are important because they perform validation to help
prevent denial of service attacks and injection attacks.  If a correct 'type' 
attribute is not specified for these elements, the validation will not work.
EXAMPLE 

This sample 'struts-config.xml' file contains 4 '<form-property>' elements which
are either missing a 'type' attribute or contain an invalid 'type' attribute.

struts-config.xml:

<?xml version="1.0" encoding="ISO-8859-1" ?>
<struts-config>

    <form-beans>
    
        <form-bean name="loginForm" type="org.apache.struts.validator.DynaValidatorForm">
            <form-property name="name_login" type="*" />  <!-- VIOLATION: "*" is not a valid type name -->
            <form-property name="password_login" type="      " />  <!-- VIOLATION: The 'type' attribute is empty -->
        </form-bean>
        
        <form-bean name="colorForm" type="org.apache.struts.action.DynaActionForm">                           
            <form-property name="color" type="" />  <!-- VIOLATION: The 'type' attribute is empty -->
            <form-property name="hue" />  <!-- VIOLATION: Missing 'type' attribute -->
        </form-bean>
    
    </form-beans>

</struts-config>
REPAIR 

Specify a fully qualified Java type name for the 'type' attribute of each 
'<form-property>' element.

struts-config.xml:
    
<?xml version="1.0" encoding="ISO-8859-1" ?>
<!DOCTYPE struts-config PUBLIC
      "-//Apache Software Foundation//DTD Struts Configuration 1.3//EN"
      "http://struts.apache.org/dtds/struts-config_1_3.dtd">
<struts-config>

    <form-beans>
    
        <form-bean name="loginForm" type="org.apache.struts.validator.DynaValidatorForm">
            <form-property name="name_login" type="java.lang.String" />
            <form-property name="password_login" type="core.EncryptedPassword" />
        </form-bean>
        
        <form-bean name="colorForm" type="org.apache.struts.action.DynaActionForm">                           
            <form-property name="color" type="java.lang.String" />
            <form-property name="hue" type="java.lang.String" />
        </form-bean>
    
    </form-beans>

</struts-config>
