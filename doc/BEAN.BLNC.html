Use appropriate signatures for listener method names in JavaBean classes [BEAN.BLNC]



DESCRIPTION 


This rule identifies Bean class listener method names with inappropriate signatures. 
An error is reported if the listener method in a JavaBean class does not follow
the following pattern:

    public void addTYPE(TYPE listener) {
    }
    public void removeTYPE(TYPE listener) {
    }
SCOPE LEVEL 
METHOD
SINCE 

v5.0
NOTES 

The rule defines a JavaBean class as a class which implements Serializable, 
contains a public default constructor, and has a name ending with "Bean".
BENEFITS 

The listener methods in JavaBeans classes are invoked through reflection.  Thus,
if the methods are not named according to the expected naming convention, they
may never get invoked.
EXAMPLE 

package examples.rules.bean;

import java.io.*;
import java.awt.event.*;

public class BLNCBean implements Serializable {

    public void addactionListener(ActionListener listener) {  // VIOLATION

    }

    public void removeactionListener(ActionListener listener) {  // VIOLATION

    }
}
REPAIR 

Rename the listener methods.

package examples.rules.bean;

import java.io.*;
import java.awt.event.*;

public class BLNCFixedBean implements Serializable {

    public void addActionListener(ActionListener listener) {  // FIXED 

    }

    public void removeActionListener(ActionListener listener) {  // FIXED 

    }
}
