Use ''local'' property attribute in local scope [SPRING.LOCAL]



DESCRIPTION 


Spring determines the range of references. This rule flags each reference to a bean which is located in the same file 
and is not marked with a local attribute.
SINCE 

v9.2
NOTES 

This method is recommended in the official documentation for all Spring Framework versions.
PARAMETERS 
-Check p-namespace
    -this option causes the rule to check the p-namespace property; it is disabled by default

-Check shortcut property form
    -this option causes the rule to check the shortcut form property; it is disabled by default
BENEFITS 

Using the ''local'' parameter accelerates file parsing by Spring and makes the XML file clearer.
EXAMPLE 

long form:

  <property name="targetName">
      <ref bean="theTargetBean" />
  </property>
  
short form:

    <property name="targetName" ref="theTargetBean" />
    
p-namespace form:

    <bean name="beanName" p:targetName-ref="theTargetBean" />
REPAIR 

Change bean to local:

  <property name="targetName">
      <ref local="theTargetBean" />
  </property>

The short and p-namespace forms do not have local scope. In this case, you must convert to long form.
