Do not use "Runtime.exec()" [CERT.IDS07.EXEC]



DESCRIPTION 


This rule identifies the use of 'Runtime.exec()'. An error is reported for
each occurrence.
SINCE 

v4.5
SECURITY RELEVANCE 

category: Erratic Application Behavior

Use of non-portable commands may cause unexpected behavior and have unintended
security implications. See BENEFITS for more information.
BENEFITS 

Calling the 'Runtime.exec()' method to spawn a process and execute an external
command may not be portable because there is no guarantee how the native
OS command will behave on different platforms.
EXAMPLE 

package examples.rules.port;

import java.io.IOException;

public class EXEC {
    public void method(String command) {
        try {
            Runtime.getRuntime().exec(command);      // violation, not portable
        } catch (IOException io) {
        }
    }
}
