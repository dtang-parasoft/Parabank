Declare arrays with '[]' brackets after the array type and before the variable name(s) [FORMAT.IAD]



DESCRIPTION 


This rule identifies array declarations that do not have '[]' brackets which after
the array type name and before the array variable name. An error is reported 
for each occurrence.

This rule does not apply to method declarations for methods that return an
array type (for example, 'int method()[]' will not be flagged).
SCOPE LEVEL 
LINE
SINCE 

v4.5
BENEFITS 

If all team members follow the same formatting style, it will be easier for each
team member to read and understand the code.
EXAMPLE 

package examples.rules.format;

public class IAD {
    private int attr [];  // VIOLATION
    private String str1 [], str2 [], str3 [];  // VIOLATION
}
REPAIR 

Move the brackets to the proper position.

package examples.rules.format;

public class IADFixed {
    private int [] attr;  // FIXED
    private String [] str1, str2, str3;  // FIXED
}
