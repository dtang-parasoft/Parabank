Use a naming convention for type parameters [NAMING.NPH]



DESCRIPTION 


This rule identifies type parameters in type declarations that do not follow the 
user-defined naming convention (as specified in a regular expression).  The 
default naming convention is that all placeholder names must consist of a single 
upper-case letter.  The regular expression used to express this is "^[A-Z]$".
SCOPE LEVEL 
LINE
SINCE 

v8.0
PARAMETERS 

-regular expression
    -Type parameter names should match the specified regular expression.  
    -The default is "^[A-Z]$".  This enforces that the names of type parameters
     consist of a single upper-case letter.
-Negate regular expression
    -If this parameter is enabled, a violation will be flagged if the name of a
     type parameter matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for type parameter names which do not match the regular expression
     specified.
-Ignore case
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of type 
     parameters must match the regular expression specified, including in case.
BENEFITS 

This rule helps enforce a naming convention.
EXAMPLE 

package examples.rules.naming;

public class NPH<PARAM> { //VIOLATION 
}
REPAIR 

package examples.rules.naming;

public class NPHFixed<P> { //FIXED 
}
