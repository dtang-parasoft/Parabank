DEPRECATED: Cyclomatic Complexity [METRICS.VG]



DESCRIPTION 


Rules from this category are deprecated - see METRIC category.

Measures the 'cyclomatic complexity' of a method.  Cyclomatic complexity
represents the number of "while", "for", "if" and "switch" statements in a
method.
NOTES 

This rule does not test "abstract", "native", and any methods that are
declared in the interface.

Jtest will report an error if the value is greater or equals 30.
BENEFITS 

Studies have found that methods with cyclomatic complexity higher than 10 are
error-prone.
REFERENCES 

Robert B. Grady: "Practical Software Metrics For Project Management and 
Process Improvement" Prentice Hall P T R, 1992.  pp.16 - 18
