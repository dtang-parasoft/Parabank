Avoid declaring "transient" fields in non-serializable classes [CWE.212.FT]



DESCRIPTION 



This rule identifies that "transient" fields are declared in a non-serializable 
class. An error is reported for each occurrence.
SINCE 

v8.2
BENEFITS 

It is either a typo which forget to implement 'java.io.Serializable' or 
this "transient" field declaration will only make the code more confusing 
and no other effect.
EXAMPLE 

package examples.rules.serial;

public class FT {

    private transient String field1; // VIOLATION
    private String field2;
}
REPAIR 

Implements 'java.io.Serializable'

package examples.rules.serial;

import java.io.*;

public class FTFixed implements Serializable {  // FIXED

    private transient String field1;
    private String field2;
}
