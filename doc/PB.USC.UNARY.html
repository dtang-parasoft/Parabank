Do not use the unary operator '+' [PB.USC.UNARY]



DESCRIPTION 


This rule identifies usages of the unary operator '+'. An error
is reported for each occurrence.
SINCE 

v8.0
BENEFITS 

The unary operator '+' can be omitted as any value without the 
unary operator '+' would be implicitly recognized as 
positive. Besides, the unary operator '+' could sometimes 
indicate possible bugs that could not be detected at the
compiler level. For example, value += 1 might accidentally
be written as value =+ 1, which is syntactically correct, 
but has completely different semantics.
EXAMPLE 

package examples.rules.pb.usc;

public class UNARY {
    public int incrementByTwo(int i) {
        return i=+ 2; //VIOLATION
    }
}
REPAIR 

package examples.rules.pb.usc;

public class UNARYFixed {
    public int incrementByTwo(int i) {
        return i+= 2; //FIXED
    }
}
