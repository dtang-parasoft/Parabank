Avoid unnecessary 'if' statements [CWE.571.UCIF]



DESCRIPTION 


This rule identifies unnecessary 'if' statements. An error is reported for each
occurrence.

Any 'if' statement that does not contain a field, local variable or method
invocation in its condition is considered to be unnecessary as the body would
either be executed all the time or not at all. Also, in  some situations, using 
flags or constants would easily solve this problem without changing the program
functionality.
SINCE 

V6.0
SECURITY RELEVANCE 

"This weakness could be an indication of a bug in the program. . . and is
an indication of poor quality. This could lead to further bugs and
the introduction of weaknesses." [CWE-563:]
BENEFITS 

If possible, unnecessary statements should be eliminated in order to decrease
program complexity. If an 'if' statement is needed (for testing etc.), using
flags or constants might be a better solution.
EXAMPLE 

package examples.rules.uc;

public class UCIF {
    public class myMethod() {
        if (true) //flag
            System.out.println("Hello"); 
    }
}
REPAIR 

package examples.rules.uc;

public class UCIFFixed {
    public class myMethod() {
        //FIXED
        System.out.println("Hello"); 
    }
}
