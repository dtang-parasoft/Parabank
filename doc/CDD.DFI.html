Avoid duplicated field initialization in constructors [CDD.DFI]



DESCRIPTION 


This rule identifies repeated field initialization in constructors. 
An error is reported for each occurrence.
SINCE 

v8.2
BENEFITS 

If the field is initialized the same value in different constructors,
it will increase the cost of code maintenance because every time you change
the initial value of the field, you need to change it in multiple places. Reducing 
or eliminating the amount of duplicated field initialization can prevent this problem.
EXAMPLE 

package examples.rules.cdd;

public class DFI {
    private int max;
    private int num;
    
    public DFI() {
        max = 100; // VIOLATION
        num = 10;
        
    }
    
    public DFI(int n) {
        max = 100; // VIOLATION
        num = n;
    }
}
REPAIR 

package examples.rules.misc;

public class DFIFixed {
    private int max = 100;  // FIXED
    private int num;
    
    public DFI() {
        num = 10;
        
    }
    
    public DFI(int n) {
        num = n;
    }
}
