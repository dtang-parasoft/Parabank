Use 'equalsIgnoreCase()' instead of calls to 'toLowerCase()' or 'toUpperCase()' [OPT.EIC]



DESCRIPTION 


This rule will flag cases where 'equals()' is called following a call to
'toLowerCase()' or 'toUpperCase()'. See NOTES and BENEFITS
SINCE 

v9.0
NOTES 

This rule checks chained method calls as well as cases where the return value of
'toLowerCase()' or 'toUpperCase()' is stored and then used in a call to
'equals()'. If the return value of 'toLowerCase()' or 'toUpperCase()' is stored
to a field or used elsewhere then a violation will not be flagged.
BENEFITS 

Mobile platforms have reduced resources, optimizations are especially important. 

It is more efficient to call 'equalsIgnoreCase()' instead of converting a String
to lowercase or uppercase and then calling 'equals()'.
EXAMPLE 

package examples.rules.opt;

public class EIC {
    boolean isMatch(String s, String t)
    {
        return s.toLowerCase().equals(t.toLowerCase()); //VIOLATION 
    }
}
REPAIR 

Use a call to 'equalsIgnoreCase()' instead.

package examples.rules.opt;

public class EICFixed {
    boolean isMatch(String s, String t)
    {
        return s.equalsIgnoreCase(t); //FIXED
    }
}
