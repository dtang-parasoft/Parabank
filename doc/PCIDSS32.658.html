6.5.8 Improper access control [PCIDSS32.658]

RULES

Avoid using "SELECT *" in SQL queries [PCIDSS32.658.AUSS]
Keep all authentication methods centralized to enforce consistency [PCIDSS32.658.CAM]
Avoid granting access permission for EJB methods to the 'ANYONE' role [PCIDSS32.658.DPANY]
Avoid defining multiple security roles with the same name in 'web.xml' files [PCIDSS32.658.DSR]
Include an appropriate '<login-config>' element to specify the type of authentication to be performed in 'web.xml' files [PCIDSS32.658.LCA]
Use the 'minlength' validator for password fields in 'validation.xml' [PCIDSS32.658.MLVP]
Ensure sufficient protection against multiple failed authentication attempts [PCIDSS32.658.PBFA]
Avoid EJB 3 methods without security annotations [PCIDSS32.658.PERMIT]
Ensure that each security role referenced in a 'web.xml' file has a corresponding definition [PCIDSS32.658.SRCD]
Ensure that an appropriate security manager is set [PCIDSS32.658.SSM]
Protect against File names injection [PCIDSS32.658.TDFNAMES]
