Enforce number of space character(s) on each side of a logical operator [FORMAT.SLOP]



DESCRIPTION 

    
This rule identifies logical operators that are not preceded and followed
by the required number of spaces. It applies to "&&" and "||". With the default 
parameters, an error is reported if a logical operator is not preceded and followed
by one space.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-Number of spaces before logical operators (default value is 1)
-Number of spaces after logical operators (default value is 1)
BENEFITS 

If all team members follow the same formatting style, it will be easier for each
team member to read and understand the code.
EXAMPLE 

package examples.rules.format;

public class SLOP {
    public void method (boolean x, boolean y) {
        if (x|| y) { }  // VIOLATION
        if (x &&y) { }  // VIOLATION
    }
}
REPAIR 

Place a single space before and after the operator.

package examples.rules.format;

public class SLOPFixed {
    public void method (boolean x, boolean y) {
        if (x || y) { }  // FIXED
        if (x && y) { }  // FIXED
    }
}
