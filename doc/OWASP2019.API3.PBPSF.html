Declare subclasses of 'Permission' and 'BasicPermission' "final" [OWASP2019.API3.PBPSF]



DESCRIPTION 


This rule flags a violation if a class is non-"final" and satisfies one of the
following:
    1. The class extends "java.security.BasicPermission"
    2. The class extends "java.security.Permission"
SINCE 

v8.4
SECURITY RELEVANCE 

category: Weak Security Controls

"Permission" classes are used to interact with sensitive data and operations.
A subclass should be declared "final" to prevent an attacker from creating a
subclass and gaining access to sensitive information.
PARAMETERS 

-Check subclasses of "java.security.Permission" (default on)
-Check subclasses of "java.security.BasicPermission" (default on)
-Ignore subclasses which are enclosed by a "final" outer class (default on)
EXAMPLE 

package examples.rules.security.wsc;

import java.security.BasicPermission;

public class PBPSF extends BasicPermission { //VIOLATION
    public PBPSF(String name) {
        super(name);
    }
    ...
}
REPAIR 

package examples.rules.security.wsc;

import java.security.BasicPermission;

public final class PBPSFFixed extends BasicPermission { //FIXED
    public PBPSFFixed(String name) {
        super(name);
    }
    ...
}
