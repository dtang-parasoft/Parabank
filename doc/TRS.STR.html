Do not perform synchronization nor call semaphore methods on an Object's 'this' reference [TRS.STR]



DESCRIPTION 


The "this" reference should not be used as the synchronizing expression in a 
"synchronized" statement.  Also, the semaphore methods "wait()", "notify()", and 
"notifyAll()" should not be called on a class' "this" reference.  This rule will 
flag a violation for any place where the "this" reference is used in either of 
these situations.
SCOPE LEVEL 
LINE
SINCE 

v8.0
SECURITY RELEVANCE 

"Each thread of execution will "hang" and prevent tasks from completing. In
some cases, CPU consumption may occur if a lock check occurs in a tight loop."
[CWE-833]
PARAMETERS 

-Check synchronize on 'this' (default on)
-Check 'wait()', 'notify()', and 'notifyAll()' calls on 'this' (default on)
BENEFITS 

Client classes that use a class may also use an instance of the class as a 
synchronizing object.  If this happens, two classes are using the same object
for synchronization.  This makes multithreaded correctness suspect.
EXAMPLE 

package examples.rules.trs;

public class STR{
    public void method(){
        synchronized(this){  //VIOLATION
            try{
                this.wait();  //VIOLATION
            }
            catch(Exception e){
            }
        }
    }
}
REPAIR 

Use a private member variable instead of the "this" reference to control 
synchronization.

package examples.rules.trs;

public class STRFixed{
    STRFixed syncObj;
    
    public void method(){
        synchronized(syncObj){  //FIXED
            try{
                syncObj.wait();  //FIXED
            }
            catch(Exception e){
            }
        }
    }
}
