Do not declare multiple "@Timeout" methods [EJB.EJB3.MTM]



DESCRIPTION 


This rule identifies Stateless and MessageDriven beans which specify more than
one "@Timeout" method. Methods are treated as a "@Timeout" method if they are
of the form "void <METHOD> (Timer timer)" and satisfy one of the following:
    1) The class implements "javax.ejb.TimedObject" and the method name is
       "ejbTimeout".
    2) The method is listed as "timeout-method" in the deployment descriptor.
    3) The method declaration has the "javax.ejb.Timeout" annotation.
SINCE 

v8.4
PARAMETERS 

Check the following:
    -Check annotations
    -Check annotations and deployment descriptors (default)
        Check deployment descriptors in the following location:
            -Check project (default)
            -Check package
EXAMPLE 

package examples.rules.ejb.ejb3;

@Stateless
public class MTM implements MTMInterface { //VIOLATION
    @Timeout
    void timeoutMethod(Timer timer) {
        method1();
    }

    @Timeout
    void timeoutMethod2(Timer timer) {
        method2();
    }
}
REPAIR 

Ensure the bean only has one "@Timeout" method. Remove or merge functionality
from the excess methods.

package examples.rules.ejb.ejb3;

@Stateless
public class MTMFixed implements MTMFixedInterface { //FIXED
    @Timeout
    void timeoutMethod(Timer timer) {
        method1();
        method2();
    }
}

Debu Panda, Reza Rahman, and Derek Lane. EJB3 in Action.
