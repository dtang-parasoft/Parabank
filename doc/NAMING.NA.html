Use a naming convention for annotations [NAMING.NA]



DESCRIPTION 


This rule identifies annotation type declarations that do not follow the 
user-defined naming convention (as specified in a regular expression). The 
default regular expression is "^[A-Z]". An error is reported for each 
occurrence.  This rule is new for JDK 1.5.
SCOPE LEVEL 
LINE
SINCE 

v8.0
PARAMETERS 

-regular expression
    -Annotation type declaration names should match the listed regular 
     expression.
    -The default is "^[A-Z]".  This enforces that the names start with a capital 
     letter.
-Negate regular expression
    -If this parameter is enabled, a violation will be flagged if the name of an
     annotation type declaration matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for annotation type declaration names which do not match the 
     regular expression specified.
-Ignore case
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of annotation 
     type declarations must match the regular expression specified, including in 
     case.
EXAMPLE 

If the regular expression is set to "^[A-Z]" to enforce that the first 
character of the name of the annotation type is a capital letter, the following 
will cause a violation:

package examples.rules.NAMING;

public class NA{
    public @interface securityPermission
    {
        String[] value();
    }
}
REPAIR 

Change the name to match the regular expression.

package examples.rules.NAMING;

public class NAFixed{
    public @interface SecurityPermission
    {
        String[] value();
    }
}
