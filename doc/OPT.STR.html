Use single quotes instead of double quotes for single character string concatenation [OPT.STR]



DESCRIPTION 


This rule identifies code that uses double quotes instead of single quotes for
single character string concatenation. An error is reported if code uses String
concatenation with the "+" operator when one of the arguments is a single
character String. This rule also checks single character String concatenations
through use of the StringBuffer or StringBuilder 'append()' method.
SCOPE LEVEL 
LINE
SINCE 

v4.5
BENEFITS 

Mobile platforms have reduced resources, optimizations are especially important.

Code is optimized when you use single quotes instead of double quotes for single
character string concatenation.
EXAMPLE 

package examples.rules.opt;

public class STR {
    public String method(String s) {
        String string = null;
        string = s + "d";  // VIOLATION
        return string;
    }
}
REPAIR 

Instead of concatenating one-character String constants, replace them with
character constants.

package examples.rules.opt;

public class STRFixed {
    public String method(String s) {
        String string = null;
        string = s + 'd';  // FIXED
        return string;
    }
}
