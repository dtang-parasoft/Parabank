Make sure implementation of Object.equals(Object) properly handles null values [BD.API.EQNULL]



DESCRIPTION 

This rule checks for Object.equals(Object) methods that return true when called with null
parameter. This practice is inconsistent with the equals method rule, which states that for any
non-null reference value x, x.equals(null) should return false.
SINCE 
v10.0

EXAMPLE 
Here is an example that triggers a violation:

public class Example
{
    @Override
    public boolean equals(Object obj)
    {
        if (obj instanceof Example) {
            return false;
        }
        return true; // VIOLATION
    }
}
REPAIR 

public class Repair
{
    @Override
    public boolean equals(Object obj)
    {
        if (obj instanceof Repair) {
            return true;
        }
        return false;
    } // NO VIOLATION
}
