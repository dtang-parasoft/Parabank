Avoid empty classes [PB.USC.EC]



DESCRIPTION 


This rule identifies empty class declarations. An error
is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v7.5
PARAMETERS 

-Ignore classes which extend from the classes in the table.
     -The table is already populated with the 'java.lang.Throwable'
     class.  No violations will be flagged for classes which extend 
     any class that is listed in the table.
BENEFITS 

Empty class declarations are meaningless. Removing
unnecessary class declarations improves code readability
and makes code less error-prone.
EXAMPLE 

package examples.rules.pb.usc;

public class EC { // VIOLATION : empty class

}
REPAIR 

package examples.rules.pb.usc;

public class ECFixed {  // FIXED: add method

    public void display() {
        System.out.println("this is ECFixed class");
    }
}
