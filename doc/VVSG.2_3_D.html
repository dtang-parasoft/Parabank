2.3-D: Hard-coded passwords and keys [VVSG.2_3_D]

RULES

Avoid using hard-coded cryptographic keys [VVSG.2_3_D.HCCK]
Avoid passing hardcoded usernames/passwords/URLs to database connection methods [VVSG.2_3_D.HCCS]
Avoid storing usernames and passwords in plain text in Castor 'jdo-conf.xml' files [VVSG.2_3_D.PCCF]
Password information should not be included in properties file in plaintext [VVSG.2_3_D.PLAIN]
Avoid using plain text passwords in Axis 'wsdd' files [VVSG.2_3_D.PTPT]
Ensure that passwords are not stored as plaintext and are sufficiently long [VVSG.2_3_D.PWDPROP]
Ensure that passwords are not stored as plaintext and are sufficiently long [VVSG.2_3_D.PWDXML]
Avoid storing unencrypted Hibernate usernames and passwords in 'web.xml' files [VVSG.2_3_D.UPWD]
Avoid using plain text passwords in Axis2 configuration files [VVSG.2_3_D.UTAX]
Avoid unencrypted passwords in WebSphere 'ibm-webservicesclient-ext.xmi' files [VVSG.2_3_D.WCPWD]
Avoid unencrypted passwords in WebSphere 'ibm-webservices-ext.xmi' files [VVSG.2_3_D.WPWD]
