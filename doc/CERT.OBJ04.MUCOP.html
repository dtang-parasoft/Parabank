Provide mutable classes with copy functionality [CERT.OBJ04.MUCOP]



DESCRIPTION 


This rule reports violations when a mutable class does not provide appropriate
copy functionality.  The parameters to this rule determine what functionality
is required.
SINCE 

v9.0
PARAMETERS 

 - Provide a public clone method(default is off)
        The class selected must be final, and must contain a method named 
        "clone" that is public, has no parameters, and returns the class type. 
 - Provide a copy constructor(default is on)
        The class selected must contain a constructor that takes a single 
        parameter which is the class type 
 - Provide a static creation method
        The class selected must contain a static method that returns the class  
        type and takes a single parameter which is the class type
EXAMPLE 

package examples.rules.OOP;

public final class MUCOP {
    public MUCOP(){   }
    int field1, field2;
    private void method() {    }
}
REPAIR 

Add appropriate methods

package examples.rules.OOP;

public final class MUCOPFixed {
    public MUCOPFixed(){   }
    public MUCOPFixed(MUCOPFixed var){   }//FIXED
    int field1, field2;
    private void method() {    }
    static MUCOPFixed method(MUCOPFixed var){//FIXED
        return new MUCOPFixed();
    }
    public MUCOPFixed clone(){  
        return new MUCOPFixed();
    }
}
