Avoid unnecessary "return" statement at the end of "void" methods [VVSG.14_2_G.VR]



DESCRIPTION 


This rule identifies unnecessary "return" statements at the end of 
methods declared as returning void; these statements are identified
because they are redundant. An error is reported for each occurrence.
SINCE 

v6.0
BENEFITS 

Extra and unnecessary code should be removed if it does not 
improve code readability.
EXAMPLE 

package examples.rules.uc;

public class VR {
    public void myMethod() {
        System.out.println("Hello");
        return; //flag, unnecessary return statement
    }
}
REPAIR 

package examples.rules.uc;

public class VRFixed {
    public void myMethod() {
        System.out.println("Hello");
        //statement removed and fixed
    }
}
