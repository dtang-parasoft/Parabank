Limit the maximum length of a line [APSC_DV.003215.LL]



DESCRIPTION 

   
This rule detects lines whose length exceeds the maximum specified value.
The rule must be parameterized to specify the maximum acceptable length of
code lines and comment lines.
SCOPE LEVEL 

LINE
SINCE 

v4.5
NOTES 

This rule also applies to JavaDoc comments.
PARAMETERS 

-Maximum line width (the default is 80 characters )- specifies the maximum
    acceptable length of code lines.
-Maximum comment line width (the default is 80 characters) - specifies
    the acceptable maximum length of comment lines.
-Tab size (the default is 4 characters) - specifies the number of characters
   a tab represents. By default, the rule assumes that a tab corresponds
   to four characters.
-Ignore trailing spaces (enabled by default)
-Ignore trailing comments 
-Ignore import statements and package declarations
BENEFITS 

This rule improves code readability by helping you avoid long code lines,
which are difficult to read, print out, and share.
EXAMPLE 

In the following example, the rule exceeds the maximum line length
specified with the parameters.

package examples.rules.format;

public class LL {
    // VIOLATION
    public void method (int i, int j, int k, int l, int m, int n, int o, int p, int q) {
    }
}
REPAIR 

To fix the code, break the single line into multiple well-formatted lines.  

package examples.rules.format;

public class LLFixed {
    public void method ( // FIXED
        int i
        , int j
        , int k
        , int l
        , int m
        , int n
        , int o 
        , int p
        , int q
    ) {
    }
}
