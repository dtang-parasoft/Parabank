Use a naming convention for non-"static" fields [NAMING.NIF]



DESCRIPTION 


This rule identifies non-"static" fields that do not follow the user-defined
naming convention (as specified in a regular expression). The rule does not 
enforce a default requirement. An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-regular expression
    -Non-"static" field names should match the listed regular expression.
    -The default is not set.
-check public field. (default is true)
-check protected field. (default is true)
-check package-private field. (default is true)
-check private field. (default is true)
-Negate regular expression
    -If this parameter is enabled, a violation will be flagged if the name of a
     non-"static" field matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for non-"static" field names which do not match the regular 
     expression specified.
-Ignore case
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of non-
     "static" fields must match the regular expression specified, including in 
     case.
BENEFITS 

This helps enforce a naming convention.
