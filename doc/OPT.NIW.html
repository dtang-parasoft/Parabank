Avoid creating new instances of BigInteger and BigDecimal [OPT.NIW]



DESCRIPTION 


This rule identifies the following instances:
  new BigInteger(0)
  new BigInteger(1)
  new BigInteger(10)
  
  new BigDecimal(0)
  new BigDecimal(1)
  new BigDecimal(10)
  
An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v9.0
BENEFITS 

Mobile platforms have reduced resources, optimizations are especially important.

Instead of creating a new BigInteger or BigDecimal instance, using a predefined
constant in BigInteger or BigDecimal can improve performance.
EXAMPLE 

package examples.rules.opt;

import java.math.*;

public class NIW {

    private BigInteger bint = new BigInteger(1);  // VIOLATION
}
REPAIR 

Use BigInteger.ONE

package examples.rules.opt;

import java.math.*;

public class NIWFixed {

    private BigInteger bint = BigInteger.ONE; // FIXED
}
