Do not use too many negation operators '!' in a single method [CODSTA.READ.DUN]



DESCRIPTION 


This rule identifies any method that uses the negation operator, '!' more than 2 times(by default).
An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-Maximum negation operator in method (default value is 2)
BENEFITS 

The negation operator makes code more difficult to read.
EXAMPLE 

package examples.rules.codsta.read;

public class DUN {
    boolean method (boolean a, boolean b) {  // VIOLATION
        if (!a)
            return (!a && !b);
        else
            return !b;
    }
}
REPAIR 

Do not use the negation operator if possible.

package examples.rules.codsta.read;

public class DUNFixed {
    boolean method (boolean a, boolean b) {
        if (a)
            return !b;
        else
            return (! (a || b) );
    }
}
