Avoid unnecessary comparisons to "null" [OPT.MUNC]



DESCRIPTION 


This rule is designed to detect unnecessary null checks.  That is, it will flag
as a violation cases where a variable's value is known to be either null or 
non-null and the variable is compared against the literal "null" anyway.
SCOPE LEVEL 
LINE
SINCE 

v8.0
BENEFITS 

Mobile platforms have reduced resources, optimizations are especially important.

It is inefficient and unnecessary to compare a variable against the value "null"
if it can be determined by looking at the code whether or not the variable will
be null.
EXAMPLE 

package examples.rules.opt;

public void foo(){
    Object o;
    o= null;
        
    for(int i= 0; i<10; i++){
        if(o== null){  //VIOLATION- o has to be null since it was set to null
                       //when last used.
        }
    }
}
REPAIR 

Eliminate the unnecessary comparison.

package examples.rules.opt;

public void foo(){
    Object o;
    o= null;
        
    for(int i= 0; i<10; i++){
        //FIXED
    }
}
