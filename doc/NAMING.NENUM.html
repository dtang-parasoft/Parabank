Use a naming convention for enum type declarations [NAMING.NENUM]



DESCRIPTION 


This rule identifies enum declarations that do not follow the user-defined
naming convention (as specified in a regular expression). The default regular 
expression is "^[A-Z]".  An error is reported for each occurrence.  This rule is 
new for JDK 1.5.
SCOPE LEVEL 
LINE
SINCE 

v8.0
PARAMETERS 

-regular expression
    -Enum type declaration names should match the regular expression specified 
     here.
    -The default is "^[A-Z]", which enforces that the names start with a capital 
     letter.
-Negate regular expression
    -If this parameter is enabled, a violation will be flagged if the name of an
     enum type declaration matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for enum type declaration names which do not match the regular 
     expression specified.
-Ignore case
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of enum type
     declarations must match the regular expression specified, including in 
     case.
BENEFITS 

This helps enforce a naming convention.
EXAMPLE 

If the regular expression is set to "^[A-Z]" to enforce that the first 
character of the name of the enum type is a capital letter, the following will
cause a violation:

package examples.rules.naming;

public class NENUM{
    private enum grade { A, B, C, D, F }; //VIOLATION
}
REPAIR 

Change the name to match the regular expression.

package examples.rules.naming;

public class NENUMFixed{
    private enum Grade { A, B, C, D, F }; //FIXED
}
