Avoid classes that implement 'Externalizable' but do not define a no-argument constructor [SERIAL.ENNAC]



DESCRIPTION 


This rule identifies any class that implements 'java.io.Externalizable', but 
does not define a zero-argument constructor. An error is reported for each 
occurrence.
SCOPE LEVEL 

CLASS
SINCE 

v6.0

EXAMPLES

package examples.rules.serial;

public class ENNAC implements Externalizable{
    public ENNAC(int i) {}
    //VIOLATION, no zero-argument constructor 
    //exception would be thrown if object of this class is deserialized
    //java.io.InvalidClassException: 
    //serialization.ExternalizableObject; no valid constructor
}
BENEFITS 

Whenever the class object that implements 'java.io.Externalizable' is
deserialized, the zero-argument constructor of that class would be 
invoked. If such a zero-argument constructor is not present, the 
deserialization process would experience errors.
REPAIR 

package examples.rules.serial;

public class ENNACFixed implements Externalizable {
    ENNACFixed() { //fixed
    //...
    }   
    //...
}
