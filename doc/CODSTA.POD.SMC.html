Avoid "switch" statements with too many or too few "case" statements [CODSTA.POD.SMC]



DESCRIPTION 


This rule identifies "switch" statements with more than 10 cases(by default).
An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-Minimum "switch" cases (default value is 0)
-Maximum "switch" cases (default value is 10)
BENEFITS 

"switch" statements with many "case" statements make code difficult to follow.  
More importantly, switches with many cases often indicate places where 
polymorphic behavior could better be used to provide different behavior for
different types.  Note that although the general principle is to avoid many 
cases in a switch, the actual cutoff point is arbitrary.

"switch" statements with too few "case" statements may be better represented
as "if"/"else" statements.
REPAIR 

Look for cleaner ways to invoke the alternative behaviors.
