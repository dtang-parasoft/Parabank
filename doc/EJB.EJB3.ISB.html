Do not inject @Stateful beans into @Stateless beans [EJB.EJB3.ISB]



DESCRIPTION 


This rule identifies @Stateful beans which are injected into a @Stateless bean.
This behavior does not make sense and is likely an error. @Stateless beans are
commonly reused and injected @Stateful beans may leave data behind.
SINCE 

v8.4
NOTES 

This rule identifies injections using the "@EJB" and "@Resource" annotations.
If an interface is injected, then a violation is flagged if any implementations
of that interface are @Stateful.
PARAMETERS 

Check the following:
    -Check annotations
    -Check annotations and deployment descriptors (default)
        Check deployment descriptors in the following location:
            -Check project (default)
            -Check package
EXAMPLE 

package examples.rules.ejb.ejb3;

@Stateless
public class ISB implements ISBInterface {
    @EJB
    StatefulBeanInterface statefulBean; //VIOLATION
    ...
}

@Stateful
class StatefulBean implements StatefulBeanInterface {
    ...
}
REPAIR 

Remove @Stateful injection or convert to a @Stateful bean.

package examples.rules.ejb.ejb3;


@Stateful //FIXED
public class ISBFixed implements ISBFixedInterface {
    @EJB
    StatefulBeanInterface statefulBean;
    ...
}

@Stateful
class StatefulBean implements StatefulBeanInterface {
    ...
}
