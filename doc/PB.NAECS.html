Include a 'case' statement for each constant of an 'enum' type in 'switch' statements [PB.NAECS]



DESCRIPTION 


This rule will flag a violation for each 'switch' statement where the type of 
the 'switch' expression is an 'enum' type and there is not a 'case' statement 
for each possible value of the 'enum' type.  If a 'switch' statement uses an
'enum' type, there should be a case for each possible value of the 'enum' type.
SCOPE LEVEL 
LINE
SINCE 

v8.0
BENEFITS 

It is often a bug if one or more of the possible values of an 'enum' type are 
missing a case in a 'switch' statement.  'switch' statements should take care of
every possible value that an expression can have.
EXAMPLE 

package examples.rules.pb;

public class NAECS {
    enum Color {
        RED, GREEN
    };

    public void printColor(Color c) {
        switch (c) {  // VIOLATION- no 'case' for the value 'GREEN'
        case RED:
            System.err.println("Red");
            break;
        default:
            System.err.println("Error");
            break;
        }
    }
}
REPAIR 

Include a 'case' statement for each possible value of the 'enum'.

package examples.rules.pb;

public class NAECSFixed {
    enum Color {
        RED, GREEN
    };

    public void printColor(Color c) {
        switch (c) {  
        case RED:
            System.err.println("Red");
            break;
        case GREEN:                       //FIXED- added 'case' for 'GREEN'
            System.err.println("Green");
            break;    
        default:
            System.err.println("Error");
            break;
        }
    }
}
