Do not call 'setSize()' in 'ComponentListener.componentResized()' [PB.API.DNCSS]



DESCRIPTION 


This rule identifies code that calls 'setSize()' in 
'ComponentListener.componentResized()'. An error is reported for
each occurrence.
SINCE 

v4.5
BENEFITS 

The 'componentResized()' method gets called when the component's size changes.
If  the 'setSize()' method is invoked from within the 'componentResized()', 
this can cause an unending sequence of resizing events:

1) User resizes component.
2) 'componentResized()' gets invoked.
3) 'componentResized()' invokes 'setSize()'.
4) 'setSize()' posts a component resized event.
5) 'componentResized()' gets invoked.
6) ...
EXAMPLE 

package examples.rules.pb.api;

import java.awt.*;
import java.awt.event.ComponentEvent;

public class DNCSS extends Component {
    public void componentResized (ComponentEvent e) {
        Dimension d = getSize();
        setSize(d.width - 10, d.height - 10);  // VIOLATION
    }
}
REPAIR 

Do not call the 'setSize()' method inside of the 'componentResized()' method's body.

package examples.rules.pb.api;

import java.awt.*;
import java.awt.event.ComponentEvent;

public class DNCSSFixed extends Component {
    public void componentResized (ComponentEvent e) {
        Dimension d = getSize();
          // FIXED
    }
}
