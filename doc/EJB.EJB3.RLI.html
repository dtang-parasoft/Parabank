Do not declare an interface both @Local and @Remote [EJB.EJB3.RLI]



DESCRIPTION 


This rule identifies interfaces which have both @Local and @Remote annotations.
There are many differences between @Local and @Remote calls. @Local involves
pass-by-reference while @Remote involved pass-by-value. Parameters for @Remote
calls must also be Serializable. Declaring an interface as both should also be
avoided for clarity, even if not explicitly disallowed by an implementation of
the EJB3 specification.
SINCE 

v8.4
EXAMPLE 

package examples.rules.ejb.ejb3;

@Local
@Remote
public interface RLI { //VIOLATION
    ...
}
REPAIR 

Make separate @Local and @Remote interfaces or remove the erroneous annotation.

package examples.rules.ejb.ejb3;

@Local
public interface RLIFixedLocal { //FIXED
    ...
}

@Remote
public interface RLIFixedRemote { //FIXED
    ...
}
