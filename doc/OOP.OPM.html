Do not override an instance "private" method [OOP.OPM]



DESCRIPTION 


This rule identifies code that attempts to override instance 
"private" methods. An error is reported if a class contains any 
instance methods with the same signature as an instance "private" 
method in a parent class.
SCOPE LEVEL 
METHOD
SINCE 

v4.5
BENEFITS 

An instance "private" method in a superclass is not overridden by
a method with the same name in the current class.  This can lead
to confusion.  Also, if the method accessibility in the superclass 
is changed to non-private, then the program semantics will change
because the method will then be overridden.
EXAMPLE 

package examples.rules.oop;

public class OPM extends Super {
    private void method () {}  // VIOLATION
}

class Super {
    private void method () {}

    public static void main (String[] args) {
        OPM x = new OPM ();
        test (x);
    }

    private static void test (Super x) {
        x.method ();  // invokes 'Super.method ()', not 'OPM.method ()'
    }  
}
REPAIR 

Use different names for the methods to clarify that they are
unrelated and to avoid possible problems if the access modifier
is changed.

package examples.rules.oop;

public class OPMFixed extends Super {
    private void renamedMethod () {}  // FIXED
}

class Super {
    private void method () {}

    public static void main (String[] args) {
        OPMFixed x = new OPMFixed ();
        test (x);
    }

    private static void test (Super x) {
        x.method ();  // invokes 'Super.method ()', not 'OPMFixed.renamedMethod ()'
    }  
}
