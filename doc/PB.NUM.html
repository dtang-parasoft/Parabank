Numerical Errors [PB.NUM]

RULES

Avoid implicit casts from integer data types to floating point data types [PB.NUM.AIC]
Do not pass floating point values to the 'BigDecimal' constructor [PB.NUM.BBDCC]
Do not use an integer outside the range of [0, 31] as the amount of a shift [PB.NUM.BSA]
Avoid using compound assignment operators in cases which may cause overflow [PB.NUM.CACO]
Do not cast primitive data types to lower precision [PB.NUM.CLP]
Possible integer overflow in a compare or compareTo method [PB.NUM.CMP]
Do not compare floating point types [PB.NUM.DCF]
Do not use floating point variables as loop indices [PB.NUM.FPLI]
Avoid calculations which result in overflow or NaN [PB.NUM.ICO]
Do not assign the result of an integer division to a floating point variable [PB.NUM.IDCD]
Avoid casting the result of an integer multiplication to "long" [PB.NUM.IMOF]
Use unsigned right shift instead of division when overflow is possible [PB.NUM.IOF]
Avoid comparisons to Double.NaN or Float.NaN [PB.NUM.NAN]
Do not initialize array dimensions with negative numbers [PB.NUM.NIA]
Provide methods to read and write little-endian data [PB.NUM.PMRWLED]
Do not use "float" and "double" if exact answers are required [PB.NUM.UBD]
Avoid passing integer values to 'Math.round()' [PB.NUM.UCM]
