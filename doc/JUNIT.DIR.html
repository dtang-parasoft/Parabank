Place each test class in the same location as the class that it tests [JUNIT.DIR]



DESCRIPTION 


This rule identifies any JUnit test class that is not in the same directory
as the class being tested. An error is reported for each occurrence.
SCOPE LEVEL 
CLASS
SINCE 

v5.0
NOTES 

This rule assumes that the test class follows the naming convention 
<class name>Test.

This rule defines a JUnit test class as any class which has a name ending in 
"Test" and which meets one of the following conditions:

1. The class extends "junit.framework.TestCase".
2. The class has at least one method with the "org.junit.Test" annotation.  This
only applies to JUnit 4.0 and later.
BENEFITS 

If a test class is in the same directory as the class it tests, both will be
compiled during a normal build.
