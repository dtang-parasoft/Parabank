Use 'is...' only for naming methods that return a "boolean" [NAMING.IRB]



DESCRIPTION 



Requires that only methods that return "boolean" values have names beginning
with 'is'.
SCOPE LEVEL 

METHOD
SINCE 

v4.5
BENEFITS 

This helps enforce a naming convention.
EXAMPLE 

package examples.rules.naming;

public class IRB {
    public int isOK () {  // VIOLATION
        return _value;
    }
    private int _value;
}
REPAIR 

Use standard naming conventions for legibility.

package examples.rules.naming;

public class IRBFixed {
    public int getValue () {  // FIXED
        return _value;
    }
    private int _value;
}
