Do not initialize array dimensions with negative numbers [CERT.NUM51.NIA]



DESCRIPTION 


This rule reports an error when a negative value is found in
array dimension declaration.
SCOPE LEVEL 
LINE
SINCE 

v8.0
SECURITY RELEVANCE 

category: Erratic Application Behavior

Unexpected numeric results may cause unexpected behavior and create security
holes. See BENEFITS for more information.
BENEFITS 

When an array is declared with a negative dimension, a NegativeArraySizeException
would be thrown. Such an exception should be avoided no matter what.
EXAMPLE 

package examples.rules.pb.num;

public class NIA {
    private static final A_SIZE= 10;
    private static final B_SIZE= 15;
    private static int[] diffSet= new int[A_SIZE- BSIZE]; //VIOLATION

}
REPAIR 

package examples.rules.pb.num;

public class NIAFixed {
    private static final A_SIZE= 10;
    private static final B_SIZE= 15;
    private static int[] diffSet= new int[A_SIZE- BSIZE>= 0 ? A_SIZE- B_SIZE : 0]; //VIOLATION

}
