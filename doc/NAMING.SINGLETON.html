Use a naming convention for singleton classes [NAMING.SINGLETON]



DESCRIPTION 


This rule reports violations when a singleton class does not follow the 
user-defined naming convention (as specified in a regular expression). By
default, the rule requires that the singleton class name end with "Singleton". 

An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v8.0
NOTES 
This rule had 3 potential errors moved into the rule OOP.SNGL, the one that
remains is truly a naming rule, as the problem and solution is only with the
name of the class
PARAMETERS 

-regular expression (default value is "^.+Singleton$")
BENEFITS 

This rule helps to enforce a naming convention.
EXAMPLE 

package examples.rules.naming;

class NCL {  // VIOLATION

    private static NCL _ncl;
    
    private NCL() {
    }
    
    public void display() {
        System.out.println("NCL");
    }
    
    public static synchronized NCL getInstance() {
        if (_ncl == null) {
            _ncl = new NCL();
        }
        return _ncl;
    }
}
