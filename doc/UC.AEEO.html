Ensure that classes do not explicitly extend 'java.lang.Object' [UC.AEEO]



DESCRIPTION 


With the default setting of its parameter, this rule identifies classes that 
explicitly extend 'java.lang.Object'.  An error is reported for each occurrence.
SCOPE LEVEL 
CLASS
SINCE 

v5.1
PARAMETERS 

-Avoid or enforce explicitly extending 'Object'?
    -Default is "Avoid".
    -If "Avoid" is selected, a violation will be flagged for each class which
     explicitly extends 'java.lang.Object'.  
    -If "Enforce" is selected, a violation will be flagged for each class which 
     implicitly extends 'java.lang.Object' but does not explicitly extend 
     'java.lang.Object'.
BENEFITS 

By design, Java implicitly extends 'java.lang.Object', so explicitly extending
'java.lang.Object' is redundant.  This is why we recommended using this rule 
with the parameter set to "Avoid".

However, some people prefer to make all superclasses explicit, meaning that even
classes which have 'java.lang.Object' as a superclass should be declared to 
explicitly extend their superclass.  This is why you might want to use this 
rule with the parameter set to "Enforce".
EXAMPLE 

package examples.rules.uc;

public class AEEO extends Object {  // VIOLATION

}
REPAIR 

Remove 'java.lang.Object'.

package examples.rules.uc;

public class AEEOFixed {  // FIXED

}
