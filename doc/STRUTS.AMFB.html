Specify a '<form-bean>' for each named '<action>' element in 'struts-config.xml' files [STRUTS.AMFB]



DESCRIPTION 


This rule checks 'struts-config.xml' files. A violation is flagged for each
'<action>' with a @name attribute and no corresponding '<form-bean>' of the same
name.
SINCE 

v8.4
SECURITY RELEVANCE 

category: Unsafe Environment Configuration

If a named '<action>' has no '<form-bean>' counterpart it points to
inconsistencies in the code. Having an improper or erroneous configuration can
lead to security problems and confusing behavior.
EXAMPLE 

The '<action>' element in the sample 'struts-config.xml' file has a @name
attribute but there is no '<form-bean>' corresponding to that name.

struts-config.xml:

<?xml version="1.0" encoding="ISO-8859-1" ?>
<struts-config>
    <form-beans></form-beans>
    <action-mappings>
        <!-- VIOLATION: Missing form-bean "logonForm" -->
        <action
            path="/LogonSubmit"
            type="app.LogonAction"
            name="logonForm"
            scope="request"
            validate="true"
            input="/pages/Logon.jsp">
            <forward name="success" path="/pages/Welcome.jsp"/>
            <forward name="failure" path="/pages/Logon.jsp"/>
        </action>
    </action-mappings>
</struts-config>
REPAIR 

Specify a '<form-bean>' corresponding to the named '<action>'.

struts-config.xml:

<?xml version="1.0" encoding="ISO-8859-1" ?>
<struts-config>
    <form-beans>
        <!-- FIXED -->
        <form-bean name="logonForm" type="app.LogonForm"/>
    </form-beans>
    <action-mappings>
        <action
            path="/LogonSubmit"
            type="app.LogonAction"
            name="logonForm"
            scope="request"
            validate="true"
            input="/pages/Logon.jsp">
            <forward name="success" path="/pages/Welcome.jsp"/>
            <forward name="failure" path="/pages/Logon.jsp"/>
        </action>
    </action-mappings>
</struts-config>
