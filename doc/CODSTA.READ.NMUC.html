Declare never-modified collections as "unmodifiable" for clarity [CODSTA.READ.NMUC]



DESCRIPTION 


This rule flags a violation when a "private" "final" Collection or Map field
is never modified beyond initialization. By wrapping it with an "unmodifiable"
"java.util.Collections" method readability is improved, however there may be a
cost in performance. See BENEFITS and DRAWBACKS for more information.
SINCE 

v8.4
NOTES 

If the field is returned from a function or passed to a method then no violation
is flagged, as it may be modified elsewhere.
BENEFITS 

If a field is wrapped with an "unmodifiable" method from "java.util.Collections"
then readability is improved as the purpose of the collections is more apparent.
DRAWBACKS 

This rule may have effects on performance.
EXAMPLE 

package examples.rules.codsta.read;

public class NMUC {
    private final List POSSIBLE_VALUES = getPossibleValues(); //VIOLATION

    boolean isPossibleValue(Object o) {
        return POSSIBLE_VALUES.contains(o);
    }
    ...
}
REPAIR 

package examples.rules.codsta.read;

public class NMUCFixed {
    private final List POSSIBLE_VALUES =
        Collections.unmodifiableList(getPossibleValues()); //FIXED

    boolean isPossibleValue(Object o) {
        return POSSIBLE_VALUES.contains(o);
    }
    ...
}
