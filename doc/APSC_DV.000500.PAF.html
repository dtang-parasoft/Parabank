Declare subclasses of 'PrivilegedAction', 'PrivilegedExceptionAction', and 'PrivilegedActionException' "final" [APSC_DV.000500.PAF]



DESCRIPTION 


This rule flags a violation if a class is non-"final" and satisfies one of the
following:
    1. The class extends "java.security.PrivilegedActionException"
    2. The class implements "java.security.PrivilegedAction"
    3. The class implements "java.security.PrivilegedExceptionAction"
SINCE 

v8.4
SECURITY RELEVANCE 

category: Weak Security Controls

"Privileged" classes and interfaces are used to interact with sensitive data and
operations. A subclass or implementation should be declared "final" to prevent
an attacker from creating a subclass and gaining access to sensitive
information.
PARAMETERS 

-Check implementations of "java.security.PrivilegedAction" (default on)
-Check implementations of "java.security.PrivilegedExceptionAction" (default on)
-Check subclasses of "java.security.PrivilegedActionException" (default on)
-Ignore subclasses which are enclosed by a "final" outer class (default on)
EXAMPLE 

package examples.rules.security.wsc;

import java.security.PrivilegedAction;

public class PAF implements PrivilegedAction { //VIOLATION
    public Object run() {
        ...
    }
}
REPAIR 

package examples.rules.security.wsc;

import java.security.PrivilegedAction;

public final class PAFFixed implements PrivilegedAction { //FIXED
    public Object run() {
        ...
    }
}
