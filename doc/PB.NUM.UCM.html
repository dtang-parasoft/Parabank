Avoid passing integer values to 'Math.round()' [PB.NUM.UCM]



DESCRIPTION 


This rule identifies code which passes integer values to 'Math.round()'. 
Doing so is unnecessary because there is no need to round an integer value.
An error is reported for each occurrence.
SINCE 

v8.2
BENEFITS 

'Math.round()' is a no-op if the argument is an integer value.  This method 
invocation either is a typo or is redundant code.
EXAMPLE 

package examples.rules.pb.num;

public class UCM {

    public int myRound(int i) {
        return Math.round(i);  // VIOLATION
    }
}
REPAIR 

'myRound()' does not do anything useful.  You should either remove this
pointless method or change the parameter type to a floating point type.

package examples.rules.pb.num;

public class UCMFixed {
 
    public int myRound(float f) {
        return Math.round(f);  // FIXED
    }
}
