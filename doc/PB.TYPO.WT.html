Ensure that arguments passed to Java wrapper classes do not contain typos [PB.TYPO.WT]



DESCRIPTION 


This rule flags identifies typos in in arguments passed to the Java wrapper
classes 'Byte', 'Short', 'Integer', 'Float', 'Double' and 'Boolean'. An error
is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v6.0
BENEFITS 

Typos in arguments will trigger NumberFormatException if they are passed into
a numeric wrapper class. For 'Boolean', a typo might alter the intended boolean
value without being noticed.
EXAMPLE 

package examples.rules.pb.typo;

public class WT {
    Integer i= new Integer("10A"); //VIOLATION
    public Boolean myMethod() {
        return Boolean.valueOf(" true"); //VIOLATION, would return "false"
    }
}
REPAIR 

package examples.rules.pb.typo;

public class WTFixed {
    Integer i= new Integer("10"); //FIXED
    public Boolean myMethod() {
        return Boolean.valueOf("true"); //FIXED
    }
}
