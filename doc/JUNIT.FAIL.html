Use the fail() method instead of forcing a failed condition using the assertTrue(false) or assertFalse(true) method [JUNIT.FAIL]



DESCRIPTION 


This rule identifies JUnit test classes that force a failed condition using
the assertTrue(false) or assertFalse(true) method instead of the fail() method.
SINCE 

v4.5
NOTES 

This rule assumes that test classes are classes that meet the following
conditions:
- In JUnit 3, a test class is a class that extends the junit.framework.TestCase
  class.
- In JUnit 4, a test class is a class that contains a method marked with
  the @Test annotation.
- In JUnit 5, a test class is a class that contains a method marked with
  one of the following annotations: @Test, @TestFactory, @TestTemplate,
  @ParameterizedTest, @RepeatedTest
BENEFITS 

This rule helps you ensure that the fail() method is properly used.
EXAMPLE 

package examples.rules.junit;

import junit.framework.*;

public class FAILTest extends TestCase {

    public void testMethod() {

         assertFalse(true);  // VIOLATION
    }
}
REPAIR 

Modify the code to use fail().

package examples.rules.junit;

import junit.framework.*;

public class FAILFixedTest extends TestCase {

    public void testMethod() {

         fail("fail");  // FIXED
    }
}
