Avoid debug levels which are too high in Tomcat's 'server.xml' [OWASP2019.API10.DLTH]



DESCRIPTION 


This rule identifies if the debug level is too high in Tomcat's 'server.xml' 
file.  An error is reported for each occurrence.
SINCE 

v8.3
SECURITY RELEVANCE 

category: Unsafe Environment Configuration

If the debug level is too high, more log messages will be written to the log 
file, including some sensitive data like usernames and passwords.
PARAMETERS 
   - Maximum debug level (by default, maximum debug level is 2)
EXAMPLE 

<?xml version="1.0" ?>
<Server>
<!-- VIOLATION: debug level is too high -->
<Realm className="com.parasoft.jtest.LoginRealm"
       resourceName="Login"
       debug="5"/>
</Server>
REPAIR 

<?xml version="1.0" ?>
<Server>
<!-- FIXED: lower the debug level -->
<Realm className="com.parasoft.jtest.LoginRealm"
       resourceName="Login"
       debug="2"/>
</Server>
