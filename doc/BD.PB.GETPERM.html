Method getPermission() is supposed to be invoked when user-defined class loaders are applied [BD.PB.GETPERM]



DESCRIPTION 

This rule detects cases when a custom class loader overrides the getPermissions() method 
and the returned object isn't created by invoking the superclass's getPermissions() method.
SINCE 
v10.2
NOTES 
N/A
SECURITY RELEVANCE 
Failure to consult the default system policy while defining a custom class loader violates
the tenets of defensive programming and can result in classes defined with unintended permissions.
PARAMETERS 
N/A
BENEFITS 
Makes sure the default systemwide security policy is applied in addition to the custom policy.
DRAWBACKS 
N/A
EXAMPLE 
Here is an example that will trigger a violation.

import java.net.URL;
import java.net.URLClassLoader;
import java.security.CodeSource;
import java.security.PermissionCollection;
import java.security.Permissions;

public class Example
    extends URLClassLoader
{
    public Example(URL[] urls)
    {
        super(urls);
    }

    protected PermissionCollection getPermissions(CodeSource cs)
    {
        PermissionCollection pc = new Permissions();
        // Allow exit from the VM anytime
        pc.add(new RuntimePermission("exitVM"));
        return pc; // VIOLATION
    }
}
REPAIR 
The following code will not trigger a violation.

import java.net.URL;
import java.net.URLClassLoader;
import java.security.CodeSource;
import java.security.PermissionCollection;

public class Repair
    extends URLClassLoader
{
    public Repair(URL[] urls)
    {
        super(urls);
    }

    protected PermissionCollection getPermissions(CodeSource cs)
    {
        PermissionCollection pc = super.getPermissions(cs);
        // Allow exit from the VM anytime
        pc.add(new RuntimePermission("exitVM"));
        return pc; // NO VIOLATION
    }
}
