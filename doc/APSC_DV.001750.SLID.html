Ensure Session-ID Length is sufficient [APSC_DV.001750.SLID]



DESCRIPTION 


Session identifiers should be at least 128 bits long to prevent brute-force attacks.
This rule check deployment descriptor files and identifies uses of Session-ID 
with a width less than that set in the parameters.

A violation is reported for each xml file that contains the following structure:

...
<session-config>
        <session-properties>
            <property name="idLengthBytes" value="..." />
            ... 
        </session-properties>
    </session-config>
...

where "value" is less than that defined in the parameters.
SINCE 

v9.2
SECURITY RELEVANCE 

category: Unsafe Environment Configuration

see BENEFITS
PARAMETERS 

-Recommended Session-ID length
BENEFITS 

Protects against brute force attacks.
EXAMPLE 

<sun-web-app>
    <session-config>
        <session-properties>
            <property name="idLengthBytes" value="8"/> // BAD
        </session-properties>
    </session-config>
</sun-web-app>
REPAIR 
<sun-web-app>
    <session-config>
        <session-properties>
            <property name="idLengthBytes" value="16"/> // OK
        </session-properties>
    </session-config>
</sun-web-app>
 
In some cases, it is better to remove this property. The default value is set by the 
server, and in most cases it is higher than (or equal to) 16.
