Avoid redundant 'finalize()' methods which only call the superclass' 'finalize()' method [VVSG.14_2_G.FCSF]



DESCRIPTION 


If a 'finalize()' method does nothing besides call 'super.finalize()', it is
redundant and can be removed.  This rule will flag a violation for each 
'finalize()' method which only calls the superclass' 'finalize()' method.
SCOPE LEVEL 
LINE
SINCE 

v8.0
BENEFITS 

Having a redundant 'finalize()' method which only calls the superclass' 
'finalize()' method serves no purpose.  It only clutters up the code.  It may
also indicate a bug if some custom finalization was supposed to be performed in 
the method but was never implemented.
EXAMPLE 

package examples.rules.uc;

public class FCSF {
    protected void finalize() throws Throwable{  //VIOLATION
        super.finalize();
    }
}
REPAIR 

Remove the redundant 'finalize()' method or perform the necessary finalization. 

package examples.rules.uc;

public class FCSFFixed {
    //FIXED
}
