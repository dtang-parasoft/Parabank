Follow convention for bean properties [SPRING.PROPLIMIT]



DESCRIPTION 

This rule specifies how the property in xml bean should be declared in Spring XML configuration files.
A violation is reported for each property that does not follow the specified convention.
SINCE 

v9.2
PARAMETERS 
-Allow p-namespace
    -this option causes the rule to allow p-namespace form properties; it is enabled by default

-Allow shortcut property form
    -this option causes the rule to allow shortcut form properties; it is enabled by default

-Allow long property form
    -this option causes the rule to allow long form properties;  it is enabled by default
BENEFITS 
Following this rule makes code easier to understand and maintain.
EXAMPLE 

Uncheck "-Allow long property form option" in the rule parameters, then run on this example:

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
  xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">


  <!-- P-namespace test -->
  <bean name="myBeanA" class="parasoft.spring.impl.BeanImpl" p:test-ref="myBeanA">
    <meta key="test meta key" value="333"/>
  </bean> 
  
  <!-- Long way test -->
  <bean name="myBeanB" class="parasoft.spring.impl.BeanImpl">
    <property name="test">
        <ref bean="myBeanA" />
    </property>
  </bean> 
  
  <!-- Short way test -->
  <bean name="myBeanC" class="parasoft.spring.impl.BeanImpl">
    <property name="test" ref="myBeanB"/>
  </bean> 

</beans>
REPAIR 

Now you should see an error on the second test:

  <!-- Long way test -->
  <bean name="myBeanB" class="parasoft.spring.impl.BeanImpl">
    <property name="test">
        <ref bean="myBeanA" />
    </property>
  </bean> 
  
After changing it to the short form:

  <bean name="myBeanB" class="parasoft.spring.impl.BeanImpl">
    <property name="test" ref="myBeanA" />
  </bean> 
  
a violation will not be reported.
