Avoid globally unused "public/protected" fields [GLOBAL.UPPF]



DESCRIPTION 


This rule identifies unused "public" and "protected" fields.
An error is reported for each occurrence.

Global analysis is required in this rule.
SCOPE LEVEL 
LINE
SINCE 

v4.5
SECURITY RELEVANCE 

category: Weak Security Controls
category: Code Quality

Unused public code might be triggered /accessed by attackers. The code may
implement features that bypass security.
PARAMETERS 

-Ignore "final" fields (default is true)
 "final" modifier could imply the declared field is a constant
 for later use. Such a constant should be ignored.
BENEFITS 

These unused entities usually point to either:

1) Old code that is no longer needed.  This makes the class more
   difficult to understand.

2) A logical flaw (if the entity needs to be used, but other classes
   incorrectly avoid using it).
REPAIR 

Remove the unused field or document the reason for its existence.
REFERENCES 

CWE-563: Assignment to Variable without Use ('Unused Variable')
https://cwe.mitre.org/data/definitions/563.html
