Always override 'onSaveInstanceState()' [MOBILE.ANDROID.AOSM]



DESCRIPTION 


This rule will flag a violation if a class which extends 'android.app.Activity',
didn't override 'onSaveInstanceState()'.
SINCE 

v9.0
BENEFITS 

In the Android mobile platform, when another application pop up over your application, 
the 'onSaveInstanceState()' will be called and then your application will be killed.
Therefore, if your application didn't override the 'onSaveInstanceState()' and save 
your data manually, your data in the application will be lost when the application 
is killed.
EXAMPLE 

package examples.rules.mobile.android;

import android.app.*;

public class AOSM extends Activity { // VIOLATION : didn't override onSaveInstanceState()

    // an anrodid activity
}
REPAIR 


package examples.rules.mobile.android;

import android.app.*;
import android.os.*;

public class AOSMFixed extends Activity {

    public void onSaveInstanceState(Bundle bundle) {  // FIXED
        // save your data
    }
}
