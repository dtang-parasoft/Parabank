Avoid using "." as a regular expression in 'String.replaceAll()' and 'String.replaceFirst()' [PB.API.REP]



DESCRIPTION 



This rule identifies when using "." as a regular expression in
String.replaceAll() and String.replaceFirst() because "." is a
wild card expression which will match all string characters. 
An error is reported for each occurrence.
SINCE 

v8.2
SECURITY RELEVANCE 

"The software specifies a regular expression in a way that causes data to be
improperly matched or compared." [CWE-185]
BENEFITS 

We consider it is a typo if users use "." regular expression in
the String.replaceAll() or String.replaceFirst(). This rule can 
help you to detect this typo.
EXAMPLE 

package examples.rules.pb.api;

public class REP {
 
    public String removeDotInString(String s) {
        return s.replaceAll(".", "");  // VIOLATION
    }
}
REPAIR 

Use "\." instead

package examples.rules.pb.api;

public class REPFixed {

    public String removeDotInString(String s) {
        return s.replaceAll("\.", ""); // FIXED
    }
}
REFERENCES 

CWE-185: Incorrect Regular Expression
