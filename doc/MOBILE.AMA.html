Avoid using 'getter' and 'setter' methods [MOBILE.AMA]



DESCRIPTION 


This rule will flag a violation if 'getter' or 'setter' method is called.
SINCE 

v9.0
BENEFITS 

Method calls are expensive in embedded devices. So we should access a field directly,
instead of accessing a field by 'getter' or 'setter' method.
EXAMPLE 

package examples.rules.mobile;

public class AMA {

    private int number;
    
    public int getNumber() {
        return number;
    }
    
    public void calculate() {
        int num = getNumber();  // VIOLATION
        // ......
    }
}
REPAIR 

package examples.rules.mobile;

public class AMAFixed {

    private int number;
    
    public int getNumber() {
        return number;
    }
    
    public void calculate() {
        int num = number;  // FIXED
        // ......
    }
}
REFERENCES 

http://developer.android.com/guide/practices/design/performance.html#internal_get_set
