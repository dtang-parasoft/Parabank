Prefer ''type'' over ''index'' for constructor argument matching [SPRING.PREFTYPE]



DESCRIPTION 


This rule enforces using the "type" attribute instead of  "index" in the <constructor-arg> tag within Spring XML configuration files.
Each use of "index" is flagged (unless both "type" and "index" are present).
SINCE 

v9.2
NOTES 

Both tags need to be specified in the same cases to avoid ambiguity.
BENEFITS 

The "type" attribute is less error-prone and easier to read than "index"
EXAMPLE 

<bean id="myPizza"
        class="examples.Pizza">
        <constructor-arg  index="0" value="Dracula"/>
        <constructor-arg  index="1" value="4"/>
</bean>
REPAIR 

<bean id="myPizza"
        class="examples.Pizza">
        <constructor-arg  type="java.lang.String"  value="Dracula"/>
        <constructor-arg  type="int" value="4"/>
</bean>
REFERENCES 

1. Twelve Best Practices For Spring XML Configurations by Jason Zhicheng Li (O'Reilly ONJava.com)
