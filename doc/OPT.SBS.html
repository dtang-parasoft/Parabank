Use 'StringBuilder' instead of 'StringBuffer' when synchronization is not required [OPT.SBS]



DESCRIPTION 


This rule checks for 'StringBuffer' local variables. A violation is flagged for
each occurrence. See BENEFITS 

Mobile platforms have reduced resources, optimizations are especially important. and DRAWBACKS for more information.
SINCE 

v9.0
NOTES 

'StringBuilder' was introduced in JDK 1.5, so only projects with a compliance
level of Java 1.5 or later are checked.
BENEFITS 

Mobile platforms have reduced resources, optimizations are especially important.

'StringBuilder' has better performance than 'StringBuffer'. Upon acquiring a
lock, all variables from a thread's working memory will be flushed. If the
thread has a large memory footprint, then the BENEFITS 

Mobile platforms have reduced resources, optimizations are especially important. of this rule may be more
apparent.
DRAWBACKS 

If the 'StringBuffer' requires synchronization, then changing it to a
'StringBuilder' could cause erroneous or unintended behavior.
EXAMPLE 

package examples.rules.opt;

public class SBS {
    private String getDelineatedString(List<Object> objects) {
        StringBuffer sb = new StringBuffer(); //VIOLATION
        for (Object o : objects) {
            if (sb.length() > 0) {
                sb.append(", ");
            }
            sb.append(o.toString());
        }
        return sb.toString();
    }
}
REPAIR 

package examples.rules.opt;

public class SBSFixed {
    private String getDelineatedString(List<Object> objects) {
        StringBuilder sb = new StringBuilder(); //FIXED
        for (Object o : objects) {
            if (sb.length() > 0) {
                sb.append(", ");
            }
            sb.append(o.toString());
        }
        return sb.toString();
    }
}
