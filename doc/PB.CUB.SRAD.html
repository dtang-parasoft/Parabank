Specify @Retention for annotation type declarations [PB.CUB.SRAD]



DESCRIPTION 


This rule identifies annotation type declarations which do not specify a
@Retention (java.lang.annotation.Retention) annotation.
SINCE 

v9.0
BENEFITS 

If a '@Retention' annotation is not specified for an annotation type declaration
then it defaults to 'RetentionPolicy.CLASS'. This means that the annotation is
retained by the class file, but the VM may dispose of it. If reflection is used
to access the annotation then 'RetetionPolicy.RUNTIME' should be used.
Otherwise, the annotation may not be visible at runtime which could cause
confusion. So, it is often beneficial to always explicitly specify '@Retention'
to clarify an annotation's intended use.
EXAMPLE 

package examples.rules.pb.cub;

public @interface SRAD {
    ...
}
REPAIR 

Specify @Retention for the annotation type declaration.

package examples.rules.pb.cub;

@Retention(RetentionPolicy.SOURCE)
public @interface SRADFixed {
    ...
}
