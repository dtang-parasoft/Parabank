Do not declare entity beans as remote [OWASP2017.A6.RR]



DESCRIPTION 


This rule identifies if entity beans are declared remote in 'ejb-jar.xml'. 
An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v8.4
SECURITY RELEVANCE 

When an application exposes a remote interface for an entity bean, it might also expose methods
that get or set the bean's data. These methods could be leveraged to read sensitive information,
or to change data in ways that violate the application's expectations, potentially leading to other
vulnerabilities.
EXAMPLE 

<ejb-jar>
    <enterprise-beans>
        <entity>
            <ejb-name>EmployeeRecord</ejb-name>
            <home>com.wombat.emp.EmployeeRecordHome</home>
            <remote>com.wombat.empl.EmployeeRecord</remote> <!-- // VIOLATION -->
        </entity>
    </enterprise-beans>
</ejb-jar>
REPAIR 

Try to declare entity beans "local" when possible. If the bean must be remotely accessible, make
sure that sensitive information is not exposed and ensure that your application logic performs
appropriate validation of any data that might be modified by an attacker.
