Do not use "char" arrays in "String" concatenations [PB.CUB.SAC]



DESCRIPTION 


This rule reports an error when a "char" array is used in 'String'
concatenation.
SCOPE LEVEL 
LINE
SINCE 

v8.0
BENEFITS 

Unlike some other languages, Java does not support concatenations 
between "char" arrays and strings. Java would not report that as an
compilation error, however, the result produced from such a concatenation
would be different from what the programmer might have expected.
EXAMPLE 

package examples.rules.pb.cub;

public class SAC {
    private String str= "";
    private String appendString(char[] charSequence) {
        str+= charSequence; //VIOLATION
        return str;
    }
}
REPAIR 

package examples.rules.pb.cub;

public class SACFixed {
    private String str= "";
    private String appendString(char[] charSequence) {
        str+= String.valueOf(charSequence); //FIXED
        return str;
    }
}
