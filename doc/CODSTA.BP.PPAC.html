Do not define "public" or "protected" members in anonymous classes [CODSTA.BP.PPAC]



DESCRIPTION 


Anonymous classes should not have any "public" or "protected" 
members.  This rule will flag a violation for each field or method 
which is declared "public" or "protected" inside an anonymous class.
SINCE 

v8.1
DEPRECATED 

This rule is deprecated and has been replaced by OPT.AAM.
NOTES 

This rule will not flag a violation for overriding methods, as the
accessibility of overriding methods cannot be decreased.
BENEFITS 

Members of anonymous classes are not accessible outside of the class
definition, so there is no point in declaring them "public" or
"protected".  The members of anonymous classes can be declared
"private".
EXAMPLE 

package examples.rules.codsta.bp;

public class PPAC {
    TestAnonClass anonObj= new TestAnonClass(){
        public int foo;  //VIOLATION
        
        public void bar(String args[]){  //VIOLATION
        }
    };
}
REPAIR 

Declare the members of the anonymous class "private" instead of
"public" or "protected".

package examples.rules.codsta.bp;

public class PPACFixed {
    TestAnonClass anonObj= new TestAnonClass(){
        private int foo;  //FIXED
        
        private void bar(String args[]){  //FIXED
        }
    };
}
