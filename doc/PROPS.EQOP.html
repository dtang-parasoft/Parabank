Always use the '=' operator [PROPS.EQOP]



DESCRIPTION 


This rule identifies any property operator that is not '='. An error
is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v8.1
PARAMETERS 

-Ignore property file name (regular expression)
    -Allows you to provide a list of property files to ignore. If a property file 
     matches any of the regular expressions, it will be exempt from this rule.
BENEFITS 

It is common to use the '=' operator in property files. Using the '=' operator will increase
the file's readability. Also, it can detect a missing line continuation character '/' on 
the preceding line.
EXAMPLE 

a  A   // VIOLATION
b : B  // VIOLATION
c = C
b e  // VIOLATION : missing '/; on the preceding line
REPAIR 

a = A  // FIXED
b = B  // FIXED
c = C /  // FIXED
b e
