Do not use lowercase letters in "final" "static" field names [NAMING.USF]



DESCRIPTION 



This rule identifies "final" "static" fields (i.e. named constants) whose
names contain lowercase letters. An error is reported if a "final" "static" 
field name does not consist solely of uppercase letters, underscores and numbers.
SINCE 

v4.5
BENEFITS 

Conforms to the Java API Language Specification's naming conventions.
EXAMPLE 

package examples.rules.naming;

public class USF {
    public static final int size = 10;  // VIOLATION
}
REPAIR 

Make sure that the name consists solely of uppercase letters, underscores and
numbers.

package examples.rules.naming;

public class USFFixed {
    public static final int SIZE = 10;  // FIXED
}
REFERENCES 

"Java API Language Specification", section 6.8.5.

Section 9 of http://java.sun.com/docs/codeconv/html/CodeConventions.doc8.html#367
