Follow class name conventions defined for annotations [NAMING.ANNS]



DESCRIPTION 


This rule checks naming conventions for classes with type annotations.
You can configure it to check conventions for different annotations. It reports
a violation for each class name that does not follow the defined convention.
SINCE 

v9.2
PARAMETERS 

The  "Configure expected name for class using annotation" table has two columns:

- first "Expected class name"
The regular expression that classes with the specified annotations (configured in the
second column) should follow.

- second "Fully qualified name of annotation" 
The annotations used with classes that should follow this naming convention.
BENEFITS 

This rule enforces naming conventions for classes with annotations such as @Controller, @Model, and others.
EXAMPLE 

Create a Test Configuration with this rule, and add a row to the parameterization table in the rule configuration. 
In first column, add the regular expression ^.*Deprecated$. In the second, add java.lang.Deprecated.
Then, run this Test Configuration against the following example:

package examples.rules.naming;

@Deprecated
public class Person {  // VIOLATION

}
REPAIR

package examples.rules.naming;
