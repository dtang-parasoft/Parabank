Do not explicitly call 'System.gc()' or 'Runtime.gc()' [GC.GC]



DESCRIPTION 


This rule identifies explicit calls of the "System.gc()" or 
"Runtime.gc()" garbage collector method. An error is reported
for each occurrence.
SINCE 

v6.0
BENEFITS 

Garbage collection could be very costly and should be avoided
by all means-- except in benchmarking code. The garbage
collector will be executed automatically by the JVM whenever it is
appropriate. If you want to gain more control over garbage
collection, you should write your own code instead of calling "gc()".
EXAMPLE 

package examples.rules.gc;

public class GC {
    public void myMethod() {
        try {
            //...
        } finally {
            //...
            System.gc(); //VIOLATION
        }
    } 
}
REPAIR 

package examples.rules.gc;

public class GCFixed {
    public void myMethod() {
        try {
            //...
        } finally {
            //...
            //FIXED, System.gc() is removed
        }
    }
}
