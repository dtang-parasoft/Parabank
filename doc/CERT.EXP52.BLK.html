Provide a '{}' block for conditional statements [CERT.EXP52.BLK]



DESCRIPTION 


Requires that conditional statements have a "{ }" block.
Conditional statements include "if", "else", "for", "while"
and "do-while" statements.
SCOPE LEVEL 
LINE
SINCE 

v6.0
SECURITY RELEVANCE 

"[T]his error may be introduced it into a production environment and provide
additional attack vectors by creating a control flow path leading to
an unexpected state in the application. The consequences will depend on
the types of behaviors that are being incorrectly executed." [CWE-483]
PARAMETERS 

-Check "if" statements (default as true)
-Check "else" statements (default as true)
-Check "for" statements (default as true)
-Check "while" statements (default as true)
-Check "do-while" statements (default as true)
BENEFITS 

Conditional statements with "{}" blocks are less error-prone than those 
without "{}" blocks.
EXAMPLE 

package examples.rules.codsta.bp;

public class BLK {
    public void method () {
        for (int i = 0; i < 10; i++)  // VIOLATION
            System.out.println (i * i);  // inside the loop
            System.out.println (i);      // outside the loop
    }
}
REPAIR 

Provide a block for each "for" statement.


package examples.rules.codsta.bp;

public class BLKFixed {
    public void method () {
        for (int i = 0; i < 10; i++) {  // FIXED
            System.out.println (i * i);  // inside the loop
            System.out.println (i);      // now inside the loop
        }
    }
}
