Declare all fields of user-defined 'Exception' as "final" [EXCEPT.IMMEX]



DESCRIPTION 


This rule identifies fields of user-defined exception classes 
that are not declared as "final" or "private". An error is reported
for each occurrence.
SINCE 

v6.0
NOTES 

Static fields would not be identified as errors.
BENEFITS 

Exception classes are supposed to carry exception information
that other classes should not be able to modify. Consequently, it makes sense to 
declare the fields as final or private to guard them from unauthorized
modifications.
EXAMPLE 

package examples.rules.except;

public class IMMEX extends java.lang.Exception { //VIOLATION
    String extraMessage;
    int severity;
    //...
}
REPAIR 

package examples.rules.except;

public class IMMEXFixed extends java.lang.Exception {
    final String extraMessage;
    private int severity;
    //...
}
