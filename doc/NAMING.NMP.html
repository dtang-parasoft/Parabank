Use a naming convention for method parameters [NAMING.NMP]



DESCRIPTION 


This rule identifies method parameters that do not follow the user-defined
naming convention (as specified in a regular expression).  By default, the rule 
enforces that the names of method parameters begin with a lower case letter.  An 
error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-regular expression
    -Method parameter names should match the listed regular expression.
    -The default is "^[a-z]".  This enforces that the names of method parameters 
     begin with a lower case letter.
-Negate regular expression
    -If this parameter is enabled, a violation will be flagged if the name of a
     method parameter matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for method parameter names which do not match the regular 
     expression specified.
-Ignore case
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of method 
     parameters must match the regular expression specified, including in case.
BENEFITS 

This helps enforce a naming convention.
