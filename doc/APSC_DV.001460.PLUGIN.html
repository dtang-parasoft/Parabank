Ensure Plugins are added in the 'struts-config.xml' [APSC_DV.001460.PLUGIN]



DESCRIPTION 


This rule identifies if the plugin names which are listed in the rule parameter 
does not exist in 'struts-config.xml'. An error is reported for each occurrence.
SINCE 

v8.4
SECURITY RELEVANCE 

Using the Struts Validator to check all program input before it is processed by 
the application can prevent vulnerabilities that result from unchecked input.
PARAMETERS 

-Checks whether the following Plugins exist in the 'struts-config.xml'
    -By default, checks "org.apache.struts.validator.ValidatorPlugIn" plugin
EXAMPLE 

<?xml version="1.0" ?>
<struts-config>
    .....
    <!-- VIOLATION : missing plugin "org.apache.struts.validator.ValidatorPlugIn" -->
</struts-config>
REPAIR 

<?xml version="1.0" ?>
<struts-config>
    .....
    <!-- FIXED : add plugin "org.apache.struts.validator.ValidatorPlugIn" -->
    <plug-in className="org.apache.struts.validator.ValidatorPlugIn">
        <set-property property="pathnames"
                      value="/WEB-INF/validator-rules.xml,/WEB-INF/validation.xml"/>
    </plug-in>
</struts-config>
