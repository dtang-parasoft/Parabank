Do not hard code '\n' or '\r' as a line separator [PORT.LNSP]



DESCRIPTION 


This rule identifies strings that contain hard coded '\n' or '\r' line 
separator characters. An error is reported for each occurrence.
SINCE 

v4.5
BENEFITS 

Different systems use different characters or characters sequences as 
line separators.  Consequently, hard coding '\n', '\r', or "\r\n" violates 
the portability of Java code.
EXAMPLE 

package examples.rules.port;

public class LNSP {
    public void sayHello (String name) {
        System.out.println("Hello\n" + name);  // VIOLATION
    }
}
REPAIR 

Use the println() method of PrintStream or PrintWriter, which automatically 
terminates a line with the line separator appropriate for the platform, or use 
the value of System.getProperty("line.separator").

package examples.rules.port;

public class LNSPFixed {
    public void sayHello (String name) {
        System.out.println("Hello");  // FIXED
        System.out.println(name);  // FIXED
    }
}
