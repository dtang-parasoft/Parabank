Declare a "public/protected" "class" "final" if it is not subclassed [GLOBAL.SPPC]



DESCRIPTION 


This rule identifies "public" or "protected" classes that have no subclasses
and that are not declared "final". An error is reported for each occurrence.

Global analysis is required in this rule.
SCOPE LEVEL 
CLASS
SINCE 

v4.5
SECURITY RELEVANCE 

category: Weak Security Controls
category: Code Quality

Declare public and protected classes to be final to avoid malicious overriding of
privileged code. If a malicious subclass is introduced, it can be used to access
and modify data, which is a security problem.
PARAMETERS 

-Check unused classes (Default value is "false")
BENEFITS 

1) It optimizes the code: because the compiler knows that nobody
   can extend the class or override its methods, it
   can generate optimized code.

2) It makes the code self documenting: somebody looking at the
   class will know that no other class extends this class.
REPAIR 

Make the class "final" or document the reason for the
class not being final.
