Do not use complicated conditional expressions in control structures [CODSTA.READ.ACCS]



DESCRIPTION 


This rule identifies complicated conditional expressions which are used as 
the conditions in control structures.  By default, the rule only checks the
conditions of "if" statements.  It can be parameterized to also check the 
conditions of "for" statements, "do" statements, and "while" statements.  
With the default settings of the parameters, an error is reported if there 
are more than 2 operators in an "if" statement.
SCOPE LEVEL 
LINE
SINCE 

v5.0
PARAMETERS 

-Maximum number of operators (default value is 2)
-Check "if" statements (default value is "true")
-Check "for" statements (default value is "false")
-Check "do" statements (default value is "false")
-Check "while" statements (default value is "false")
BENEFITS 

Following this rule makes code more readable and less error-prone.
EXAMPLE 

package examples.rules.codsta.read;

public class ACCS {

    public void method(int i, int j, int k, int l) {

        if (i > 10 && j > 11 && k > 12 && l > 13) {  // VIOLATION

        }
    }
}
REPAIR 

Create a method to handle the complicated conditional statement.

package examples.rules.codsta.read;

public class ACCSFixed {

    public boolean isConditionTrue(int i, int j, int k, int l) {

        return (i > 10 && j > 11 && k > 12 && l > 13);

    }

    public void method(int i, int j, int k, int l) {

        if (isConditionTrue(i, j, k, l)) {  // FIXED
    
        }
    }
}
