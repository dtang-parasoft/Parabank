Serialization [SERIAL] 

Rules that enforce proper serialization.
RULES

Create a 'serialVersionUID' for all 'Serializable' classes [SERIAL.DUID]
Avoid classes that implement 'Externalizable' but do not define a no-argument constructor [SERIAL.ENNAC]
Implement Externalizable instead of Serializable [SERIAL.EZEE]
Avoid declaring "transient" fields in non-serializable classes [SERIAL.FT]
Avoid re-initializing fields in the 'readExternal()' method of 'Externalizable' classes [SERIAL.IRX]
Ensure the return type of 'readResolve()' and 'writeReplace()' methods are 'java.lang.Object' [SERIAL.MRWD]
Ensure Serializable classes are correct [SERIAL.NSFSC]
Ensure outer class is serializable if its inner class is serializable [SERIAL.OC]
Ensure that the 'readObject()' and 'writeObject()' methods have the correct signature [SERIAL.ROWO]
Define a "readResolve" method for all instances of Serializable types [SERIAL.RRSC]
Ensure that all fields are assigned by the 'readObject()' method and written out by the 'writeObject()' method [SERIAL.RWAF]
Always declare writeObject and readObject methods for Serializable subclasses of non-Serializable parents [SERIAL.SCBNP]
Avoid serializable classes that extend a superclass without a zero-argument constructor [SERIAL.SNNAC]
Do not store non-serializable objects as HttpSession attributes [SERIAL.SNSO]
Declare 'serialPersistentFields' "private static final" [SERIAL.SPF]
Do not declare SerialVersionUID in classes that do not implement Serializable [SERIAL.SRLZ]
Do not declare the 'readObject()' method as "synchronized" [SERIAL.SROS]
Validate objects before deserialization [SERIAL.VOBD]
