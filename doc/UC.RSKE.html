Avoid redundant 'static' keywords in enum type declarations [UC.RSKE]



DESCRIPTION 


This rule will flag a violation for each enum type which is explicitly declared 
as "static".
SCOPE LEVEL 
LINE
SINCE 

v8.0
BENEFITS 

Enum types are automatically static, so there is no need to use the "static" 
keyword when declaring an enum type.  Doing so will unnecessarily complicate the
code.
EXAMPLE 

package examples.rules.uc;

public class RSKE {
    public static enum Rank { DEUCE, THREE, FOUR, FIVE, SIX, //VIOLATION
        SEVEN, EIGHT, NINE, TEN, JACK, QUEEN, KING, ACE }
}
REPAIR 

package examples.rules.uc;

public class RSKEFixed {
    public enum Rank { DEUCE, THREE, FOUR, FIVE, SIX,        //FIXED
        SEVEN, EIGHT, NINE, TEN, JACK, QUEEN, KING, ACE }
}
