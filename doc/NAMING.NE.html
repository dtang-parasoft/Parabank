Use a naming convention for exceptions [NAMING.NE]



DESCRIPTION 


This rule identifies exceptions that do not follow the user-defined
naming convention (as specified in a regular expression). By default, the
rule requires that the exception name end with 'Exception'. An 
error is reported for each occurrence.
SCOPE LEVEL 
CLASS
SINCE 

v4.5
PARAMETERS 

-regular expression
    -Exception class names should match the listed regular expression.
    -The default is "Exception$".  This enforces that the names of exception
     classes end with "Exception".
-check public class. (default is true)
-check protected class. (default is true)
-check package-private class. (default is true)
-check private class. (default is true)
-Negate regular expression
    -If this parameter is enabled, a violation will be flagged if the name of an
     exception class matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for exception class names which do not match the regular expression
     specified.
-Ignore case
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of exception
     classes must match the regular expression specified, including in case.
BENEFITS 

This helps enforce a naming convention.
EXAMPLE 

package examples.rules.naming;

public class NE {  

    class NEInvalidFormat extends Exception {  // VIOLATION

    }
}
REPAIR 

Change the identifier to follow the convention.

package examples.rules.naming;

public class NEFixed {  

    class NEInvalidFormatException extends Exception {  // FIXED 

    }
}
