Present "import" statements in alphabetical order [CODSTA.ORG.ORIMP]



DESCRIPTION 


This rule identifies improperly-ordered "import" statements. An error is 
reported when "import" statements are not presented in correct order.
SCOPE LEVEL 
FILE
SINCE 

v5.0
PARAMETERS 

-Lower case letters should come before upper case letters
    -Default is "false".
    -The default lexicographic ordering is for upper case letters to come 
     before lower case letters.  However, some people prefer to have lower 
     case letters come before upper case letters.  If you prefer to swap the
     default ordering for lower case and upper case letters, you should 
     enable this parameter.
BENEFITS 

Makes code easier to read and less error-prone.
EXAMPLE 

package examples.rules.codsta.org;

import java.util.Vector;
import java.util.Stack;  // VIOLATION

public class ORIMP {
}
REPAIR 

Reorder the import statements.

package examples.rules.codsta.org;

import java.util.Stack;
import java.util.Vector; // FIXED

public class ORIMPFixed {
}
