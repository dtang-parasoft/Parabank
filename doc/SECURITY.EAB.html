Erratic Application Behavior [SECURITY.EAB]

RULES

Avoid calling specified methods from web components and EJBs [SECURITY.EAB.ACWC]
Do not use AWT classes in Web components [SECURITY.EAB.AWT]
Do not pass byte arrays to ObjectOutputStream in the 'writeObject()' method [SECURITY.EAB.CBA]
Do not compare Class objects by name [SECURITY.EAB.CMP]
Enforce returning a defensive copy in 'clone()' methods [SECURITY.EAB.CPCL]
Do not stop the JVM in a web component [SECURITY.EAB.JVM]
Limit the number of "AccessController.doPrivileged" calls per class [SECURITY.EAB.LDP]
Do not pass user-given mutable objects directly to certain types [SECURITY.EAB.MPT]
Implement 'readObject()' and 'writeObject()' for all 'Serializable' classes [SECURITY.EAB.OROM]
Limit the number of lines in "privileged" code blocks [SECURITY.EAB.PCL]
Do not declare "static" fields in web components [SECURITY.EAB.SF]
Do not change the input streams of 'java.lang.System' in a web component [SECURITY.EAB.SIS]
Do not store user-given mutable objects directly into variables [SECURITY.EAB.SMO]
Inspect 'static' fields which may have intended to be declared 'static final' [SECURITY.EAB.SPFF]
