Follow a naming convention for setter methods [NAMING.SETA]



DESCRIPTION 


This rule identifies setter methods whose names do not follow the user-defined
naming convention (as specified in a regular expression).  By default, the rule 
requires that the name of each setter method begin with "set".  An error is 
reported for each occurrence.
SCOPE LEVEL 
METHOD
SINCE 

v4.5
PARAMETERS 

-regular expression
    -Setter method names should match the listed regular expression.  
    -The default is "^set.".  This enforces that the names of setter methods 
     begin with "set".)
-check public method. (default is true)
-check protected method. (default is true)
-check package-private method. (default is true)
-check private method. (default is false)
-Negate regular expression
    -If this parameter is enabled, a violation will be flagged if the name of a
     setter method matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be 
     flagged for setter method names which do not match the regular expression
     specified.
-Ignore case
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of setter
     methods must match the regular expression specified, including in case.
BENEFITS 

This follows the JavaBeans naming conventions.  Also, using the 'set' prefix 
makes it clear that the method is a mutator that sets the value of the field.
EXAMPLE 

package examples.rules.naming;

public class SETA {
    public void method(int count) {  // VIOLATION
        _count = count;
    }
    private int _count = 0;
}
REPAIR 

Change the setter method name to 'set<Field name>'.

package examples.rules.naming;

public class SETAFixed {
    public void setCount(int count) {  // FIXED
        _count = count;
    }
    private int _count = 0;
}
