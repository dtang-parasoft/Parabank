Implement a no-argument 'ejbCreate()' method for all Message-driven bean classes [EJB.MDBC]



DESCRIPTION 


This rule identifies message-driven bean classes that do not 
implement a no-argument 'ejbCreate()' method. An error is reported 
if a class implementing the "javax.ejb.MessageDrivenBean" interface
does not have a method named 'ejbCreate()' with no arguments.
SCOPE LEVEL 

CLASS
SINCE 

v4.5
BENEFITS 

According to the EJB specification (version 2.0), message-driven
bean classes must implement an 'ejbCreate()' method with no 
arguments.
REFERENCES 

Sections 15.4.4 (p. 316) of the EJB specification (version 2.0).  
The EJB specification is available online at
http://download.oracle.com/otndocs/jcp/7294-ejb-2.0-fr2-spec-oth-JSpec/
