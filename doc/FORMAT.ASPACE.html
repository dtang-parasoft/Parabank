Ensure proper spacing in array references [FORMAT.ASPACE]



DESCRIPTION 


This rule identifies code that has incorrect spacing before array references.
The default number of spaces between a type name or variable name and a bracket
is set to 0.
SINCE 

v9.0
PARAMETERS 

-Number of spaces before the opening '[' bracket (default is 0)
BENEFITS 

If all team members follow the same formatting style, it will be easier for each
team member to read and understand the code.
EXAMPLE 

package examples.rules.format;

public class ASPACE {
    int     [] array = new int   [5]  //VIOLATION
}
REPAIR 

Remove spaces between the array reference '[' bracket.

package examples.rules.format;

public class ASPACEFIXED {
    int[] array = new int[5]  //FIXED
}
REFERENCES 

DISA Application Security and Development Security Technical Implementation Guide (STIG)
APSC-DV-003215: The application development team must follow a set of coding standards
