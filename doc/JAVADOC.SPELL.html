Avoid misspelling words in Javadoc comments and string literals [JAVADOC.SPELL]



DESCRIPTION 


This rule identifies word that is misspelled. An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v8.0
NOTES 

This rule checks the following content:
   - description in Javadoc comment
   - value of '@param' tag
   - value of '@exception' tag
   - value of '@deprecated' tag
   - value of '@verbose' tag
   - string literal
PARAMETERS 

- Ignore length of word less than (default value is 3)
- Ignore uppercase word (turn on by default)
- Ignore words in string literal (turn on by default)
- A list of ignored words (separated with a semicolon)
BENEFITS 

Misspelled word will make javadoc hard to read and understand.
EXAMPLE 

package examples.rules.javadoc; 

/**
 * Spull class    // VIOLATION
 */
public class SPELL { 

}
REPAIR 

Fixed word 'Spull' to 'Spell'

package examples.rules.javadoc; 

/**
 * Spell class   // FIXED
 */
public class SPELLFixed {

}
