Poor Object Oriented Design [CODSTA.POD]

RULES

Avoid constant interface anti-pattern [CODSTA.POD.ACIAP]
Avoid "static" methods when the declaring class is a parameter type [CODSTA.POD.ASM]
Use less specific types to accomplish loose coupling [CODSTA.POD.AUVT]
Use chain constructors in classes with multiple constructors [CODSTA.POD.CHAIN]
Avoid chains of "instanceof" comparisons [CODSTA.POD.CIOC]
Define a no argument constructor whenever possible [CODSTA.POD.DCTOR]
Do not access static members indirectly [CODSTA.POD.IASM]
Do not define constants in interfaces [CODSTA.POD.ISACF]
Avoid "switch" statements with too many or too few "case" statements [CODSTA.POD.SMC]
Use "enum" types instead of a series of "static final" constants [CODSTA.POD.UET]
Enforce or avoid the use of parameterized types [CODSTA.POD.UPT]
