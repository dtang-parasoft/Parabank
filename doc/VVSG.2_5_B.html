2.5-B: Unsafe concurrency [VVSG.2_5_B]

RULES

Do not use variables of the unsafe type 'java.lang.ThreadGroup' [VVSG.2_5_B.AUTG]
Do not cause deadlocks by calling a "synchronized" method from a "synchronized" method [VVSG.2_5_B.CSFS]
Avoid unsafe implementations of the "double-checked locking" pattern [VVSG.2_5_B.DCL]
Variable should be used in context of single critical section [VVSG.2_5_B.DIFCS]
Avoid double locking [VVSG.2_5_B.DLOCK]
Do not use an instance lock to protect shared static data [VVSG.2_5_B.INSTLOCK]
Do not abandon unreleased locks [VVSG.2_5_B.LOCK]
Ensure that nested locks are ordered correctly [VVSG.2_5_B.LORD]
Release Locks in a "finally" block [VVSG.2_5_B.RLF]
Do not synchronize on "public" fields since doing so may cause deadlocks [VVSG.2_5_B.SOPF]
Avoid calling unsafe deprecated methods of 'Thread' and 'Runtime' [VVSG.2_5_B.THRD]
Do not call 'Thread.sleep()' while holding a lock since doing so can cause poor performance and deadlocks [VVSG.2_5_B.TSHL]
Do not use blocking methods while holding a lock [VVSG.2_5_B.TSHL2]
