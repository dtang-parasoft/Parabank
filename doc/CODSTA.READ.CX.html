Enforce or avoid usage of conditional operators [CODSTA.READ.CX]



DESCRIPTION 


This rule can be set to identify either code that does or does not use 
conditional operators (? :). An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v6.0
PARAMETERS 

-Option
    -Enforce usage of conditional operators (default option)
        -Use conditional operator instead of 'if (cond) a = b; else a = c;' (default is true)
        -Use conditional operator instead of 'if (cond) return a; else return b;' (default is true)
    -Avoid usage of conditional operators
BENEFITS 

-Enforce usage of conditional operators
    The conditional operator provides a single expression that yields 
    one of two values based on a boolean expression. This results in a more
    compact expression.
    
-Avoid usage of conditional operators
    Although inline conditional operator could reduce the size of the
    code, they could make the code difficult to read and understand. 
    Consequently, some people prefer to use if-else statements instead of
    conditional expressions (a ? b : c).
EXAMPLE 

By default, the rule enforces usage of conditional operators (? :)

package examples.rules.codsta.read;

public class CX {
    public int getAbsoluteValue (int i) {  
        if (i>= 0) {    // VIOLATION
            return i;    
        }
        else {
            return i* -1;
        }
    }
}
REPAIR 

Use conditional operator

package examples.rules.codsta.read;

public class CXFixed {
    public int getAbsoluteValue (int i) {  
        return i>= 0 ? i : i* -1; //FIXED
    }
}
