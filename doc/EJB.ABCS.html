Do not use Servlet code in EJB classes [EJB.ABCS]



DESCRIPTION 


This rule identifies Servlet code inside EJB classes. An error
is reported for each occurrence.
SCOPE LEVEL 

CLASS
SINCE 

v5.1
SECURITY RELEVANCE 

category: Backdoor Validation
category: Input Validation

See BENEFITS.
BENEFITS 

Implementation details specific to one tier should not be
introduced in another tier. The service API exposed by the
business tier to the presentation tier will likely be used
by other clients as well. If the service API accepts 
parameters with types, such as HttpServletRequest, then
every client to the service is forced to package its data
in a servlet request data structure. This drastically 
reduces the service's reusability.
REFERENCES 

"Core J2EE Patterns - Best Practices and Design Strategies"
