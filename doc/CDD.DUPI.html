Avoid duplicate import statements [CDD.DUPI]



DESCRIPTION 


This rule looks for duplicated "import" statements.  The rule will flag a 
violation for each import statement where the object being imported has already
been imported.
SCOPE LEVEL 
LINE
SINCE 

v8.0
NOTES 

This rule will not flag a violation for the case where a class is imported and 
the package in which it is defined is imported separately using the "*" form of
import.
BENEFITS 

Duplicate import statements serve no purpose and are usually just a careless 
mistake.  They should be eliminated to help clean up the code.
EXAMPLE 

package examples.rules.cdd;

import java.util.Vector;
import java.util.Vector;  //VIOLATION
import javax.naming.*;
import javax.naming.*;  //VIOLATION

public class DUPI {
    Vector v;
    NamingException ne;
}
REPAIR 

package examples.rules.cdd;

import java.util.Vector;  //FIXED
import javax.naming.*;  //FIXED

public class DUPIFixed {
    Vector v;
    NamingException ne;
}
