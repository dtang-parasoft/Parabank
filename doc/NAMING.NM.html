Use a naming convention for non-"static" methods [NAMING.NM]



DESCRIPTION 


This rule identifies non-"static" methods that do not follow the user-defined
naming convention (as specified in a regular expression). By default, the
rule requires that each non-"static" method name starts with a lower case character.
An error is reported for each occurrence.
SCOPE LEVEL 
METHOD
SINCE 

v4.5
PARAMETERS 

-regular expression
    -Non-"static" method names should match the listed regular expression.
    -The default is "^[a-z]".  This enforces that the names of non-"static"
     methods begin with a lower case letter.
-check public method. (default is true)
-check protected method. (default is true)
-check package-private method. (default is true)
-check private method. (default is true)
-Negate regular expression
    -If this parameter is enabled, a violation will be flagged if the name of a
     non-"static" method matches the specified regular expression.
    -By default, this parameter is off.  This means that a violation will be
     flagged for non-"static" method names which do not match the regular
     expression specified.
-Ignore case
    -If this parameter is enabled, the regular expression will not be case
     sensitive.
    -By default, this parameter is off.  This means that the names of non-
     "static" methods must match the regular expression specified, including in
     case.
BENEFITS 

This helps enforce a naming convention.
EXAMPLE 

package examples.rules.naming;

public class NM {
    void Method () {  // VIOLATION
    }
}
REPAIR 

Change the identifier to follow the convention.

package examples.rules.naming;

public class NMFixed {
    void method () {  // FIXED
    }
}
