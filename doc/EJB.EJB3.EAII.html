Do not expose @AroundInvoke method in business interface [EJB.EJB3.EAII]



DESCRIPTION 


This rule flags a violation if a method annotated with "@AroundInvoke" is
exposed in the business interface. These methods are automatically invoked by
the container and should not be exposed to clients.
SINCE 

v8.4
EXAMPLE 

package examples.rules.ejb.ejb3;

@Stateless
public class EAII implements EAIIInterface {
    @AroundInvoke
    public void method(InvocationContext ic) throws Exception { //VIOLATION
        ...
    }
}

@Remote
interface EAIIInterface {
    public void aroundInvokeMethod(InvocationContext ic) throws Exception;
}
REPAIR 

Remove the method from the interface.

package examples.rules.ejb.ejb3;

@Stateless
public class EAIIFixed implements EAIIFixedInterface {
    @AroundInvoke
    public void method(InvocationContext ic) throws Exception {
        ...
    }
}

@Remote
interface EAIIFixedInterface {
    //FIXED
}
REFERENCES 

Debu Panda, Reza Rahman, and Derek Lane. EJB3 in Action.
