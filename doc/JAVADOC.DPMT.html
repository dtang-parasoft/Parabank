Avoid unused Javadoc tags [JAVADOC.DPMT]



DESCRIPTION 


This rule identifies unused Javadoc tags in method comments. An error is 
reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v5.0
NOTES 

This rule will not report an error for any Design by Contract (DbC) tags.

The "Quick Fix All" feature is not available for this rule because removing the
unused Javadoc tag is often not a good fix.  Many times, the reason why the
Javadoc tag is unused is because the user changed the method definition but
forgot to update the Javadoc.  In these cases, the Quick Fix should not be
applied.
PARAMETERS 

-Check unused @param tags (default is true)
-Check unused @return tags (default is true)
-Check unused @throws and @exception tags (default is true)
    -ignore @throws and @exception tags of 'RuntimeException' (default is true)
        If enabled, the rule would check the exceptions used by the tags.
        If they are any of the runtime exceptions, then no violation would be
        reported even if they are not declared in "throws" clauses.
-Check unused @see and @link tags (default is true)
BENEFITS 

Leaving unused Javadoc tags in methods can lead to confusion. In general,
documentation that does not reflect the actual code behavior may cause
confusion and bugs.
EXAMPLE 

package examples.rules.javadoc;

public class DPMT {

    /**
     * @param id1
     * @return int
     */ // VIOLATION
    public void setId (String id) {
        _id = id;
    }
    private String _id;
}
REPAIR 

Change '@param id1' to @param id.
Remove '@return int'.

package examples.rules.javadoc;

public class DPMTFixed {

    /**
     * @param id is set as the new id.
     */  // FIXED
    public void setId (String id) {
        _id = id;
    }
    private String _id;
}
