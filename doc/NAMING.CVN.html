Use conventional variable names [NAMING.CVN]



DESCRIPTION 



This rule identifies code that uses local variable names in
unconventional ways. An error is reported if code does not use local
variables for their conventional purposes, by default is

b for a byte
c for a char
d for a double
f for a float
i, j, and k for integers
l for a long
o for an Object
s for a String
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-Local variable naming convention table
  - In "Local variable" column
    User can input the name of local variable that
    is reserved to a specific type of local variable
  - In "Type" column
    User can input the type of the local variable. Fully
    qualified name is required, if the local variable is
    non-primitive type.
BENEFITS 

Misusing these variable names can lead to confusion.
EXAMPLE 

package examples.rules.naming;

public class CVN {
    void method () {
        int b = 1;  // VIOLATION
    }
}
REPAIR 

Use conventional variable names.

package examples.rules.naming;

public class CVNFixed {
    void method () {
        int i = 1;  // FIXED
    }
}
