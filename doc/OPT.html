Optimization [OPT] 

Rules that check for non-optimal constructs.
RULES

Change non-"private" members to "private" in anonymous classes [OPT.AAM]
Use abbreviated assignment operators [OPT.AAS]
Do not use the 'new String(String)' constructor [OPT.ACDO]
Do not get the 'Class' object through new object instantiation [OPT.AGC]
Avoid new instantiations of primitive wrapper objects, whose values are cached [OPT.ANIPW]
Avoid unnecessary Map operations [OPT.AUMO]
Simplify "boolean" returns [OPT.BR]
Close all "java.io.Closeable" resources in a "finally" block [OPT.CCR]
Avoid putting constant expressions inside loops [OPT.CEIL]
Do not call methods in loop condition statements [OPT.CEL]
Close input and output resources in "finally" blocks [OPT.CIO]
Check the logging level before calling potentially expensive logging operations [OPT.CLL]
Do not convert a value to a String by concatenating the empty String [OPT.CPTS]
Avoid duplicate calls to the "get" methods of "ResultSet" [OPT.CRSG]
Close resources as early as possible [OPT.CRWD]
Do not use a "private" field that is accessed in only one method; change it to a local variable [OPT.CTLV]
Define initial capacities for 'ArrayList', 'HashMap', 'HashSet', 'Hashtable', 'Vector' and 'WeakHashMap' [OPT.DIC]
Use 'String.length() == 0' instead of 'String.equals("")' [OPT.EES]
Use 'equalsIgnoreCase()' instead of calls to 'toLowerCase()' or 'toUpperCase()' [OPT.EIC]
Avoid calling expensive operations in the body of a loop [OPT.EOIL]
When calling "Math.ceil()", do not pass in an integer that has been cast to a floating point type [OPT.ICDPC]
Avoid accessing members which will require synthetic accessor methods [OPT.ICGA]
Do not iterate through Lists using the 'get()' method [OPT.ILUG]
Use 'System.arraycopy()' instead of using a loop to copy arrays [OPT.IRB]
Do not instantiate a class which contains only static fields and methods [OPT.ISC]
Move invariants outside of loops wherever possible [OPT.LIOL]
Do not instantiate variables in a loop body [OPT.LOOP]
Make getter and setter methods for instance fields "final" [OPT.MAF]
Avoid unnecessary comparisons to "null" [OPT.MUNC]
Avoid 'null' check before 'instanceof' [OPT.NCIO]
Use the 'nextInt()' method to generate a random integer [OPT.NIVND]
Avoid creating new instances of BigInteger and BigDecimal [OPT.NIW]
Declare "final" fields with a known compile time value as "static" [OPT.NSF]
Use 'charAt()' instead of 'startsWith()' for one character comparisons [OPT.PCTS]
Do not instantiate the wrapper classes for primitive types [OPT.PRIM]
Specify an initial 'StringBuffer' capacity [OPT.SB]
Avoid converting StringBuffer or StringBuilder to String to check length [OPT.SBL]
Use 'StringBuilder' instead of 'StringBuffer' when synchronization is not required [OPT.SBS]
Avoid using synchronized data structures for local variables [OPT.SDLS]
Declare member classes "static" if possible [OPT.SI]
Avoid using 'String.replaceAll()' with literal values [OPT.SRA]
Do not call ''String.toCharArray()'' unnecessarily [OPT.STCA]
Use single quotes instead of double quotes for single character string concatenation [OPT.STR]
Simplify methods that return strings that are always concatenated [OPT.STRBUF]
Avoid unnecessary calls to 'String' methods [OPT.STS]
Avoid use of if statements if they can be replaced with switch statements [OPT.SWIF]
Do not call a synchronized method inside of a loop [OPT.SYN]
Use collection size for target array in 'toArray()' [OPT.TOARRAY]
Do not compare boolean variables with "true" [OPT.UEQ]
Avoid unnecessary "instanceof" evaluations [OPT.UISO]
Do not call the methods from 'java.lang.Math' if the result is a constant which can be easily determined [OPT.UMATH]
Avoid unnecessary casting [OPT.UNC]
Use 'StringBuffer' instead of 'String' when concatenating strings [OPT.USB]
Use 'String' instead of 'StringBuffer' for constant strings [OPT.USC]
Avoid questionable uses of non-short-circuit logic [OPT.USCL]
Use 'stack' variables whenever possible [OPT.USV]
