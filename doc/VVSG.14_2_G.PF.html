Avoid unused "private" fields [VVSG.14_2_G.PF]



DESCRIPTION 


This rule identifies unused "private" fields. An error is reported for each 
occurrence.
SCOPE LEVEL 
CLASS
SINCE 

v4.5
SECURITY RELEVANCE 

"The variable's value is assigned but never used, making it a dead store. . .
This weakness could be an indication of a bug in the program or a deprecated
variable that was not removed and is an indication of poor quality. This could
lead to further bugs and the introduction of weaknesses." [CWE-563]
PARAMETERS 

-Ignore "final" fields (default is true)
 The "final" modifier could imply that the declared field is a constant
 for later use. Such a constant should be ignored.
BENEFITS 

Each private field makes instances of the class larger.  Mobile platforms have 
reduced resources, so optimizations are especially important.

An unused field could indicate a logical flaw in the corresponding
class, or it could be leftover from a previously-deleted section of code.
EXAMPLE 

package examples.rules.uc;

public class PF {
    private int _unusedField;  // VIOLATION
}
REPAIR 

Either correct the logic to use the field or remove the field.

package examples.rules.uc;

public class PFFixed {
      // FIXED
}
