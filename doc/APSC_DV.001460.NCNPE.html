Do not catch 'NullPointerException' [APSC_DV.001460.NCNPE]



DESCRIPTION 


This rule identifies any attempt to catch 'NullPointerException'.
An error is reported for each occurrence.
SINCE 

V6.0
SECURITY RELEVANCE 

category: Unsafe Error Handling and Logging

Following this rule may introduce security problems because
NullPointerExceptions that are thrown by the application might be
left uncaught.
BENEFITS 

'NullPointerException's are likely to be thrown by the VM in unexpected
situations; code should be fixed to handle the unexpected situations.
EXAMPLE 

package examples.rules.except;

public class NCNPE {
    public void myMethod() {
	try {
            //...
	} catch (NullPointerException e) {//VIOLATION
            //...
        } catch (IOException e) {
            //...
        }
    }
}
REPAIR 

package examples.rules.except;

public class NCNPEFixed {
    public void myMethod() {
	try {
            //...
        }
        //FIXED, the catch block is removed
        catch (IOException e) {
            //...
        }
    }
}
