Inspect 'static' fields which may have intended to be declared 'static final' [CWE.668.SPFF]



DESCRIPTION 


This rule identifies "static" fields which may have been intended to be declared
"static final" instead.

This rule will check the following types:
    - primitive types
    - java.lang.Boolean
    - java.lang.Byte
    - java.lang.Character
    - java.lang.Double
    - java.lang.Float
    - java.lang.Integer
    - java.lang.Long
    - java.lang.Short
    - java.lang.String

An error is reported for each occurrence.
SINCE 

v8.4
SECURITY RELEVANCE 

category: Erratic Application Behavior

A "static" field which is not also declared "final" may be changed by other
classes and break the logic of the application.
PARAMETERS 

-Check fields of accessibility
    -public (default is on)
    -protected (default is off)
    -package-private (default is off)
    -private (default is off)
-Check fields of type
    -primitive type (default is on)
    -primitive wrapper type (default is on)
    -java.lang.String (default is on)
EXAMPLE 

package examples.rules.security.eab;

public class SPFF {
    private static String ID = 1;  // VIOLATION
}
REPAIR 

Change the field to "static final".

package examples.rules.security.eab;

public class SPFF_Fixed {
    private static final String ID = 1;  // FIXED
}
