Enforce number of space character(s) after every semicolon [APSC_DV.003215.SAS]



DESCRIPTION 

    
This rule identifies semicolons that are not followed by the required number of space
characters. With the default parameters, an error is reported if any semicolon that appears
in the header of a 'for' loop is not followed by a single ' ' space character. 

Semicolons that appear as statement terminators are not subject to this rule. Semicolons
that are located at the end of a line or followed only by whitespace or
comments on the same line are also exempt from this rule. There is also a parameter where 
the user can specify whether or not to make an exception for the common idiom 'for (;;)'.
SCOPE LEVEL 
LINE
SINCE 

v4.5
PARAMETERS 

-Number of spaces after every semicolon (default value is 1)
-Ignore rule for the special case "for(;;)" (default is true)
   -When this parameter is true, a "for" statement of the form "for(;;)..." 
    (with no spaces around the semicolons) will be ignored regardless of the 
    setting of the other parameter.
BENEFITS 

If all team members follow the same formatting style, it will be easier for each
team member to read and understand the code.
EXAMPLE 

package examples.rules.format;

public class SAS {
    public void method () {
        for (int i = 0;i < 10;i++) {  // VIOLATION
            System.out.println ("i: " + i);
        }
    }
}
REPAIR 

Place a single space after each semicolon.

package examples.rules.format;

public class SASFixed {
    public void method () {
        for (int i = 0; i < 10; i++) {  // FIXED
            System.out.println ("i: " + i);
        }
    }
}
