6.2.4.5 Attacks on access control mechanisms [PCIDSS40.6245]

RULES

Use "read-only" AccessMode for Castor queries [PCIDSS40.6245.CQRO]
Avoid DNS lookups for decision making [PCIDSS40.6245.DNSL]
Avoid defining multiple security roles with the same name in 'web.xml' files [PCIDSS40.6245.DSR]
Avoid using the 'getRequestedSessionId' method from the 'HttpServletRequest' class [PCIDSS40.6245.HGRSI]
Always call 'HttpSession.invalidate()' before 'LoginContext.login()' [PCIDSS40.6245.ISL]
Avoid storing usernames and passwords in plain text in Castor 'jdo-conf.xml' files [PCIDSS40.6245.PCCF]
Avoid EJB 3 methods without security annotations [PCIDSS40.6245.PERMIT]
Avoid using plain text passwords in Axis 'wsdd' files [PCIDSS40.6245.PTPT]
Ensure proper session expiration [PCIDSS40.6245.RUIM]
Ensure that each security role referenced in a 'web.xml' file has a corresponding definition [PCIDSS40.6245.SRCD]
Ensure that sessions are configured to time out in 'web.xml' files [PCIDSS40.6245.STTL]
Protect against using unprotected credentials [PCIDSS40.6245.TDPASSWD]
Use the ''getSecure()'' and ''setSecure()'' methods to enforce the use of secure cookies [PCIDSS40.6245.UOSC]
Avoid storing unencrypted Hibernate usernames and passwords in 'web.xml' files [PCIDSS40.6245.UPWD]
Avoid using plain text passwords in Axis2 configuration files [PCIDSS40.6245.UTAX]
