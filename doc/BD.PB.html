Possible Bugs [BD.PB] 

Rules that check for possible bugs in the code
(for example, bugs that result from typos or 
common mistakes).
RULES

Avoid accessing arrays out of bounds [BD.PB.ARRAY]
Avoid incorrect shift operations [BD.PB.BADSHIFT]
Avoid conditions that always evaluate to the same value [BD.PB.CC]
Consistently check the returned value of non-void methods [BD.PB.CHECKRET]
Do not check for null after dereferencing [BD.PB.DEREF]
Always catch exceptions [BD.PB.EXCEPT]
Method getPermission() is supposed to be invoked when user-defined class loaders are applied [BD.PB.GETPERM]
Avoid infinite recursion [BD.PB.INFREC]
Avoid integer overflows [BD.PB.INTOVERF]
Do not create multiple buffered wrappers on a single byte or character stream [BD.PB.MULBUF]
Avoid use before explicit initialization [BD.PB.NOTEXPLINIT]
Avoid use of fields before initialization in constructors and static initializers [BD.PB.NOTINITCTOR]
Prevent external processes from blocking on input and output streams [BD.PB.PBIOS]
Avoid overwriting method parameters before each use [BD.PB.POVR]
Methods shall not call themselves, either directly or indirectly [BD.PB.RECFUN]
Restore prior object state on method failure [BD.PB.REVOBJ]
Ensure proper usage of StringBuilder/StringBuffer objects [BD.PB.SBONE]
Do not append null value to strings [BD.PB.STRNULL]
Avoid switch with unreachable branches [BD.PB.SWITCH]
Avoid unused values [BD.PB.VOVR]
Do not invoke overridable methods from the readObject() method [BD.PB.VREADOBJ]
Avoid division by zero [BD.PB.ZERO]
