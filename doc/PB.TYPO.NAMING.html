Do not give methods and fields the same name as the enclosing class or each other [PB.TYPO.NAMING]



DESCRIPTION 


This rule checks that fields and methods do not share the same name as their
enclosing class. It also checks that fields do not share the same name as a
member method.
SCOPE LEVEL 
CLASS
SINCE 

v4.5
PARAMETERS 
-check methods with same name as class
-check fields with same name as class
-check fields with same name as member methods
BENEFITS 

Fields and methods that share the same name as their enclosing class can lead
to confusion. In the case of methods, it may even be an error as the method may
have been intended to be a constructor instead. Fields that share the same name
as a member method can also cause confusion as to whether a member field or
method is attempting to be accessed.
EXAMPLE 

package examples.rules.pb;

public class NAMING {
    private NAMING NAMING; //VIOLATION
    class NAMING_1 {
        private boolean _field;
        public void NAMING_1() { //VIOLATION
            _field = true;
        }
    }
    class NAMING_2 {
        private boolean _field; //VIOLATION
        public void _field() {
            _field = true;
        }
    }
}
REPAIR 

package examples.rules.pb;

public class NAMINGFixed {
    private NAMING _field; //FIXED
    class NAMING_1 {
        private boolean _field;
        public NAMING_1() { //FIXED
            _field = true;
        }
    }
    class NAMING_2 {
        private boolean _field; //FIXED
        public void fieldMethod() {
            _field = true;
        }
    }
}
