Avoid classes with no accessible members [PB.USC.NACC]



DESCRIPTION 


If the class constructor is not publicly available and the class does not have
any non-private static methods or fields, the methods in the class are not
usable, because the class object cannot be instantiated from outside the class.

This rule detects classes that meet the following conditions:
- the class is defined as a top level class.
- the class does not have any non-private constructors.
- the class does not have any non-private static member fields, methods,
or types.
SINCE 

v7.5
NOTES 

The rule does not analyze interfaces.
BENEFITS 

This rule helps you ensure classes are usable and prevent programmer confusion.
EXAMPLE 

In the following example, the class does not have any non-private constructors
or non-private static methods, fields, or types.  

package examples.rules.pb.usc;

public class NACC {  // VIOLATION

    private static NACC _nacc;

    private NACC() {
    }

    public NACC getInstance() {
        if (_nacc == null) {
            _nacc = new NACC();
        }
        return _nacc;
   }
}
REPAIR 

To fix the code, declare the method as "static".

package examples.rules.pb.usc;

public class NACCFixed {  // FIXED

    private NACCFixed _nacc;

    private NACCFixed() {
    }

    public static NACCFixed getInstance() {
        if (_nacc == null) {
            _nacc = new NACCFixed();
        }
        return _nacc;
    }
}
