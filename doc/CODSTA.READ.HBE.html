Avoid using Hexadecimal binary exponents [CODSTA.READ.HBE]



DESCRIPTION 


This rule identifies the use of hexadecimal binary exponents for hexadecimal
floating-point literals.
SINCE 

v8.4
NOTES 

This rule only works on Java 5 source code as this new syntax was introduced in
JLS, 3rd edition.
BENEFITS 

This rule enhances readability as use of hexadecimal binary exponents is
uncommon and may confuse the reader. The "0x" prefix also commonly suggests
"int" or "long" literals and may cause an unexpected overloaded method to be
called. Additionally, the hexadecimal floating-point may be a typo.
EXAMPLE 

package examples.rules.codsta.read;
public class HBE {
    public void printNumber() {
        System.out.println(0x0ABp4); //VIOLATION
    }
}
REPAIR 

Replace the hexadecimal value with the corresponding decimal value.

package examples.rules.codsta.read;
public class HBEFixed {
    public void printNumber() {
        System.out.println(2736.0); //FIXED
    }
}
REFERENCES 

JLS, 3rd Edition - Floating-Point Literals
http://docs.oracle.com/javase/specs/jls/se5.0/html/lexical.html#3.10.2
