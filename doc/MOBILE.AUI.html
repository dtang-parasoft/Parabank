Avoid declaring "interface" types [MOBILE.AUI]



DESCRIPTION 


This rule will flag a violation if the declaring type is "interface".

This rule only applies to Android version 2.1 and older, 2.2 introduces a JIT.
SINCE 

v9.0
BENEFITS 

In embedded system, calling method through an "interface" reference will take 
longer time than calling method through "class" reference
EXAMPLE 


package examples.rules.mobile;

import java.util.*;

public class AUI {

    public void add(String key, String value) {
       Map map = new HashMap();  // VIOLATION
       map.put(key, value);
       //....
   }
       
}
REPAIR 


package examples.rules.mobile;

import java.util.*;

public class AUIFixed {

    public void add(String key, String value) {
       HashMap map = new HashMap();  // FIXED
       map.put(key, value);
       //....
   }
       
}
