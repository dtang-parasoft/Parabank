Provide an '@invariant' contract for all getter methods [DBC.IGM]



DESCRIPTION 


This rule identifies any class without a '@invariant' contract for "getter" method
An error is reported for each occurrence.
SCOPE LEVEL 
LINE
SINCE 

v7.5
BENEFITS 

All accessor methods to the class should have specification using invariant contracts
because those methods may be called at any time.
EXAMPLE 

package examples.rules.dbc;

/**
 * class IGM
 */
public class IGM {  // VIOLATION

    public int getSize () {  
        return _size;
    }
    private int _size;
}
REPAIR 

Provide an '@invariant' contract for the getSize() method.

package examples.rules.dbc;

/** 
 * @invariant (getSize() >= MIN_GETSIZE && getSize() <= MAX_GETSIZE)
 */
public class IGMFixed {  // FIXED
    /**
     * Automatically generated variable: MIN_GETSIZE
     */
    private static final int MIN_GETSIZE = 0;
    
    /**
     * Automatically generated variable: MAX_GETSIZE
     */
    private static final int MAX_GETSIZE = 100;
    
    public int getSize () {
        return _size;
    }
    private int _size;
}
