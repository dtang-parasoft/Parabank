Use abbreviated assignment operators [OPT.AAS]



DESCRIPTION 


This rule identifies code that does not use abbreviated assignment operators. 
An error is reported if an assignment statement does not use the abbreviated
assignment operators (e.g. "+=" instead of "+", "*=" instead of "*", etc.).
SCOPE LEVEL 
LINE
SINCE 

v4.5
BENEFITS 

Mobile platforms have reduced resources, optimizations are especially important.

Some compilers may run faster when code uses abbreviated assignment operators.
Using abbreviated assignment operators also makes code more concise.
EXAMPLE 

package examples.rules.opt;

public class AAS {
    private int _i = 0;

    void decrementByTwo () {
        _i = _i - 2;  // VIOLATION: should use "-="
    }
}
REPAIR 

Use abbreviated assignment operators.

package examples.rules.opt;

public class AASFixed {
    private int _i = 0;

    void decrementByTwo () {
        _i -= 2;  // FIXED
    }
}
