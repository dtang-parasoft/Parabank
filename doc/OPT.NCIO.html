Avoid 'null' check before 'instanceof' [OPT.NCIO]



DESCRIPTION 


This rule will flag cases where a 'null' check immediately precedes an
'instanceof' check.
SINCE 

v9.0
NOTES 

This rule checks expressions of the following forms:
    1) obj != null && obj instanceof Integer
    2) obj == null || !(obj instanceof Integer)
BENEFITS 

Mobile platforms have reduced resources, optimizations are especially important. 

The 'instanceof' check will return false if a 'null' object is checked.
Therefore, a preceding 'null' check is unnecessary and can simplify code if
removed.
EXAMPLE 

package examples.rules.opt;

public class NCIO {
    public boolean isInteger(Object obj) {
        return (obj != null && obj instanceof Integer); //VIOLATION 
    }
}
REPAIR 

Remove the unnecessary 'null' check.

package examples.rules.opt;

public class NCIOFixed {
    public boolean isInteger(Object obj) {
        return (obj instanceof Integer); //FIXED 
    }
}
